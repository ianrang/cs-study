# 기업 보안 인프라 구조

## 개요

기업 보안 인프라는 **방어 위치(Where)** 를 기준으로 6개 영역으로 구분된다.
각 영역은 독립적으로 존재하지 않으며, 다층 방어(Defense in Depth) 원칙에 따라 상호 연동되어 운영된다.

---

## 1. 신원 및 접근 관리 (IAM)

사용자의 신원을 확인하고, 자원에 대한 접근 권한을 통제하는 영역이다.

| 솔루션 | 설명 |
|--------|------|
| **IAM** (Identity and Access Management) | 입사~퇴사까지 사용자의 디지털 신원과 권한을 통합 관리하는 체계 |
| **MFA** (Multi-Factor Authentication) | 비밀번호 외에 OTP, 생체인증 등 추가 인증 요소를 요구하는 다중 인증 |
| **SSO** (Single Sign-On) | 한 번의 인증으로 복수의 시스템에 접근할 수 있는 통합 인증 |
| **PAM** (Privileged Access Management) | 시스템 관리자 등 특권 계정의 접근을 엄격히 통제하고 감사하는 솔루션 |

---

## 2. 엔드포인트 보안 (Endpoint Security)

사용자가 직접 사용하는 단말기(PC, 노트북, 모바일, 서버)를 보호하는 영역이다.

| 솔루션 | 설명 |
|--------|------|
| **EPP** (Endpoint Protection Platform) | 시그니처 기반으로 알려진 악성코드를 탐지/차단하는 기존 백신의 진화형 |
| **EDR** (Endpoint Detection and Response) | 엔드포인트의 행위를 실시간 모니터링하여 미지의 위협을 탐지하고 대응 |
| **NAC** (Network Access Control) | 보안 정책(백신 설치, 패치 적용 등)을 충족한 단말만 네트워크 접속을 허용 |

### EPP vs EDR

| 구분 | EPP | EDR |
|------|-----|-----|
| 탐지 방식 | 시그니처 매칭 (알려진 위협) | 행위 분석 (알려지지 않은 위협 포함) |
| 대응 방식 | 자동 차단 | 탐지 + 조사 + 대응 (격리, 포렌식) |
| 적용 시점 | 사전 예방 중심 | 사후 탐지/대응 포함 |

---

## 3. 네트워크 보안 (Network Security)

외부 인터넷과 내부 네트워크 사이의 경계를 방어하고, 내부 통신을 감시하는 영역이다.

| 솔루션 | 설명 |
|--------|------|
| **Firewall** (방화벽) | IP/포트 기반으로 허용된 트래픽만 통과시키는 1차 방어선 |
| **IPS/IDS** (침입 방지/탐지 시스템) | 패킷 패턴을 분석하여 알려진 공격 트래픽을 탐지/차단 |
| **VPN** (가상 사설망) | 외부에서 내부 네트워크로 접근할 때 암호화된 터널을 제공 |
| **WAF** (Web Application Firewall) | SQL Injection, XSS 등 웹 애플리케이션 대상 공격을 전문 방어 |
| **NDR** (Network Detection and Response) | 네트워크 트래픽을 AI/ML로 분석하여 비정상 통신 패턴을 탐지하고 대응 |

---

## 4. 데이터 보안 (Data Security)

조직의 핵심 정보 자산 자체를 보호하는 영역이다.
상세 내용은 [[01.data-security]] 참조.

| 솔루션 | 설명 |
|--------|------|
| **DLP** (Data Loss Prevention) | 데이터 이동 경로를 감시하여 비인가 유출을 차단 |
| **DRM** (Digital Rights Management) | 문서를 암호화하고 사용자별 열람 권한을 통제 |
| **DB 암호화/접근 제어** | 데이터베이스 내 민감 정보를 암호화하고 접근 이력을 감사 |

---

## 5. 클라우드 보안 (Cloud Security)

온프레미스에서 클라우드로 전환됨에 따라 등장한 보안 영역이다.

| 솔루션 | 설명 |
|--------|------|
| **CWPP** (Cloud Workload Protection Platform) | 클라우드 서버, 컨테이너, 서버리스 워크로드를 보호 |
| **CSPM** (Cloud Security Posture Management) | 클라우드 리소스의 보안 설정 오류(S3 퍼블릭 노출 등)를 탐지하고 교정 |
| **CASB** (Cloud Access Security Broker) | SaaS 이용 시 데이터 유출, 비인가 접근을 중개 지점에서 통제 |

---

## 6. 보안 관제 및 위협 대응 (Security Operations)

모든 보안 영역에서 발생하는 이벤트를 통합 수집하고, 위협을 탐지하여 대응하는 영역이다.

| 솔루션 | 설명 |
|--------|------|
| **SIEM** (Security Information and Event Management) | 전사 보안 장비의 로그를 수집/상관 분석하여 위협을 탐지 |
| **SOAR** (Security Orchestration, Automation and Response) | 탐지된 위협에 대한 대응 절차를 자동화(차단, 격리, 알림 등) |
| **XDR** (eXtended Detection and Response) | 엔드포인트, 네트워크, 클라우드 등 다중 영역의 텔레메트리를 통합 분석/대응 |

### SIEM vs XDR

| 구분 | SIEM | XDR |
|------|------|-----|
| 주요 기능 | 로그 수집 + 상관 분석 | 탐지 + 자동 대응 |
| 데이터 소스 | 이기종 장비의 로그 (범용) | 자사 보안 제품의 텔레메트리 (긴밀 연동) |
| 구축 복잡도 | 높음 (다수 벤더 연동) | 상대적으로 낮음 (단일 벤더 생태계) |
| 강점 | 컴플라이언스 감사, 장기 로그 보존 | 빠른 위협 탐지 및 즉각 대응 |

---

## 7. 제로 트러스트 (Zero Trust)

현대 보안 인프라를 관통하는 핵심 아키텍처 원칙이다.

### 핵심 원칙

**"Never Trust, Always Verify"** - 내부 네트워크에 있다는 이유만으로 신뢰하지 않는다.

### 기존 경계 보안과의 차이

| 구분 | 경계 보안 (Perimeter) | 제로 트러스트 (Zero Trust) |
|------|----------------------|--------------------------|
| 신뢰 기준 | 내부 네트워크 = 신뢰 | 신뢰하지 않음, 항상 검증 |
| 인증 시점 | 네트워크 진입 시 1회 | 매 요청마다 지속 검증 |
| 접근 권한 | 네트워크 접속 시 광범위 접근 | 최소 권한 원칙 (Least Privilege) |

### 구현 요소

제로 트러스트는 단일 제품이 아니라, 6개 보안 영역을 연동하여 구현하는 아키텍처이다.

```
[사용자 요청]
    ↓
IAM → 신원 확인 (MFA + SSO)
    ↓
Endpoint → 단말 보안 상태 검증 (EDR/EPP)
    ↓
Network → 마이크로세그멘테이션으로 최소 경로만 허용
    ↓
Data → 권한 기반 접근 통제 (DLP/DRM)
    ↓
Security Operations → 전 과정 실시간 모니터링 (SIEM/XDR)
```

---

## 8. 다층 방어와 공격 경로

보안 인프라는 공격자가 목표에 도달하기까지 여러 계층의 방어를 통과해야 하도록 설계한다.

```
공격자 → [네트워크 보안] Firewall/IPS 통과
       → [엔드포인트 보안] EPP/EDR 우회
       → [IAM] 인증 정보 탈취
       → [데이터 보안] DRM 암호화 해제
       → [데이터 보안] DLP 탐지 회피
       → [Security Ops] SIEM/XDR 미탐지
```

어느 한 계층이 뚫리더라도 다음 계층에서 차단할 수 있도록 중첩 방어하는 것이 핵심이다.
