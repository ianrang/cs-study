# 데이터 보안: DLP와 DRM

## 개요

데이터 보안은 조직의 핵심 정보 자산을 보호하는 영역이다.
대표적인 솔루션으로 **DLP**(유출 경로 차단)와 **DRM**(문서 자체 암호화)이 있으며, 이 둘은 상호 보완적으로 운용된다.

---

## 1. DLP (Data Loss Prevention, 데이터 유출 방지)

### 목적

기밀 정보(개인정보, 소스코드, 재무 데이터 등)가 조직 외부로 유출되는 것을 차단한다.

### 작동 방식

데이터가 이동하는 모든 경로(네트워크, USB, 이메일, 메신저, 클라우드 등)를 감시한다.
정책에 위반되는 전송 시도를 탐지하면 즉시 차단하고 로그를 기록한다.

**탐지 예시:**
- 주민등록번호 100건 이상 포함된 파일을 외부 메일로 전송 시도
- 기밀 등급 폴더의 파일을 USB로 복사 시도

### 유형

| 유형 | 설치 위치 | 차단 대상 |
|------|-----------|-----------|
| Endpoint DLP | 사용자 PC | USB 복사, 화면 캡처, 프린트 |
| Network DLP | 네트워크 관문 | 이메일, 웹 업로드, 메신저 전송 |

### 한계

DLP를 우회하여 파일이 외부로 반출되면, 해당 파일을 누구나 열람할 수 있다.

---

## 2. DRM (Digital Rights Management, 디지털 권한 관리)

기업 보안에서는 **E-DRM(Enterprise DRM)** 또는 **문서 보안**이라는 명칭으로 사용된다.

### 목적

문서 자체를 암호화하여, 유출되더라도 권한 없는 사용자는 열람할 수 없도록 보호한다.

### 작동 방식

1. 문서 생성/저장 시점에 파일 자체를 암호화한다
2. 사용자별로 차등 권한을 부여한다 (읽기, 수정, 인쇄 등)
3. 문서 열람 시 암호화 서버에 권한을 조회한 후, 허가된 범위 내에서만 접근을 허용한다

**권한 설정 예시:**
- A 팀장: 읽기 / 수정 / 인쇄 가능
- B 사원: 읽기만 가능
- 외부인: 열람 불가

### 한계

사용자 경험(UX)이 저하될 수 있으며, 에이전트 설치가 필요하다. 권한 서버 장애 시 문서 접근이 불가능해지는 가용성 리스크가 존재한다.

---

## 3. DLP vs DRM 비교

| 구분 | DLP | DRM |
|------|-----|-----|
| **보호 대상** | 데이터 이동 경로 | 데이터(파일) 자체 |
| **핵심 기술** | 내용 탐지 및 전송 차단 | 파일 암호화 및 권한 통제 |
| **유출 시** | 파일이 외부로 나가면 열람 가능 | 권한 없으면 열람 불가 |
| **보안 분류** | 데이터 보안 + 네트워크/엔드포인트 보안 | 데이터 보안 + 엔드포인트 보안 |

---

## 4. 실무 적용: 다층 방어 (Defense in Depth)

실무에서는 DLP와 DRM을 함께 운용하여 다층 방어 체계를 구성한다.

```
[문서 생성] → DRM 자동 암호화 (1차 보호)
     ↓
[외부 전송 시도] → DLP 차단 (2차 보호)
```

- **DRM**: 문서 생성 시점에 자동 암호화 → 유출되어도 열람 불가
- **DLP**: 암호화 여부와 무관하게, 비인가 전송 시도 자체를 차단

단일 솔루션의 한계를 보완하기 위해 여러 계층의 보안 통제를 중첩 적용하는 것이 다층 방어의 핵심 원칙이다.

## 5. 추가 데이터 보안 솔루션

| 솔루션 | 설명 |
|--------|------|
| **DB 암호화** | 데이터베이스에 저장된 개인정보(주민등록번호, 계좌번호 등)를 암호화하고 접근 이력을 기록 |
| **DB 접근 제어** | 데이터베이스 쿼리 수준에서 비인가 접근을 차단하고 감사 로그를 생성 |
