## 0. 들어가며 — 왜 "관리" 가 별도 단원인가

리눅스의 보안 수준은 **OS 가 제공하는 기능 자체** 보다 **관리자가 그 기능을 어떻게 설정했는가** 에 더 크게 좌우된다. 03 단원에서 본 `/etc/passwd` · `/etc/shadow` · 권한 비트 · `xinetd` · PAM · syslog 가 아무리 잘 설계되어 있어도, 패스워드 복잡성이 비활성이거나 `/etc/shadow` 가 644 로 노출되면 그 모두가 무력화된다.

이 단원에서 다루는 보안 설정 항목은 KISA 의 *주요정보통신기반시설 기술적·관리적 보호조치 기준* 의 UNIX 서버 점검 항목과 거의 1:1 로 대응한다. 자격증 1과목에서는 다음 다섯 영역이 반복 출제된다.

| 영역 | 핵심 포인트 |
|---|---|
| **계정 관리** | UID 0 유일성, 로그인 차단 셸, Session Timeout |
| **패스워드 정책** | 복잡성·최소 길이·최대/최소 사용기간·shadow 보호 |
| **파일·디렉터리** | PATH 환경변수, 소유자/소유그룹, world writable, 주요 파일 권한 |
| **서비스 관리** | Simple TCP·R 계열·RPC·익명 FTP 비활성화 |
| **로그 관리** | 원격 로그 서버 운영(514/udp), 실시간 모니터링 |

리눅스 관리는 OS 마다 설정 파일이 다르므로, 1과목.txt 는 SOLARIS · LINUX · AIX · HP-UX 4종 매핑을 함께 제시한다. 자격증에서는 이 매핑 자체가 출제 포인트가 되므로 본문에서도 표로 보존한다.

## 1. 계정 관리

### 1-1. root 이외의 UID 0 금지

리눅스는 사용자를 이름이 아닌 **UID** 로 식별한다(03 §2 참조). 따라서 root 와 동일한 UID 0 을 가진 다른 계정이 존재하면, 그 계정은 이름이 무엇이든 root 권한으로 시스템에 접근할 수 있다. 침해 시 추가 공격 통로가 될 수 있다.

**점검·조치**

```bash
# 1. /etc/passwd 파일에 UID 0 인 계정이 root 외에 존재하는지 확인
awk -F: '$3 == 0 {print $1}' /etc/passwd

# 2. UID 변경 (linux 는 일반 사용자 UID 500 이상)
usermod -u [UID 500 이상] [계정명]
```

OS 별 UID 권장 범위와 명령은 다음과 같다.

| OS | 일반 사용자 UID | UID 변경 명령 |
|---|---|---|
| **Linux** | 500 이상 | `usermod -u 2016 algisa` |
| **Solaris / HP-UX** | 100 이상 | `usermod -u 2016 algisa` |
| **AIX** | 100 이상 | `chuser id=2016 algisa` |

### 1-2. 로그인이 불필요한 계정의 로그인 차단

서비스 운영용 계정(`bin`, `daemon`, `nobody`, `apache` 등) 은 인증된 사용자 로그인이 필요하지 않다. 이런 계정으로 셸 로그인이 가능하면 침해 시 추가 공격 통로가 될 수 있다.

**조치**: `/etc/passwd` 파일에서 해당 계정의 로그인 셸을 `/bin/false` 또는 `/sbin/nologin` 으로 변경한다.

```
algisa:x:500:500::/home/algisa:/sbin/nologin
```

### 1-3. Session Timeout 설정

계정이 접속된 상태로 방치되면 권한이 없는 사용자에게 중요 시스템이 노출되어 악의적인 목적으로 사용될 수 있다. 일정 시간 동안 이벤트가 발생하지 않으면 연결을 종료하도록 **세션 타임아웃** 을 설정한다.

> *원본 표기 보정*: 1과목.txt line 1411 의 "방지하면" 은 "방치하면" 의 OCR 오류. 통합자료 line 998 ~ 999 의 표기가 정확하다.

**셸별 설정 파일**

| 셸 | 설정 파일 | 설정 형식 |
|---|---|---|
| `sh`, `ksh`, `bash` | `/etc/profile` 또는 `/etc/.profile` | `TMOUT=600` 후 `export TMOUT` (초 단위) |
| `csh` | `/etc/csh.login` 또는 `/etc/csh.cshrc` | `set autologout=10` (분 단위) |

**권장값**: 600초 이내(= 10분).

```bash
# /etc/profile
TMOUT=600
export TMOUT
```

### 시험 포인트
- root 외 UID 0 계정은 **존재 자체가 취약점**. `usermod -u` 로 변경 (Linux 500 / 그 외 100 이상).
- 로그인 차단 셸: **`/bin/false`, `/sbin/nologin`**.
- Session Timeout 권장값 = **600초 (= 10분)**, `bash` 는 `TMOUT`, `csh` 는 `autologout`.

## 2. 패스워드 정책

### 2-1. 패스워드 복잡성 설정

사용자 계정 암호를 유추하기 쉽게 설정하면 비인가자의 시스템 접근을 허용하게 된다. 1과목.txt 는 다음 권장 기준을 제시한다.

> 영문 대문자 / 소문자 / 숫자 / 특수문자 4종을 조합하되,
> - **2종류 조합** 시 최소 **10자리** 이상
> - **3종류 이상 조합** 시 최소 **8자리** 이상
>
> 주기적으로 변경하고 시스템마다 다른 패스워드를 사용한다.

> *원본 표기 보정*: 1과목.txt line 1316 ~ 1317 의 "특수무자" 는 "특수문자" 의 OCR 오류로, 문맥상 명백하다.

**Linux 의 복잡성 설정** — `pam_pwquality` 모듈[^2] 을 사용하며, 설정 파일은 `/etc/security/pwquality.conf` 이다.

| 옵션 | 의미 | 통합자료 권장값 |
|---|---|---|
| `minlen` | 패스워드 최소 길이 | `8` |
| `lcredit` | 소문자 최소 개수(음수 = 최소 N개 강제) | `-1` (소문자 최소 1개) |
| `ucredit` | 대문자 최소 개수 | `-1` (대문자 최소 1개) |
| `dcredit` | 숫자 최소 개수 | `-1` (숫자 최소 1개) |
| `ocredit` | 특수문자 최소 개수 | `-1` (특수문자 최소 1개) |

### 2-2. 패스워드 최소 길이 설정

무작위 공격이나 추측 공격에 대응하기 위해 패스워드 최소 길이를 강제한다. **일반 시스템은 8자 이상**, **국가기관은 9자 이상**으로 설정한다.

OS 별 설정 파일과 옵션이 다르다 — 자격증에 자주 출제되는 매핑이다.

| OS | 설정 파일 | 옵션 |
|---|---|---|
| **Linux** | `/etc/login.defs` | `PASS_MIN_LEN 8` |
| **Solaris** | `/etc/default/passwd` | `PASSLENGTH=8` |
| **AIX** | `/etc/security/user` | `minlen=8` |
| **HP-UX** | `/etc/default/security` | `MIN_PASSWORD_LENGTH=8` |

### 2-3. 패스워드 최대 사용기간 설정

패스워드 최대 사용기간을 설정하지 않으면 일정 기간 경과 후에도 유출된 패스워드로 접속할 수 있다. 주기적으로 변경하도록 강제한다. **권장값: 90일(= 12주)**.

| OS | 설정 파일 | 옵션 (예시값) |
|---|---|---|
| **Linux** | `/etc/login.defs` | `PASS_MAX_DAYS 90` (일 단위) |
| **Solaris** | `/etc/default/passwd` | `MAXWEEKS=12` (주 단위) |
| **AIX** | `/etc/security/user` | `maxage=12` (주 단위) |
| **HP-UX** | `/etc/default/security` | `PASSWORD_MAXDAYS=90` (일 단위) |

### 2-4. 패스워드 최소 사용기간 설정

패스워드 최소 사용기간을 설정하지 않으면 사용자가 익숙한 패스워드로 즉시 되돌릴 수 있다. 패스워드를 재사용하면 정기적인 변경이 무의미해지므로 최소 사용기간을 둔다.

> *원본 표기 보정*: 1과목.txt line 1369 의 "안ㅇ흐면" 은 "않으면" 의 OCR 오류로, 문맥상 명백하다.

| OS | 설정 파일 | 옵션 (예시값) |
|---|---|---|
| **Linux** | `/etc/login.defs` | `PASS_MIN_DAYS 1` (또는 `90`) |
| **Solaris** | `/etc/default/passwd` | `MINWEEKS=12` |
| **AIX** | `/etc/security/user` | `minage=12` |
| **HP-UX** | `/etc/default/security` | `PASSWORD_MINDAYS=90` |

> 통합자료(line 984) 는 `PASS_MIN_DAYS 1`, 1과목.txt(line 1381) 는 `PASS_MIN_DAYS 90` 으로 표기. 권장값은 정책에 따라 다르며, 자격증 시험에서는 **"최소 1일 이상"** 의 의미만 정확히 이해하면 된다.

### 2-5. 패스워드 파일 보호

패스워드 정보를 평문으로 저장하면 정보 유출 피해가 발생할 수 있으므로 암호화하여 보호해야 한다. 03 §2 에서 본 대로, 리눅스는 `/etc/shadow` 파일에 암호화된 패스워드를 저장하고, `/etc/passwd` 의 두 번째 필드는 `x` 로 표시한다.

**점검 절차**

```bash
# 1. /etc/passwd 의 패스워드 필드가 x 표시인지 확인
cat /etc/passwd

# 2. shadow 패스워드 정책으로 변경 (일반 → shadow)
pwconv
```

OS 별 저장 위치는 다음과 같다.

| OS | 패스워드 저장 위치 |
|---|---|
| **Linux / Solaris** | `/etc/shadow` (`/etc/passwd` 두 번째 필드 = `x`) |
| **AIX** | `/etc/security/passwd` (기본 암호화) |
| **HP-UX** | `/tcb/files/auth/[계정 이니셜]/[계정명]` (trustmode 전환 시) |

### 시험 포인트
- 복잡성 권장: **2종 10자 / 3종 이상 8자 / 국가기관 9자**.
- 길이 옵션: Linux `PASS_MIN_LEN`, Solaris `PASSLENGTH`, AIX `minlen`, HP-UX `MIN_PASSWORD_LENGTH`.
- 최대 사용기간 권장: **90일(= 12주)**, 옵션은 `PASS_MAX_DAYS` / `MAXWEEKS` / `maxage` / `PASSWORD_MAXDAYS`.
- 파일 보호: `/etc/passwd` 두 번째 필드 = `x`, 변환 명령은 `pwconv`.

## 3. 파일 및 디렉터리 관리

### 3-1. root 홈·PATH 환경변수 설정

root 계정의 PATH 환경변수에 현재 디렉터리(`.`) 가 앞쪽이나 중간에 포함되어 있으면, 관리자가 의도하지 않은 현재 디렉터리에 위치한 명령어가 실행될 수 있다. 공격자가 현재 디렉터리에 동일 이름의 악성 실행파일(예: `ls`) 을 심어두면, root 가 그 디렉터리에서 명령을 입력하는 순간 악성코드가 root 권한으로 실행된다.

> *원본 표기 보정*: 1과목.txt line 1429 의 "PAH 마지막으로" 는 "PATH 마지막으로" 의 OCR 오류이다.

**점검·조치**

```bash
# 1. PATH 경로 확인
echo $PATH

# 2. /etc/profile 또는 $HOME/.bash_profile 에서 $PATH 변수 설정
#    "." 을 제거하거나 PATH 의 마지막으로 이동
```

**셸별 환경설정 파일** — 셸마다 참조 파일이 다르므로 점검 시 주의한다.

| 셸 | 환경설정 파일 |
|---|---|
| `bin/sh` | `/etc/profile`, `$HOME/.profile` |
| `bin/bash` | `/etc/profile`, `$HOME/.bash_profile` |
| `bin/ksh` | `/etc/profile`, `$HOME/.profile`, `$HOME/.kshrc` |
| `bin/csh` | `/etc/.login`, `$HOME/.cshrc`, `$HOME/.login` |

> *원본 표기 보정*: 1과목.txt line 1438 ~ 1439 의 `$HOME/,login` 은 `$HOME/.login` 의 OCR 오류이다.

### 3-2. 파일 및 디렉터리 소유자/소유그룹 설정

소유자 또는 소유그룹이 존재하지 않는 파일·디렉터리는 다음 중 하나의 상황을 의미한다.

- 사용자를 삭제했지만 그가 만든 파일이 남아 있는 상태(UID/GID 만 표시됨)
- 관리 소홀로 생긴 파일

사용하지 않는 디렉터리·파일이면 시스템 자원 낭비가 되고, 중요 파일이면 관리되지 않는 문제가 발생한다.

**점검·조치**

```bash
# 1. 소유자/소유그룹이 없는 파일·디렉터리 검색
find / \( -nouser -o -nogroup \) -exec ls -al {} \;

# 2. 소유자 변경
chown [계정명] [파일명]

# 3. 소유그룹 변경
chgrp [그룹명] [파일명]
```

### 3-3. World Writable 파일 점검

모든 사용자(`other`) 에게 쓰기 권한(`w`) 이 부여된 파일은 일반 사용자가 실수로 또는 악의적으로 주요 정보를 변경할 수 있다. 시스템 장애나 추가 공격에 활용될 수 있는 문제점이 있다.

**점검·조치**

```bash
# 1. other 에 쓰기 권한이 부여된 파일 검색 (-2 = -0002 와 동일)
find / -perm -2 -exec ls -al {} \;

# 2. other 의 쓰기 권한 제거
chmod o-w [파일명]
```

### 3-4. 주요 파일 소유자 및 권한 설정

자격증에서 가장 자주 출제되는 항목 중 하나다. **권한 숫자를 정확히 외워야 한다.**

| 파일 | 역할 | 권장 소유자 | 권장 권한 |
|---|---|---|---|
| `/etc/passwd` | 사용자 정보 | root | **644 이하** |
| `/etc/shadow` | 암호화된 패스워드 | root | **400 이하** |
| `/etc/hosts` | IP ↔ 호스트명 매핑 | root | **600 이하** (금융위 기준 644) |
| `/etc/(x)inetd.conf` | (x)inetd 데몬 설정 | root | **600 이하** |
| `/etc/syslog.conf` | syslogd 데몬 설정 | root | **644 이하** |
| `/etc/services` | 서비스 ↔ 포트/프로토콜 | root | **644** |

각 파일의 권한 근거는 다음과 같다.

- **`/etc/shadow` (400)**: 암호화된 패스워드가 노출되면 무차별 공격에 의한 정보 유출 피해가 발생할 수 있으므로 root 만 읽기 허용.
- **`/etc/(x)inetd.conf` (600)**: 외부의 불법적 접근으로 악의적 프로그램을 등록하면 root 권한으로 실행될 수 있다.
- **`/etc/syslog.conf` (644)**: 변경되면 로그가 정상적으로 기록되지 않아 침입 흔적·시스템 오류를 분석할 수 없다.

### 3-5. find 명령 활용

`find` 는 시스템 점검에서 가장 광범위하게 활용되는 명령이다. 자격증에서는 옵션 매핑이 자주 출제된다.

| 검색 기준 | 명령 예시 | 의미 |
|---|---|---|
| **권한** | `find / -perm -4000` | SUID 권한이 설정된 파일 |
| **속성** | `find / -type f -mtime -5` | 정규 파일 중 마지막 수정일이 5일 미만 |
| **이름** | `find / -name "passwd"` | 파일명이 `passwd` |
| **크기** | `find / -type f -size -5M` | 정규 파일 중 크기가 5MB 미만 |
| **소유자/그룹 부재** | `find / \( -nouser -o -nogroup \) -exec ls -al {} \;` | §3-2 참조 |
| **other 쓰기 가능** | `find / -perm -2 -exec ls -al {} \;` | §3-3 참조 |

### 시험 포인트
- PATH 에 **`.` 이 앞·중간에 있으면 안 됨**. 제거 또는 마지막으로 이동.
- 주요 파일 권장 권한: **`/etc/passwd` 644 / `/etc/shadow` 400 / `/etc/hosts` 600 / `/etc/syslog.conf` 644**.
- World writable 검색: **`find / -perm -2`**. SUID 검색: **`find / -perm -4000`**.

## 4. 서비스 관리

### 배경

03 §8 에서 본 대로, 리눅스의 네트워크 서비스는 **Stand-Alone** 또는 **(x)inetd** 의 두 가지 방식으로 기동된다. (x)inetd 방식 서비스는 `/etc/(x)inetd.conf` 의 한 줄로 활성/비활성이 결정되므로, 점검 시 이 파일에서 **불필요하거나 취약한 서비스를 주석 처리**하는 것이 핵심이다.

이 영역에서 출제되는 4 그룹은 다음과 같다.

| 그룹 | 위험 | 대상 서비스 |
|---|---|---|
| **Simple TCP 서비스** | DoS 공격에 취약 | `echo`, `discard`, `daytime`, `chargen` (포트 7/9/13/19) |
| **R 계열 서비스** | 인증 없이 관리자 원격 접속 가능 | `rlogin`, `rsh`, `rexec` |
| **RPC 서비스** | 서버 응용 프로그램에 접근하여 작업 호출 가능 | `rpc.cmsd`, `rusersd` |
| **기타 평문/취약 서비스** | 정보 유출, 인증 부재, 평문 통신 | `finger`, `tftp`, `talk` |

### 4-1. Simple TCP 서비스 비활성화

`echo` · `discard` · `daytime` · `chargen` 은 **DoS 공격에 취약한 Simple TCP 서비스** 로, 운영상 필요하지 않으면 (x)inetd 설정에서 주석 처리한다. 03 §8 의 잘 알려진 포트 매핑 기준으로 각각 7/9/13/19 번 포트에 대응한다.

### 4-2. R 계열 서비스 비활성화

`rlogin` · `rsh` · `rexec` 는 **인증 없이 관리자의 원격 접속을 가능하게 하는 r 계열 서비스** 로, (x)inetd 설정에서 주석 처리한다. 인증·암호화가 필요한 원격 접속에는 SSH(22/tcp) 를 사용한다.

### 4-3. RPC 서비스 비활성화

`rpc.cmsd` · `rusersd` 등 RPC(Remote Procedure Call) 서비스는 외부에서 서버 응용 프로그램에 접근하여 임의 작업을 호출할 수 있다. 사용하지 않는 RPC 서비스는 비활성화한다.

### 4-4. 기타 평문/취약 서비스 비활성화

| 서비스 | 위험 |
|---|---|
| **`finger`** | 원격 시스템의 사용자 정보(로그인명, 마지막 접속 등)를 조회 — 정보 유출 |
| **`tftp`** | 사용자 인증 없는 평문 통신 FTP — 부팅 시점 외에는 비활성화 |
| **`talk`** | 암호화·보안 기능이 부족한 실시간 채팅 |

### 점검 명령

```bash
# (x)inetd 기반: 설정 파일에서 해당 서비스 줄을 주석 처리 (#)
vi /etc/xinetd.conf
vi /etc/xinetd.d/[서비스명]

# 활성 포트 확인
netstat -anp | grep LISTEN

# 또는 (서비스별 데몬이 (x)inetd 가 아닌 Stand-Alone 인 경우)
systemctl disable [서비스명]
```

### 시험 포인트
- **Simple TCP = `echo`/`discard`/`daytime`/`chargen` (포트 7/9/13/19)** — DoS 위험.
- **R 계열 = `rlogin`/`rsh`/`rexec`** — 인증 없는 원격 접속.
- **RPC = `rpc.cmsd`/`rusersd`** — 응용 프로그램 작업 호출.
- **`finger` (정보 유출), `tftp` (평문), `talk` (암호화 부재)** — 항목 매칭 출제.

## 5. 로그 관리

### 5-1. 로그 인프라 — 03 §10 으로 위임

`syslog` / `rsyslog` 의 facility · priority · action 형식, `logrotate` 옵션, 주요 로그 파일(utmp/wtmp/btmp/lastlog/secure/messages/dmesg/boot.log/cron/maillog/xferlog) 의 매핑은 [03. 리눅스 기본 §10 로깅 시스템](./03.linux-basic.md#10-로깅-시스템) 에 정리되어 있다.

이 단원에서는 운영 관점에서의 두 가지 추가 작업만 다룬다 — **원격 로그 서버 운영**, **실시간 모니터링**.

### 5-2. 원격 로그 서버 운영

침해 시 로컬 로그 파일이 변조·삭제될 위험에 대비해, 로그를 **별도의 원격 서버로 송신**하여 보존한다. `rsyslog` 는 514/udp 로 원격 송신 기능을 제공한다.

**받는 서버 (로그 수집 서버) 설정**

```bash
# 1. /etc/rsyslog.conf 에 UDP 수신 포트 활성화
$UDPServerRun 514

# 2. rsyslogd 데몬 구동 확인
ps -ef | grep rsyslogd

# 3. 514/udp 포트 오픈 확인
netstat -enup | grep rsyslogd

# 4. 방화벽에서 514/udp 허용
iptables -A INPUT -p udp --dport 514 -j ACCEPT
```

**보내는 서버 설정**

```bash
# /etc/rsyslog.conf 에서 모든 facility · priority 의 로그를
# 원격 수집 서버로 송신
*.* @[받는 서버 주소]
```

> 03 §10-1 에서 언급한 대로, 평문 syslog 는 기밀성·무결성·가용성을 고려하지 않고 개발되었다. RFC 3195 는 **TCP·BEEP** 사용을 권장한다.

### 5-3. 실시간 모니터링

```bash
# 텍스트 로그 실시간 확인 (binary 로그는 사용 불가)
tail -f /var/log/[로그파일]
```

binary 로그(utmp, wtmp, lastlog, btmp, pacct) 는 03 §10-3 의 전용 명령(`w`, `last`, `lastlog`, `lastb`, `lastcomm`) 으로 확인한다.

**로그 저장 디렉터리**

| OS | 디렉터리 |
|---|---|
| **Linux** | `/var/log` |
| **Unix** | `/var/adm` |

### 시험 포인트
- 원격 로그 수신: `$UDPServerRun 514` + `iptables ... --dport 514 -j ACCEPT`.
- 원격 로그 송신: `*.* @[수신 서버 주소]` 형식.
- 실시간 모니터링: **`tail -f`**(텍스트 한정), binary 는 전용 명령.

## 단원 요약

1. **계정 관리** — UID 0 유일성(Linux 500 이상). 로그인 차단 셸 = `/bin/false`/`/sbin/nologin`. Session Timeout = `TMOUT=600`(bash) / `autologout=10`(csh).
2. **패스워드 정책** — 복잡성: 2종 10자 / 3종 이상 8자 / 국가기관 9자. 최대 사용기간 90일(=12주). OS 별 옵션 매핑 외우기.
3. **파일·디렉터리** — PATH 의 `.` 제거. 주요 파일 권한: passwd 644 / shadow 400 / hosts 600 / syslog.conf 644. `find` 옵션: SUID `-perm -4000`, world writable `-perm -2`.
4. **서비스 관리** — Simple TCP(7/9/13/19) DoS, R 계열·RPC 인증 부재, finger/tftp/talk 평문/정보 유출.
5. **로그 관리** — 03 §10 의 인프라 위에 원격 로그 서버(514/udp)와 `tail -f` 실시간 모니터링이 운영 표준.
