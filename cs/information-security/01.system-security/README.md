1. **01 → 02** (Windows 계열): 인증·권한 구조부터 운영·로그까지 연속 학습

## 4. 작성 원칙 (1과목 적용)

[외부자료 검증 체크리스트](../docs/외부자료-검증체크리스트.md) §3 을 따른다. 1과목 추가 사항:

1. **누락 금지**: 위 매핑표의 통합자료·1과목.txt 라인 범위를 반드시 빠짐없이 흡수한다.
2. **추론 금지**: 양쪽 원본에 없는 내용은 외부 자료 출처를 각주로 명시한 경우에만 추가한다.
3. **충돌 시 우선순위**: 출제기준 → 통합자료(체계적 표) → 1과목.txt(보충 사례)
4. **명령·경로는 검증 가능한 형식으로**: 예 `/etc/shadow`, `find -perm 4000` 등은 원본의 표기 그대로 사용하고, 명령 결과 출력은 임의 가공 금지.
5. **표/명령어/예시는 가능한 보존**: 통합자료의 표 구조와 1과목.txt 의 명령 예시는 학습 가치가 높으므로 유지한다.
6. **서술 원칙**: 각 H2/H3 단위로 "배경 → 정의 → 본문 → 예시/표 → 시험 포인트" 흐름을 따른다. OCR 표기 오류는 문맥상 명백할 때만 보정하며, 수치·플래그·식별자는 보정하지 않는다.
7. **보충 출처**: 일반 CS 상식 외의 보안 개념·법령·표준은 각주로 1차 출처(NIST·KISA·RFC·법령 등)를 명시한다. 위키피디아·민간 학원 자료는 1차 출처로 사용하지 않는다.
