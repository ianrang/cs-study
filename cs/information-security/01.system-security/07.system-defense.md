# 07. 시스템 보안 — 방어·분석 도구·솔루션

> **이 문서가 다루는 것**: 출제기준 *3.시스템 보안위협 및 공격에 대한 예방과 대응* 의 **시스템보안 대응기술 / 시스템 분석 도구 / 시스템 보안 솔루션**. 보안 운영체제·보안커널·TCB·참조 모니터·TPM·SSH 의 정의와 구성, Nmap·Kali Linux 의 활용, HIDS·HIPS·EDR·FIM 등 시스템 보안 솔루션. 자격증 1과목 1~3 영역의 *대응* 측면을 마무리하는 단원이다. Nmap·Kali·솔루션 영역은 [외부자료 검증 체크리스트](../docs/외부자료-검증체크리스트.md) §2 의 1차 출처(Nmap 공식 문서, KISA SW 보안 가이드, NIST SP 800-94)로 보강한다.
>
> **출제기준 매핑**: 3.예방과 대응 — 1) 시스템보안 대응기술 / 2) 시스템 분석 도구(Nmap, Kali Linux 등) / 3) 시스템 보안 솔루션.
>
> **원본 매핑**: 통합자료 line 1312 ~ 1394 (시스템 보안). 1과목.txt 의 해당 영역(1496 ~ 1580 윈도우 서버 취약점)은 02 단원에서 이미 흡수. Nmap·Kali·시스템 보안 솔루션은 외부 보강.

## 0. 들어가며 — 시스템 방어의 4 계층

01~06 단원에서 본 공격기법(BoF·레이스 컨디션·포맷 스트링·자원 고갈·APT 등) 은 *OS 의 한 약점* 을 노린다. 따라서 방어도 **OS 자체의 신뢰성** 부터 **운영 중인 시스템의 모니터링** 까지 다층으로 구성된다. 자격증 1과목의 시스템 방어는 다음 4계층으로 구분된다.

| 계층 | 다루는 영역 | 본 단원 § |
|---|---|---|
| **OS 의 신뢰 기반** | 보안 OS, 보안커널, TCB, 참조 모니터 | §1 ~ §3 |
| **하드웨어 신뢰 기반** | TPM (Trusted Platform Module) | §4 |
| **암호화 채널** | SSH | §5 |
| **분석·솔루션** | Nmap·Kali Linux, HIDS·HIPS·EDR·FIM | §6, §7 |

핵심 개념의 연결 구조는 다음과 같다.

```
[ 신뢰 컴퓨팅 기반 (TCB) ]
        ├── 보안커널 ── 참조 모니터 ── SKDB
        ├── 신뢰된 경로 (TP)
        └── 보안경계 (SP)

[ 하드웨어 ]
        └── TPM ─ 인증 부트 / 인증 / 암호화 서비스

[ 운영 ]
        └── SSH (22/tcp) ─ Telnet/FTP 대체
```

이 단원이 정리되면 1과목 7개 단원이 모두 마무리되며, 학습자는 **공격기법(06) → 방어 도구·솔루션(07)** 의 짝으로 1과목 전체를 회수할 수 있다.

## 1. 보안 운영체제 (Secure OS)

### 1-1. 정의

> *원본 표기 보정*: 통합자료 line 1314 ~ 1315 는 OCR 과정에서 줄바꿈이 깨져 "설명통합" 과 "시스템을 보호하기 위해 기존의 운영체제 내에 보안기능을" 이 분리 표기됨. 정상 표기는 다음과 같다.

**시스템을 보호하기 위해 기존의 운영체제 내에 보안기능을 통합**한 운영체제로, **보안커널을 추가로 이식**한 운영체제다.

### 1-2. 보호 대상

보안 OS 가 보호하는 자원은 다음 5가지다.

| 보호 대상 | 의미 |
|---|---|
| **메모리** | 실행 중 메모리의 무단 접근·변조 차단 |
| **보조기억장치 상의 파일·데이터 집합** | 디스크의 파일 무결성 보호 |
| **메모리상에서 실행 중인 프로그램** | 프로세스 격리 (03 §6 의 RUID·EUID 위) |
| **파일들의 디렉터리** | 디렉터리 권한·소유자 보호 (03 §3 의 inode 위) |
| **하드웨어 장치** | 직접 하드웨어 접근 제한 |

### 1-3. 보호 방법 4종

| 방법 | 의미 |
|---|---|
| **물리적 분리** | 사용자별로 **별도의 장비만 사용하도록 제한**하는 방법 |
| **시간적 분리** | 프로세스가 **동일 시간에 하나씩만 실행**되도록 하는 방법 |
| **논리적 분리** | 각 프로세스에 **논리적인 구역을 지정**하는 방법 |
| **암호적 분리** | 내부에서 사용되는 정보를 외부에서 알 수 없도록 **암호화** 하는 방법 |

### 1-4. 파일 보호기법 3종

| 기법 | 의미 |
|---|---|
| **이름 명명** | 파일의 이름을 추측하기 어렵게 변경 |
| **패스워드 설정** | 패스워드를 알아야만 파일을 이용할 수 있게 하는 방법 |
| **암호화 설정** | 파일 **내용 자체를 암호화** 하여 인가된 사용자만 그 내용을 파악 |

### 시험 포인트
- 보안 OS = **기존 OS + 보안기능 통합**, 핵심은 **보안커널 이식**.
- 보호 방법 4종: **물리적 / 시간적 / 논리적 / 암호적 분리**.
- 파일 보호 3종: **이름 명명 / 패스워드 / 암호화** — 정의 매칭 출제.

## 2. 보안커널과 신뢰 컴퓨팅 기반 (TCB)

### 2-1. 보안커널 (Security Kernel)

**보안커널**은 **TCB 내에 있는 하드웨어·소프트웨어·펌웨어로 구성** 되며, **참조 모니터 개념을 구현하고 집행** 한다. 핵심 특성은 다음과 같다.

- **주체와 객체 사이의 모든 접근과 기능을 중재**
- TCB 의 핵심이고, 신뢰할 수 있는 컴퓨터 시스템을 구축하기 위해 가장 보편적으로 사용

### 2-2. 신뢰 컴퓨팅 기반 (TCB, Trusted Computing Base)

**TCB** 는 운영체제·하드웨어·펌웨어·소프트웨어 등이 포함된 **컴퓨터 시스템 내의 총제적 보호 메커니즘** 이다. TCB 안에는 두 가지 핵심 개념이 포함된다.

| 개념 | 의미 |
|---|---|
| **신뢰된 경로** (TP, Trusted Path) | 사용자·프로그램과 커널과의 **통신채널** |
| **보안경계** (SP, Security Perimeter) | 시스템의 구성요소 중 **신뢰/비신뢰 구성요소의 경계** 로, 가상의 경계선 |

### 시험 포인트
- 보안커널 = **TCB 의 핵심 + 참조 모니터 구현체**.
- TCB = OS·HW·FW·SW 의 **총제적 보호 메커니즘**.
- TP = 통신채널 / SP = 신뢰·비신뢰 가상 경계선 — 약어 매칭 빈출.

## 3. 참조 모니터 (Reference Monitor)

### 3-1. 정의

**주체의 객체에 대한 모든 접근통제를 담당하는 추상머신** 이다. 보안커널이 이 개념을 구현한다.

### 3-2. 3대 요건

참조 모니터가 보안 모델로 작동하려면 다음 3대 요건을 만족해야 한다.

| 요건 | 의미 |
|---|---|
| **부정조작이 없어야 함** (Tamper-proof) | 외부에서 변조할 수 없어야 한다 |
| **항상 무시되지 않고 호출되어야 함** (Always-invoked) | 모든 접근 시 반드시 거쳐야 한다 |
| **분석·테스트를 통해 확인할 수 있어야 함** (Verifiable) | 모든 동작을 분석·테스트로 검증 가능해야 한다 |

### 3-3. SKDB 참조

참조 모니터는 **SKDB(Security Kernel Database)** 를 참조하여 객체에 대한 **접근 허가 여부**를 결정한다.

### 시험 포인트
- 참조 모니터 = **모든 접근통제를 담당하는 추상머신**.
- 3대 요건: **Tamper-proof / Always-invoked / Verifiable**.
- 접근 결정의 근거 = **SKDB**.

## 4. 신뢰 플랫폼 모듈 (TPM)

### 4-1. 정의

> *원본 표기 보정*: 통합자료 line 1360 ~ 1361 의 OCR 줄바꿈 결함("설명되는 하드웨어 모듈" / "신뢰 컴퓨팅을 위한 하드웨어/소프트웨어 방법에서 핵심이") 을 정상 표기로 보정한다.

**신뢰 컴퓨팅을 위한 하드웨어/소프트웨어 방법에서 핵심이 되는 하드웨어 모듈** 이다. TPM 은 메인보드에 부착되거나 CPU 에 통합된 보안 칩이다.

### 4-2. TPM 의 3대 서비스

| 서비스 | 설명 |
|---|---|
| **인증된 부트 서비스** | 전체 운영체제를 단계적으로 부팅하고, 운영체제가 적재될 때 운영체제의 각 부분이 **사용을 위해 승인된 버전임을 보장** |
| **인증 서비스** | TPM 에 의해 설정이 완성되고 로그인 되면, **TPM 은 다른 부분의 설정을 인증** |
| **암호화 서비스** | **특정 기계가 특정 설정으로 되어 있을 때만** 그 기계에서 데이터의 복호화를 수행하는 방식으로 데이터의 암호화를 지원 |

### 시험 포인트
- TPM = **신뢰 컴퓨팅의 핵심 하드웨어 모듈**.
- 3대 서비스: **인증된 부트 / 인증 / 암호화** — 각 서비스의 정의 매칭.
- 05 §1-3 의 iOS Secure Boot Chain 도 동일 원리(하드웨어 키 → 단계적 검증) 의 한 구현이다.

## 5. 보안 쉘 (SSH)

### 5-1. 정의

**암호 통신을 이용하여 네트워크상의 다른 컴퓨터에 접속**하기 위한 응용 프로그램 또는 프로토콜이다. 원격으로 명령을 실행하거나 파일을 조작한다.

- **포트**: 22/tcp (03 §8 의 잘 알려진 포트)

### 5-2. 보안 쉘 서비스

| 서비스 | 설명 |
|---|---|
| **원격 터미널** | 암호화된 원격 터미널 서비스 제공 — **Telnet 평문 서비스 대체** |
| **파일 송수신** | 암호화된 파일 송수신 서비스 제공 — **FTP 평문 서비스 대체** |

### 5-3. 04 단원과의 연계

04 §4-2 에서 본 R 계열 서비스(`rlogin`/`rsh`/`rexec`) 의 비활성화는 **SSH 로의 대체**를 전제로 한다. R 계열은 인증 없이 명령을 실행하지만 SSH 는 22/tcp 의 단일 포트에서 **암호화된 원격 터미널·파일 송수신** 을 모두 제공한다.

### 시험 포인트
- SSH = **22/tcp**, 평문 프로토콜의 안전한 대체.
- 대체 관계: **Telnet → SSH 원격 터미널**, **FTP → SSH 파일 송수신(SFTP/SCP)**.

## 6. 시스템 분석 도구 (외부 보강)

### 6-1. Nmap (Network Mapper)

**Nmap** 은 네트워크 탐색 및 보안 감사용 오픈소스 도구로, 시스템 보안 점검에서 가장 널리 사용된다[^1].

#### 주요 스캔 옵션

| 옵션 | 의미 |
|---|---|
| **`-sS`** | TCP SYN 스캔 (Half-open scan) — 기본 스캔, 빠르고 흔적이 적음 |
| **`-sT`** | TCP Connect 스캔 — 3-way handshake 완료, 흔적이 명확 |
| **`-sU`** | UDP 스캔 |
| **`-sV`** | 서비스 버전 탐지 |
| **`-O`** | 운영체제 탐지 (OS Fingerprinting) |
| **`-sn`** | Ping 스캔 (포트 스캔 없이 호스트 생존 여부) |
| **`-p [포트]`** | 특정 포트 스캔 (예: `-p 22,80,443`) |
| **`-A`** | 공격적 스캔 (`-sV` + `-O` + 스크립트 + traceroute) |

#### 사용 예

```bash
# 기본 SYN 스캔
nmap -sS 192.168.1.1

# 1~1024 포트 + 서비스 버전 + OS 탐지
nmap -sS -sV -O -p 1-1024 192.168.1.1

# 네트워크 대역 호스트 생존 확인
nmap -sn 192.168.1.0/24
```

### 6-2. Kali Linux

**Kali Linux** 는 Offensive Security 가 유지보수하는 **침투 테스트(Pentest)·디지털 포렌식 전용 Debian 기반 배포판** 으로, 600 종 이상의 보안 도구가 사전 설치되어 있다[^2].

#### Kali 에 포함된 대표 도구 카테고리

| 카테고리 | 대표 도구 |
|---|---|
| **정보 수집** | Nmap, Maltego, theHarvester |
| **취약점 분석** | OpenVAS, Nikto |
| **무선 공격** | Aircrack-ng, Reaver |
| **웹 애플리케이션 분석** | Burp Suite, OWASP ZAP, sqlmap |
| **패스워드 공격** | John the Ripper, Hydra (06 §1 의 패스워드 크래킹과 연결) |
| **익스플로잇 프레임워크** | Metasploit |
| **포렌식** | Autopsy, Foremost |

### 시험 포인트
- Nmap 핵심 옵션: **`-sS` (SYN 스캔), `-sT` (Connect 스캔), `-sU` (UDP), `-sV` (버전), `-O` (OS), `-A` (공격적)**.
- Kali Linux = **Debian 기반 침투 테스트 배포판**, **Offensive Security** 가 유지보수.
- Kali 의 도구는 06 §1 의 패스워드 크래킹·취약점 점검에서 등장한 도구(John the Ripper 등) 를 포함.

## 7. 시스템 보안 솔루션 (외부 보강)

### 7-1. 솔루션 분류

자격증 1과목의 시스템 보안 솔루션은 **탐지 / 차단 / 응답 / 무결성 검증** 4축으로 분류된다[^3].

| 솔루션 | 영문 | 위치 | 핵심 기능 |
|---|---|---|---|
| **HIDS** | Host-based IDS | 호스트 | 호스트 내부의 이상 행위 **탐지** (시그니처·이상행위 기반) |
| **HIPS** | Host-based IPS | 호스트 | 탐지 + **차단** 까지 능동 수행 |
| **NIDS / NIPS** | Network-based IDS/IPS | 네트워크 경계 | 네트워크 트래픽 기반 탐지·차단 (자세히는 2과목 네트워크 보안) |
| **EDR** | Endpoint Detection and Response | 엔드포인트 | **탐지 + 조사 + 응답** 의 통합 솔루션 (행위 분석 중심) |
| **FIM** | File Integrity Monitoring | 호스트 | **파일 무결성 변경 감지** (Tripwire, AIDE 등) |
| **DLP** | Data Loss Prevention | 호스트·네트워크 | 민감 데이터 외부 유출 방지 |
| **EPP** | Endpoint Protection Platform | 엔드포인트 | **백신 + 방화벽 + HIPS** 등 예방 기능 통합 (예방 중심) |

### 7-2. HIDS vs HIPS

| 항목 | HIDS | HIPS |
|---|---|---|
| **동작** | 탐지·로깅 (수동) | 탐지 + **차단** (능동) |
| **위험** | 탐지만 하므로 차단까지 시간 지연 | 오탐(false positive) 시 정상 동작 차단 가능 |
| **적용** | 모든 호스트에 광범위 | 중요 자산 호스트에 집중 |

06 §5 의 백도어 대응에서 본 **포트 활동 탐지** 가 HIDS 의 대표 기능 중 하나다.

### 7-3. EDR (Endpoint Detection and Response)

**EDR** 은 엔드포인트(서버·PC·모바일) 에서 발생하는 보안 이벤트를 **수집·분석·조사·응답** 하는 통합 솔루션이다[^3].

| 단계 | 기능 |
|---|---|
| **Detection** | 행위 분석·시그니처로 위협 탐지 |
| **Investigation** | 침해 경로·확산 범위 조사 |
| **Response** | 격리·차단·복구 자동화 |

EPP(예방 중심) 와 차이는 **사후 침해 대응에 강점** 이라는 점이며, APT 같은 시나리오 위협(06 §7) 에 대해 EPP 의 예방 기능 위에 EDR 의 *행위 기반 탐지·조사·응답* 이 보완 단계로 배치된다[^3].

### 7-4. FIM (File Integrity Monitoring)

**FIM** 은 **파일·디렉터리·레지스트리의 무결성 변화를 감지** 하는 솔루션이다[^3]. 대표 도구는 다음과 같다.

| 도구 | 플랫폼 | 특징 |
|---|---|---|
| **Tripwire** | Unix/Linux | FIM 의 시조, 해시 기반 무결성 검증 |
| **AIDE** (Advanced Intrusion Detection Environment) | Linux | Tripwire 의 오픈소스 대안 |
| **OSSEC** | 다중 플랫폼 | HIDS + FIM 통합 |

### 시험 포인트
- HIDS = **탐지** / HIPS = **차단**. 약어 차이.
- EDR = **Detect + Investigate + Respond** 통합, APT 대응의 핵심.
- FIM 대표 도구: **Tripwire / AIDE / OSSEC**.
- EPP = 예방 / EDR = 사후 대응 — 보완 관계.

## 단원 요약

1. **보안 OS** — 기존 OS + 보안커널 이식. 보호 대상 5종, 보호 방법 4종(물리·시간·논리·암호 분리), 파일 보호 3종.
2. **보안커널·TCB** — 보안커널 = TCB 의 핵심 + 참조 모니터 구현. TCB 안에 TP(통신채널), SP(가상 경계선).
3. **참조 모니터** — 모든 접근통제를 담당하는 추상머신. 3대 요건 = Tamper-proof / Always-invoked / Verifiable. SKDB 참조.
4. **TPM** — 신뢰 컴퓨팅의 핵심 하드웨어 모듈. 3대 서비스 = 인증된 부트 / 인증 / 암호화.
5. **SSH** — 22/tcp. Telnet/FTP 의 안전한 대체.
6. **Nmap·Kali** — Nmap 옵션 `-sS/-sT/-sU/-sV/-O/-A`. Kali = Debian 기반 침투 테스트 배포판.
7. **시스템 보안 솔루션** — HIDS(탐지)/HIPS(차단)/EDR(통합 응답)/FIM(무결성)/EPP(예방). 6 §5 백도어 대응의 H-IDS 가 여기로 연결.
