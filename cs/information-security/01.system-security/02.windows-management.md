# 02. 윈도우 관리

> **이 문서가 다루는 것**: 앞 단원([01. 윈도우 기본](./01.windows-basic.md))이 윈도우의 *구조*를 설명했다면, 이 단원은 그 구조 위에서 *관리자가 실제로 어떤 정책을 설정해 시스템을 안전하게 운영하는가* 를 다룬다. 계정·패스워드·계정 잠금·공유·서비스·감사 정책 — 이 여섯 영역의 보안 설정 표준이 핵심이다.
>
> **출제기준 매핑**: 1.정보 시스템의 범위 — 시스템 정보(인증정보·시스템·감사 로그) / 3.예방과 대응 — 시스템보안 대응기술(보안설정·인증강화).
>
> **원본 매핑**: 통합자료 line 265 ~ 369, 1과목.txt line 1497 ~ 1580.

## 0. 들어가며 — 왜 "관리" 가 별도 단원인가

윈도우의 보안 수준은 **OS 가 제공하는 기능 자체** 보다 **관리자가 그 기능을 어떻게 설정했는가** 에 더 크게 좌우된다. 예를 들어 NTLMv2 가 안전한 알고리즘이라 해도 관리자가 LM 호환 모드를 켜 두면 무력해지고, Administrator 라는 이름이 그대로 노출되면 무차별 대입의 표적이 된다.

이 단원에서 다루는 보안 설정 항목은 대부분 **윈도우 보안 점검(취약점 진단) 의 표준 항목**이며, KISA 의 *주요정보통신기반시설 기술적·관리적 보호조치 기준*[^1] 과도 정합한다. 시험에서는 권장값(예: "패스워드 최소 길이 8자", "계정 잠금 임계값 5 이하") 이 자주 출제된다.

설정의 진입 경로는 대부분 다음 두 가지다.

- **로컬 보안 정책** (`secpol.msc`): 시작 → 실행 → `secpol.msc`. 계정 정책·로컬 정책·감사 정책 등 보안 설정의 중심.
- **컴퓨터 관리** (`compmgmt.msc`): 사용자·그룹·공유 폴더·서비스 등 객체 관리.

## 1. 계정 관리

### 배경

기본 계정과 그룹은 어떤 윈도우 시스템에나 동일한 이름·SID 로 존재하므로, 공격자에게는 표적의 위치가 미리 알려진 상태다. 따라서 관리의 첫 단계는 **표적이 되기 쉬운 기본 계정의 노출을 줄이고, 권한이 강한 그룹에 들어간 계정 수를 최소화하는 일**이다.

### 1-1. Administrator 계정 이름 변경

기본 관리자 계정명을 그대로 사용하면 악의적인 공격자가 패스워드 무차별 공격에 쉽게 활용할 수 있다. 쉽게 유추하기 어려운 이름으로 변경할 것을 권장한다.

> 단, [01. 윈도우 기본](./01.windows-basic.md) §3 에서 본 대로 RID 500 은 변경되지 않으므로 RID 기반 식별 공격은 여전히 가능하다. 이름 변경은 자동화된 일반 공격을 떨어뜨리는 1차 방어선이다.

**설정 경로**

```
로컬 보안 정책 → 로컬 정책 → 보안 옵션 →
[계정: Administrator 계정 이름 바꾸기]
```

### 1-2. Guest 계정 사용 제한

Guest 계정은 소프트웨어·하드웨어 설치, 설정 변경, 암호 만들기 등을 할 수 없고 외부 사용자가 컴퓨터를 잠시 접근할 수 있는 계정이다. 그러나 불특정 사용자의 시스템 접근을 허용하는 취약한 계정이므로 **사용 안 함을 권장**한다. 불특정 다수의 접근이 필요한 경우에도 게스트 계정보다는 일반 사용자 계정을 별도로 생성해 사용한다.

**설정 경로**

```
컴퓨터 관리 → 로컬 사용자 및 그룹 → 사용자 →
[Guest → 계정 사용 안 함]
```

### 1-3. 불필요한 계정 제거

불필요한 계정, 의심스러운 계정, 장기간 사용하지 않는 계정은 계정 정보 유출의 위험이 있으므로 삭제 또는 계정 잠금을 설정한다.

**설정 경로**

```
컴퓨터 관리 → 로컬 사용자 및 그룹 → 사용자 →
[불필요한 계정 삭제 또는 사용 안 함]
```

### 1-4. 관리자 그룹에 최소한의 사용자만 포함

관리 업무를 수행하는 계정과 일반 업무를 수행하는 계정은 분리해서 사용하는 것이 바람직하며, 관리자 그룹(Administrators) 에는 최소한의 계정만 포함한다.

**설정 경로**

```
컴퓨터 관리 → 로컬 사용자 및 그룹 → 그룹 →
[Administrators → 불필요한 계정 확인 및 제거]
```

### 시험 포인트
- "기본 계정 이름 변경 / Guest 비활성화 / 관리자 그룹 최소화" 는 윈도우 보안 점검의 1단계 항목으로 묶어 외운다.

## 2. 패스워드 정책

### 배경

패스워드 정책은 인증 시스템의 강도를 결정하는 가장 단순하면서 가장 효과적인 통제다. 정책이 약하면 무차별 공격(brute force)·사전 공격(dictionary)·재사용 패스워드 공격이 모두 성립한다. 윈도우는 이를 방어하기 위해 **복잡성·기억·사용기간·길이·암호화 옵션** 다섯 축을 정책으로 제공한다.

### 설정 경로 (공통)

```
로컬 보안 정책 → 계정 정책 → 암호 정책
```

### 2-1. 정책 항목과 권장값

| 정책 항목 | 권장값 | 의미 |
|---|---|---|
| **암호는 복잡성을 만족해야 함** | 사용 | 영문 대/소문자·숫자·특수문자 조합 강제. 2종 조합 시 10자리 이상, 3종 이상 조합 시 8자리 이상 |
| **최근 암호 기억** | 12개 이상 권장 | 직전 N개의 암호를 재사용할 수 없게 함. 암호 변경의 의미를 보존 |
| **암호 최대 사용 기간** | 90일 이하 | 유출된 암호의 유효기간을 제한. 계정 속성의 "암호 사용 기간 제한 없음" 은 해제해야 적용 |
| **암호 최소 사용 기간** | 1일 | 사용자가 즉시 N+1 번 변경해 "최근 암호 기억" 을 우회하는 것을 막음 |
| **암호 최소 길이** | 8자 이상 | 무차별 공격에 걸리는 시간을 늘림 |
| **해독 가능한 암호화 사용** | 사용 안 함 | 일부 호환성 목적 외에는 사용하지 않음. 사용 시 패스워드를 사실상 평문으로 보관하는 효과가 발생 |

### 2-2. 부적절한 패스워드 — 피해야 할 예시

- 길이가 짧은 것
- 사전에 나오는 단어나 단어들의 조합
- 키보드 자판의 연속된 문자(예: `qwerty`, `asdf`)
- 사용자 계정 정보와 유사한 단어(이름·생일·사번 등)

### 시험 포인트
- 권장값(8자/12개/90일/1일) 의 정확한 수치가 그대로 출제된다.
- "최소 사용 기간이 0이면 어떤 우회가 가능한가?" → 사용자가 N+1번 즉시 변경해 이전 암호로 되돌아갈 수 있다.
- "해독 가능한 암호화" 옵션의 위험성 — 평문 저장과 동등한 위험.

## 3. 계정 잠금 정책

### 배경

암호 정책이 *재료* 라면, 계정 잠금 정책은 *시도 횟수의 상한* 이다. 무차별 공격은 본질적으로 "수많은 시도" 를 전제로 하므로, 일정 횟수 이상의 실패에 계정을 잠그면 공격 자체가 성립하기 어려워진다.

### 설정 경로

```
로컬 보안 정책 → 계정 정책 → 계정 잠금 정책
```

### 정책 항목과 권장값

| 정책 항목 | 권장값 | 의미 |
|---|---|---|
| **계정 잠금 기간** | 60분 이상 | 잠긴 계정이 자동 해제되기까지 걸리는 시간 |
| **계정 잠금 임계값** | 5 이하 | 연속 로그인 실패가 이 횟수에 도달하면 계정이 잠김 |
| **다음 시간 후 계정 잠금 수를 원래대로 설정** | 60분 이상 | 실패 카운터를 0으로 되돌리기까지의 경과 시간 |

### 시험 포인트
- 세 값(60분 / 5 이하 / 60분) 모두 권장 수치 그대로 출제 단골.
- 임계값을 0 으로 두면 "계정을 잠그지 않음" 을 의미해 정책이 무력화됨.

## 4. 공유 및 서비스 관리

### 배경

[01. 윈도우 기본 §7](./01.windows-basic.md) 에서 본 기본 공유(C$, D$, ADMIN$, IPC$) 는 OS 설치 시 자동 생성된다. 또한 윈도우는 사용 빈도가 낮은 일부 서비스도 기본으로 활성화하는 경향이 있다. 운영자는 **불필요한 표면(공유·서비스)을 줄여 공격 면적을 최소화** 해야 한다.

### 4-1. 하드디스크 기본 공유 제거

**대상**

- 하드디스크 기본 공유: `C$`, `D$` 등 (드라이브 단위)
- 원격 관리·IPC 용 기본 공유: `ADMIN$`, `IPC$`

**설정 경로 (GUI)**

```
컴퓨터 관리 → 시스템 도구 → 공유 폴더 → 공유
```

**명령어**

```cmd
> net share C$ /delete
```

**레지스트리 영구 적용**

`net share` 로 공유를 중지해도 운영체제가 재시작하면 다시 생성되므로, 영구적으로 막으려면 레지스트리를 수정한다.

```
HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\LanmanServer\Parameters
[AutoShareServer (DWORD) = 0]
```

> *원본 표기 보정*: 통합자료에는 `AuthShareServer` 로 기재되어 있으나, 1과목.txt 와 Microsoft 공식 표기 모두 `AutoShareServer` 가 정확하다. 키 이름은 OS 가 인식해야 하는 식별자이므로 정확한 표기를 사용한다.

### 4-2. 공유 권한 및 사용자 그룹 설정

공유 폴더에 **Everyone 그룹** 이 포함되어 있으면 익명 사용자에 의한 접근이 가능하므로 Everyone 그룹을 제거한다.

**설정 경로**

```
컴퓨터 관리 → 시스템 도구 → 공유 폴더 → 공유 →
[리소스 속성 → Everyone 그룹 제거]
```

### 4-3. 불필요한 서비스 제거

취약한 서비스가 디폴트로 설치·실행되고 있으면 공격 지점이 될 수 있다. 사용자 환경에서 필요하지 않은 서비스는 중지·사용 안 함 처리한다.

**대표적인 취약 서비스**

| 서비스 | 위험성 |
|---|---|
| **Alerter** | 관리자 알림용. 메시지 스푸핑·악용 가능 |
| **Clipbook** | 클립보드 공유. 정보 유출 경로 |
| **Messenger** | 네트워크 메시징. 스팸·스푸핑에 악용 |
| **Simple TCP/IP Services** | 다음 네 서비스의 묶음으로, DoS 공격에 악용된다 |
| └ Echo (7/tcp) | 받은 패킷을 그대로 되돌림 |
| └ Discard (9/tcp) | 받은 패킷을 버림 |
| └ Character Generator (19/tcp) | 임의 문자열을 생성·전송 |
| └ Daytime (13/tcp) | 현재 시각 텍스트 응답 |
| └ Quote of the Day | 짧은 인용문 응답 |

**설정 경로**

```
서비스 → [불필요한 서비스] → 속성 → 일반 →
시작 유형: 사용 안 함
서비스 상태: 중지
```

### 시험 포인트
- 기본 공유 4종(C$ / D$ / ADMIN$ / IPC$) 의 용도 + 제거 명령(`net share /delete`) + 영구 적용 레지스트리 키(`AutoShareServer`).
- Simple TCP/IP Services 의 4종 포트(7/9/13/19) 는 자주 출제되며, 이 포트들은 [03. 어플리케이션 보안] 과 [02. 네트워크 보안] 에서도 다시 등장한다.
- "Everyone 그룹 제거" 는 공유 폴더 보안의 1차 항목.

## 5. 감사 정책 설정

### 배경

감사 정책은 **"어떤 보안 이벤트를 어디까지 기록할지" 를 정의하는 규칙**이다. 정책이 너무 높으면 로그가 과도하게 쌓여 진짜 중요한 이벤트와 혼동되고 시스템 성능에도 영향을 준다. 반대로 정책이 너무 낮으면 사고 발생 시 원인 파악이 어렵고 법적 증거로도 사용하기 어렵다. 따라서 적절한 균형이 핵심이다.

### 설정 경로

```
로컬 보안 정책 → 보안 설정 → 로컬 정책 → 감사 정책
```

### 9개 정책 항목과 권장 설정

| 감사 항목 | 권장 설정 | 비고 |
|---|---|---|
| **개체 액세스** | 감사 안 함 | 활성화 시 로그가 폭발적으로 증가 |
| **계정 관리** | 실패 | 계정 생성·삭제·변경 실패 추적 |
| **계정 로그온 이벤트** | 성공, 실패 | **도메인 계정에 대한 로그온** 이벤트 |
| **권한 사용** | 실패 | 권한 행사 실패 추적 |
| **디렉터리 서비스 액세스** | 실패 | AD 객체 접근 실패 추적 |
| **로그온 이벤트** | 성공, 실패 | **로컬 계정에 대한 로그온** 이벤트 |
| **시스템 이벤트** | 감사 안 함 | 시스템 시작·종료, 보안 로그 관련 |
| **정책 변경** | 성공, 실패 | 보안 정책 자체의 변경 추적 |
| **프로세스 추적** | 감사 안 함 | 활성화 시 로그가 폭발적으로 증가 |

### 5-1. 감사 정책 분류 — 시험 출제 패턴

원본은 9개 항목을 **세 카테고리** 로 묶어 외우게 한다.

| 카테고리 | 항목 |
|---|---|
| **감사 안 함** | 개체 액세스 / 시스템 이벤트 / 프로세스 추적 |
| **실패에 대해서만 감사** | 계정 관리 / 권한 사용 / 디렉터리 서비스 액세스 |
| **모두 감사 (성공 · 실패)** | 계정 로그온 이벤트 / 로그온 이벤트 / 정책 변경 |

### 5-2. 계정 로그온 이벤트 vs 로그온 이벤트

이름이 비슷해 헷갈리기 쉬운 두 항목의 차이는 다음과 같다.

| 항목 | 대상 |
|---|---|
| **계정 로그온 이벤트** | 도메인 계정에 대한 로그온 성공·실패 |
| **로그온 이벤트** | 로컬 계정에 대한 로그온 |

### 시험 포인트
- 9개 항목의 권장값을 전부 외우는 것이 부담스러우면, 위 **세 카테고리 분류** 만이라도 정확히 외운다.
- "계정 로그온 = 도메인" / "로그온 = 로컬" 의 매핑은 함정 문제로 자주 나온다.
- 감사 정책의 균형 — 너무 높으면 성능·노이즈, 너무 낮으면 증거 부족.

## 단원 요약

1. **계정 관리 4종** — Administrator 이름 변경 / Guest 사용 안 함 / 불필요 계정 제거 / 관리자 그룹 최소화.
2. **패스워드 정책 6종** — 복잡성 사용 / 최근 암호 12개 / 최대 사용 90일 / 최소 사용 1일 / 최소 길이 8자 / 해독 가능 암호화 사용 안 함.
3. **계정 잠금 정책 3종** — 잠금 기간 60분 이상 / 임계값 5 이하 / 다음 시간 후 60분 이상.
4. **공유 관리** — 기본 공유(C$/D$/ADMIN$/IPC$) 제거, `net share /delete`, 영구 적용은 레지스트리 `AutoShareServer = 0`. Everyone 그룹 제거.
5. **불필요한 서비스 제거** — Alerter / Clipbook / Messenger / Simple TCP/IP (Echo 7, Discard 9, Daytime 13, Chargen 19, Quote of the Day).
6. **감사 정책 9종** — 감사 안 함(개체액세스·시스템·프로세스) / 실패만(계정관리·권한·디렉토리) / 모두(계정로그온·로그온·정책변경). 계정 로그온 = 도메인, 로그온 = 로컬.
