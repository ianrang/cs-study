# 06. 시스템 공격

> **이 문서가 다루는 것**: 출제기준 *2.시스템 보안위협 및 공격기법* 의 시스템 공격기법. 패스워드 크래킹·버퍼 오버플로우(BoF)·레이스 컨디션·포맷 스트링·백도어·시스템 자원 고갈 공격의 정의·공격 과정·대응 방안. 자격증 1과목에서 03·04 와 함께 출제 비중이 가장 높은 영역이며, **공격기법의 정의와 대응 명령·옵션**이 빈출 포인트다. APT 단계와 MITRE ATT&CK 매트릭스는 [외부자료 검증 체크리스트](../docs/외부자료-검증체크리스트.md) §2 의 1차 출처(MITRE ATT&CK, KISA 침해사고 분석 보고서)로 보강한다.
>
> **출제기준 매핑**: 2.시스템 보안위협 및 공격기법 — 1) 시스템 보안위협(보안 취약점에 의한 도청·정보유출·변조·비인가 접근 등) / 2) 시스템 공격기법(APT, 알려진 취약점, 취약점 스캐너, SQL Injection, Buffer overflow 등). SQL Injection 은 3과목(애플리케이션 보안) 에서 다룬다.
>
> **원본 매핑**: 통합자료 line 1136 ~ 1311 (시스템 공격), 1과목.txt line 1236 ~ 1308 (시스템 해킹 — BoF·레이스·포맷 3종 한정). APT·MITRE ATT&CK 는 외부 보강.

## 0. 들어가며 — 시스템 공격은 어디를 노리는가

01~04 단원이 시스템의 *구조와 보안 설정* 을 다뤘다면, 이 단원은 그 구조의 약점을 노리는 *공격기법* 을 다룬다. 자격증 1과목의 시스템 공격은 크게 다섯 갈래다.

| 공격 갈래 | 노리는 약점 | 대표 공격 |
|---|---|---|
| **인증 우회** | 약한 패스워드 / 해시 | 패스워드 크래킹, 레인보우 테이블 |
| **메모리 변조** | 입력 검증 부재 / 버퍼 경계 | 버퍼 오버플로우(스택·힙), 포맷 스트링 |
| **타이밍 공격** | 임시 파일·심볼릭 링크 처리 | 레이스 컨디션 |
| **사후 침투 유지** | 미관리 통로 / 포트 | 백도어 |
| **자원 소진** | 디스크·메모리·프로세스 한계 | 시스템 자원 고갈 |

이 갈래들은 단독으로 사용되기도 하지만, 실제 침해사고에서는 **APT(Advanced Persistent Threat)** 시나리오 안에서 단계적으로 결합된다. §7 에서 MITRE ATT&CK 매트릭스로 그 결합 구조를 정리한다.

이 단원의 대응 방안은 03·04 에서 본 보안 설정과 직접 연결된다 — 예를 들어 레이스 컨디션의 **umask 022** 권장은 03 §5 의 권한·umask 설정에서 그대로 이어지고, 백도어 탐지의 **포트 활동 점검** 은 04 §4 의 서비스 관리에서 기반을 다졌다.

## 1. 패스워드 크래킹

### 1-1. 정의

> *원본 표기 보정*: 통합자료 line 1138 ~ 1139 는 OCR 과정에서 줄바꿈이 깨져 "설명기술을 통해 사용자의 ID, 패스워드를 찾는 도구로 활용" 과 "악의적인 목적을 가지고 시스템에 침입하는 행위로 크래킹" 이 분리 표기됨. 정상 표기는 다음과 같다.

악의적인 목적을 가지고 시스템에 침입하는 행위로, **크래킹 기술을 통해 사용자의 ID·패스워드를 찾는 도구로 활용** 된다. 공격 원리는 **ID·패스워드를 대입하여 맞는지·틀리는지를 지속적으로 수행** 하는 방법이다.

### 1-2. 공격 종류

| 공격 | 원리 |
|---|---|
| **사전 공격** (Dictionary Attack) | 패스워드에 자주 사용되는 사전 단어를 미리 사전 파일로 만들어 하나씩 대입 |
| **무차별 공격** (Brute Force Attack) | 패스워드에 사용될 수 있는 문자열의 범위 내에서 생성 가능한 모든 패스워드를 대입 |
| **혼합 공격** (Hybrid Attack) | 사전 공격 + 무차별 공격 — 사전 파일에 있는 문자열에 문자·숫자 등을 추가로 무작위 대입 |
| **레인보우 테이블 공격** | 해시 테이블과 R 함수의 반복 수행을 통해 일치하는 해시값을 찾아 패스워드를 역산. 윈도우의 LM 패스워드를 몇 분 만에 크래킹하여 유명해짐 |

### 1-3. 대표 도구

| 도구 | 용도 |
|---|---|
| **John the Ripper** | 패스워드 점검 도구로 가장 잘 알려진 프로그램 |
| **pwdump** | 윈도우에서 패스워드를 덤프할 수 있는 도구 |
| **L0phtCrack** | 원격 및 로컬 서버·PC 에 대해 패스워드를 점검 |
| **ipccrack** | 사용자 계정 패스워드를 원격지에서 추측하여 취약점을 점검 |
| **chntpw** | 물리적 접근이 가능한 시스템에서 패스워드를 리셋 |
| **ERD Commander** | 윈도우 시스템에서 패스워드를 복구해야 하는 경우에 사용 |

### 시험 포인트
- **사전 / 무차별 / 혼합 / 레인보우 테이블** — 4종 매칭.
- 레인보우 테이블 = **윈도우 LM 해시 크래킹** 과 짝지어 출제.
- 도구별 매핑: **John the Ripper(범용 점검) / pwdump(윈도우 덤프) / L0phtCrack(원격·로컬) / chntpw(물리 리셋) / ERD Commander(복구)**.

## 2. 버퍼 오버플로우 (Buffer Overflow, BoF)

### 2-1. 정의

연속된 메모리 공간을 사용하는 프로그램에서 **할당된 메모리의 범위를 넘어선 위치에 자료를 읽거나 쓰려고 할 때** 버퍼 오버플로우가 발생한다. 발생 시 다음 결과가 나타난다.

- 프로그램의 오작동 유발
- 악의적인 코드를 실행시켜 **공격자가 프로그램을 통제할 수 있는 권한을 획득**

### 2-2. 메모리 구조

BoF 의 동작 원리를 이해하려면 프로세스 메모리 구조를 알아야 한다.

| 영역 | 저장 내용 | 특성 |
|---|---|---|
| **Text** | 프로그램 코드와 상수 | 읽기만 가능 |
| **Data** | 전역 변수, 정적 변수 | 읽기/쓰기 |
| **Heap** | 동적 메모리 호출에 의해 할당 | 사용자가 동적으로 할당 |
| **Stack** | 함수 인자, 함수 내 지역 변수, 함수의 반환 주소(RET) | **상위 메모리 주소 → 하위 메모리 주소** 로 데이터 저장 |

### 2-3. BoF 분류

| 분류 | 원리 | 결과 |
|---|---|---|
| **스택 버퍼 오버플로우** | 스택에 할당된 버퍼들이 정의된 버퍼의 한계치를 넘는 경우 | **복귀 주소(RET) 를 변경**하고 공격자가 원하는 임의 코드 실행 |
| **힙 버퍼 오버플로우** | 힙에 할당된 버퍼들이 최초 정의된 힙의 메모리 사이즈를 넘는 경우 | **데이터·함수 주소** 를 변경하여 공격자가 원하는 임의 코드 실행 |

### 2-4. 공격 메커니즘 (관련 용어·문법)

| 용어 | 의미 |
|---|---|
| **null 문자** | `0x00` (`\0`) 으로 문자열의 끝을 표현 |
| **문자열 정의** | `ABCD` 라는 4개 문자로 구성된 문자열을 정의하면 `ABCD\0` |
| **`argc`** | 전달되는 매개변수의 개수 |
| **`argv[0]`** | 프로그램 명 |
| **`argv[1]`** | 입력 문자열 |

**Perl 스크립트를 이용한 공격 예시**:

```bash
[프로그램] `perl -e 'print "A"x16, "[메모리 주소]"'`
```

`main` 함수 호출이 끝나면 RET 영역의 주소를 참조하여 **Shell_code** 를 실행 (root 권한 쉘 획득).

**Shell_code 예시** (root 쉘 획득용):

```c
shell_code()
{
    setreuid(0, 0);     // ruid 와 euid 를 root 로 설정
    setregid(0, 0);     // rgid 와 egid 를 root 그룹으로 설정
    system("/bin/bash"); // root 및 root 그룹으로 쉘 실행
}
```

### 2-5. 대응 방안

| 대응 | 원리 |
|---|---|
| **스택 가드** (Stack Guard / Canary) | 메모리 상에서 프로그램의 복귀 주소와 변수 사이에 **특정 값(카나리)** 을 저장. 그 값이 변경되었을 경우 프로그램 실행을 중단 |
| **스택 쉴드** (Stack Shield) | 함수 시작 시 복귀 주소를 **글로벌 RET** 라는 특수 스택에 저장. 함수 종료 시 저장된 값과 스택의 RET 값을 비교해 다를 경우 프로그램 실행을 중단 |
| **ASLR** (Address Space Layout Randomization) | **주소 공간 배치를 난수화** 하여 실행 시마다 메모리 주소를 변경. 악성코드에 의한 특정 주소 호출을 방지 |
| **보안 코딩** | `strlen(argv[1]) >= sizeof(buffer)` 검사로 입력 문자열이 버퍼보다 클 때 `exit(1)`. `strcpy` 대신 **`strncpy`**(길이 제한) 사용 |

**ASLR 활성화 옵션** (Linux):

```bash
echo 0 > /proc/sys/kernel/randomize_va_space   # 난수화 안 함
echo 1 > /proc/sys/kernel/randomize_va_space   # 힙 제외 모두 난수화
echo 2 > /proc/sys/kernel/randomize_va_space   # 모두 난수화
```

### 시험 포인트
- 메모리 구조 4영역: **Text(읽기 전용) / Data(전역·정적) / Heap(동적) / Stack(지역·인자·RET)**. 스택은 **상위 → 하위** 로 저장.
- 분류: 스택 BoF = **RET 변경**, 힙 BoF = **데이터·함수 주소 변경**.
- 대응 3종: **스택 가드(카나리) / 스택 쉴드(글로벌 RET) / ASLR(난수화)** + 보안 코딩(`strncpy`).
- ASLR 옵션 **0 / 1 / 2** 의미.

## 3. 레이스 컨디션 공격 (Race Condition)

### 3-1. 정의

> *원본 표기 보정*: 통합자료 line 1229 ~ 1230 의 OCR 줄바꿈 결함("설명때 악의적인 프로그램을 통해 그 프로세스의 실행 중에" / "둘 이상의 프로세스나 스레드가 공유자원에 동시에 접근할") 을 정상 표기로 보정한다.

둘 이상의 프로세스나 스레드가 공유자원에 동시에 접근할 때 **악의적인 프로그램을 통해 그 프로세스의 실행 중에 끼어들어 악의적인 행위를 수행** 하는 공격이다. 접근하는 순서에 따라 비정상적인 결과가 발생하는 조건/상황을 의미한다.

| 용어 | 의미 |
|---|---|
| **임시 파일** | 실행되는 프로세스가 임시로 만드는 정상적인 파일 |
| **목적 파일** | 프로세스 실행 중에 끼어들어 임시파일을 심볼릭 링크한 파일 |

특히 **`setuid` 가 설정되어 있다면** 중요 자원에 root 로 접근하는 심각한 문제가 발생할 수 있다.

### 3-2. 공격 과정

```bash
# 1. 목적파일에 데이터를 쓸 수 있는 소스파일 생성
#    (파일에 데이터를 쓸 수 있는 코드)

# 2. 소스파일을 컴파일하여 setuid 설정
#    (실행 중인 동안 root 권한으로 파일 접근)
chmod 4755 [실행파일]   # 03 §5 의 SUID 4 비트 설정

# 3. 임시파일(tmp.dat) 에 심볼릭 링크 생성
ln -s /etc/shadow /tmp/tmp.dat

# 4. 컴파일한 실행파일 [목적파일] [조작 값]
#    (목적파일이 가리킨 파일 조작)
```

### 3-3. 대응 방안

| 대응 | 원리 |
|---|---|
| **임시파일 생성 금지** | 가능하면 임시파일을 생성하지 않도록 함 |
| **파일 생성/쓰기 금지** | 파일 생성 시 이미 동일한 파일이 존재하는 경우 생성/쓰기 금지 |
| **링크파일 실행 중단** | 사용하고자 하는 파일에 링크가 걸려 있으면 실행을 중단 |
| **`umask` 확인** | 최하 **022** 정도로 유지하여 임시로 생성한 파일이 (다른 사용자에게) 변조되지 않도록 함 |

### 시험 포인트
- 정의: **공유자원 동시 접근 + 순서 의존 비정상 결과**.
- 핵심 기법: **임시파일 + 심볼릭 링크 + setuid**.
- 대응 4종: 임시파일 생성 금지 / 동일 파일 존재 시 생성·쓰기 금지 / 링크 존재 시 실행 중단 / **umask 022**.

## 4. 포맷 스트링 공격 (Format String Attack)

### 4-1. 정의

> *원본 표기 보정*: 통합자료 line 1259 ~ 1260 의 OCR 줄바꿈 결함("설명 스트링을 그대로 사용하는 경우 포맷 스트링 공격 발생" / "외부로부터 입력된 값을 검증하지 않고 입출력 함수의 포맷") 을 정상 표기로 보정한다.

외부로부터 입력된 값을 검증하지 않고 입출력 함수의 **포맷 스트링을 그대로 사용하는 경우** 발생하는 공격이다. 포맷 스트링은 **C 언어의 `printf()` 등의 함수에서 사용하는 문자열의 입/출력 형태를 정의하는 문자열** 로, **서식 문자열** 이라고도 표현한다.

공격 시 가능한 행위:

- 프로세스를 죽게 만듦
- 프로세스 메모리 보기
- 임의의 메모리 덮어쓰기

### 4-2. 포맷 문자

| 문자 | 의미 |
|---|---|
| **`%d`** | 10진수 정수 출력 |
| **`%f`** | 실수 출력 |
| **`%c`** | 문자 출력 |
| **`%s`** | 문자열 출력 |
| **`%x`** | 16진수 정수 출력 |
| **`%n`** | 이전까지 출력한 총 바이트 수를 지정한 변수에 저장 (**값을 저장하는 용도**) |

### 4-3. 공격 과정

1. **`%x`** 을 통해 메모리 내용 참조 및 원하는 위치로 이동
2. **`%n`** 을 통해 **Return Address 를 악성코드가 위치한 주소로 변조** 하여 실행

### 4-4. 대응 방안

**포맷 스트링 직접 사용 금지** — 함수 사용 시 포맷 스트링을 명시적으로 지정하여 간접 참조가 되도록 한다.

```c
// 취약 코드 — 외부 입력을 포맷 스트링으로 직접 사용
printf(argv[1]);

// 안전 코드 — 포맷 스트링을 "%s" 로 명시 지정
printf("%s", argv[1]);
```

### 시험 포인트
- 정의: **외부 입력값을 검증하지 않은 포맷 스트링** = 취약점.
- 핵심 문자: **`%x`(메모리 참조) + `%n`(메모리 쓰기)** 조합으로 RET 변조.
- 대응: **포맷 스트링 명시 지정** (`printf("%s", argv[1])`).

## 5. 백도어 (Backdoor)

### 5-1. 정의

시스템의 보안을 약화시키고 비인가자가 시스템에 불법적으로 접근할 수 있게 하는 통로다.

- 서비스 기술자나 유지보수 프로그래머의 **접근 편의를 위해 시스템 설계자가 고의적으로 만들어 놓은 통로**
- 침해사고 분석 보고서에서는 **공격자가 침투 후 재접근을 위해 설치한 통로** 도 백도어로 분류한다[^1].

### 5-2. 대응 방안

| 대응 | 원리 |
|---|---|
| **현재 동작 중인 프로세스 확인** | 정상 프로세스를 미리 외워두고 `Csrss`, `Svchost` 등 의심 프로세스 확인 |
| **H-IDS (Host-based IDS) 사용** | 백도어가 특정 포트에 대하여 대기하고 있기 때문에 **포트의 활동을 탐지** |

### 시험 포인트
- 정의: **고의적으로 만들어진 우회 통로**.
- 대응: **정상 프로세스 목록 숙지** + **H-IDS 의 포트 활동 탐지**.

## 6. 시스템 자원 고갈 공격

### 6-1. 정의

> *원본 표기 보정*: 통합자료 line 1294 ~ 1295 의 OCR 줄바꿈 결함("설명정상적인 동작을 방해" / "CPU 사용률 높이거나 메모리를 소모하도록 하여 시스템의") 을 정상 표기로 보정한다.

**CPU 사용률을 높이거나 메모리를 소모하도록 하여 시스템의 정상적인 동작을 방해** 하는 공격이다.

### 6-2. 공격 종류

| 종류 | 원리 |
|---|---|
| **가용 디스크 자원 고갈 공격** | 파일을 생성하고 그 파일에 계속 내용을 써 나가 가용 디스크 자원을 고갈시킴 |
| **가용 메모리 자원 고갈 공격** | `malloc()` 함수를 통해 **메모리 할당만 계속 수행** 하여 가용 메모리 자원을 고갈시킴 |
| **가용 프로세스 자원 고갈 공격** | `fork()` 라는 프로세스 할당 함수를 **무한대로 호출** 하여 가용 프로세스 자원을 고갈시킴 (Fork bomb) |
| **프로세스 죽이기 공격** | 공격자가 **루트 권한을 획득한 상태** 에서 스크립트를 통해 사용 중인 프로세스를 죽여 서비스가 동작하지 못하게 함 |

### 시험 포인트
- 4종 매핑: **디스크 = 파일 생성 / 메모리 = `malloc` / 프로세스 = `fork` / 프로세스 죽이기 = root 권한 + 스크립트**.
- `fork()` 무한 호출 = **Fork bomb** 의 원리.

## 7. APT 공격과 MITRE ATT&CK (외부 보강)

### 7-1. APT (Advanced Persistent Threat)

**APT** 는 특정 표적을 대상으로 **장기간에 걸쳐 은밀하게** 침투·잠복·정보 유출을 수행하는 지능형 지속 위협이다. §1~§6 의 공격기법이 단독으로 쓰이는 것과 달리, APT 는 여러 공격기법을 **시간 축으로 결합** 하여 사용한다[^1].

### 7-2. Cyber Kill Chain (Lockheed Martin 모델)

침해 시나리오를 7단계로 표준화한 모델로, 자격증 1과목 APT 출제 영역의 핵심 프레임워크다[^2].

| 단계 | 영문 | 설명 |
|---|---|---|
| **1. 정찰** | Reconnaissance | 표적의 정보 수집 (도메인·IP·인력·기술 스택 등) |
| **2. 무기화** | Weaponization | 수집한 정보를 바탕으로 익스플로잇·악성코드 제작 |
| **3. 전달** | Delivery | 표적에 무기 전달 (스피어 피싱 메일·USB·악성 사이트 등) |
| **4. 취약점 익스플로잇** | Exploitation | 전달된 무기가 취약점을 이용해 코드 실행 |
| **5. 설치** | Installation | 백도어·악성코드 설치 (§5 참조) |
| **6. 명령·통제** | Command & Control (C2) | 외부 C2 서버와 통신 채널 확립 |
| **7. 목적 달성** | Actions on Objectives | 정보 유출·시스템 파괴·권한 탈취 등 본래 목적 수행 |

### 7-3. MITRE ATT&CK 매트릭스

**MITRE ATT&CK** 는 실제 침해사고에서 관측된 공격기법을 **Tactics(전술) × Techniques(기법)** 매트릭스로 정리한 지식 베이스다[^3]. Kill Chain 이 시간 축 모델이라면 ATT&CK 는 *공격자가 각 단계에서 실제로 어떤 기법을 사용하는지* 의 카탈로그다.

**Enterprise ATT&CK 의 14 Tactics**:

| # | 전술 | 의미 |
|---|---|---|
| 1 | **Reconnaissance** | 정찰 — 표적 정보 수집 |
| 2 | **Resource Development** | 자원 개발 — 인프라·계정·도구 준비 |
| 3 | **Initial Access** | 초기 접근 — 표적 환경에 첫 진입 |
| 4 | **Execution** | 실행 — 악성 코드 실행 |
| 5 | **Persistence** | 지속성 — 재부팅 후에도 접근 유지 |
| 6 | **Privilege Escalation** | 권한 상승 — root/admin 권한 획득 |
| 7 | **Defense Evasion** | 방어 회피 — 탐지 우회 |
| 8 | **Credential Access** | 자격증명 접근 — 패스워드·키 탈취 (§1 의 패스워드 크래킹과 연결) |
| 9 | **Discovery** | 탐색 — 내부 자원·계정·구조 파악 |
| 10 | **Lateral Movement** | 측면 이동 — 다른 시스템으로 확산 |
| 11 | **Collection** | 수집 — 표적 정보 수집 |
| 12 | **Command and Control** | 명령·통제 — C2 서버 통신 |
| 13 | **Exfiltration** | 정보 유출 — 수집한 데이터 외부 전송 |
| 14 | **Impact** | 영향 — 데이터·시스템 파괴·암호화·서비스 거부 |

### 7-4. Cyber Kill Chain vs MITRE ATT&CK

| 비교 항목 | Cyber Kill Chain | MITRE ATT&CK |
|---|---|---|
| **개발 주체** | Lockheed Martin (2011) | MITRE Corporation (2013~) |
| **구성** | 7단계 시간 축 모델 | 14 Tactics × 수백 Techniques 매트릭스 |
| **관점** | 공격의 *진행 순서* | 공격의 *기법 카탈로그* |
| **활용** | 침해 시나리오 단계 식별 | 탐지 룰 개발, 방어 매핑, Red/Blue Team 훈련 |

### 시험 포인트
- APT = **장기·은밀·표적 지향**. 단일 공격기법이 아니라 **결합된 시나리오**.
- Kill Chain **7단계** 순서: 정찰 → 무기화 → 전달 → 익스플로잇 → 설치 → C2 → 목적 달성.
- ATT&CK = **Tactics × Techniques** 매트릭스. 14 Tactics 중 **Initial Access → Privilege Escalation → Lateral Movement → Exfiltration** 의 흐름은 침해사고 시나리오 단계 식별의 표준 축이다.
- Kill Chain = **시간 축**, ATT&CK = **기법 카탈로그**.

## 단원 요약

1. **패스워드 크래킹** — 사전·무차별·혼합·레인보우 테이블 4종. 도구: John the Ripper / pwdump / L0phtCrack / chntpw / ERD Commander.
2. **버퍼 오버플로우** — 메모리 4영역(Text/Data/Heap/Stack). 스택 BoF(RET 변경) vs 힙 BoF(데이터·함수 주소 변경). 대응 = 스택 가드(카나리) / 스택 쉴드(글로벌 RET) / ASLR(0/1/2) / `strncpy`.
3. **레이스 컨디션** — 공유자원 동시 접근 + 순서 의존. 임시파일 + 심볼릭 링크 + setuid 가 핵심. 대응 = umask 022.
4. **포맷 스트링** — `%x`(읽기) + `%n`(쓰기) 조합으로 RET 변조. 대응 = `printf("%s", argv[1])` 처럼 포맷 스트링 명시.
5. **백도어** — 고의적 우회 통로. 대응 = 정상 프로세스 숙지 + H-IDS 포트 활동 탐지.
6. **시스템 자원 고갈** — 디스크(파일) / 메모리(`malloc`) / 프로세스(`fork`, Fork bomb) / 프로세스 죽이기(root + 스크립트).
7. **APT · MITRE ATT&CK** — Kill Chain 7단계(시간 축) + ATT&CK 14 Tactics(기법 카탈로그). 단일 기법이 아닌 **시나리오 단위** 위협.
