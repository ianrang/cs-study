* 정보보호의 일반적 정의: 조직의 정보 자산이 다양한 내/외부 위협으로부터 기밀성, 무결성, 가용성 등이 보장될 수 있
  도록 관리작, 기술적, 물리적 보호조치를 마련함

* 정보보호의 목표 : 기밀성, 무결성, 가용성을 보장 -> 정보보호의 삼요소 (CIA Traid )
-> 다양한 정보보호 환경을 고려하여 인증성, 책임성이 추가

* 기밀성 : 권한있는자만이 정보자산에 접근하여 그 내용을 알 수 있도록 하는 것, 정당하지 않은 자에 의해 정보가 노출되
  지 않도록 보장하는것 -> 대칭키 암호화/비대칭키 암호화

* 무결성 : 권한있는자만이 정보자산에 접근하여 그 내용을 생성하거나 삭제할 수 있도록 보장하는 것 , 정당하지 않은 자
  에 의해 정보가 위변조 되지 않도록 보장하는 것 -> 암호학적 해시 함수, 메시지 인증코드, 디지털서명/전자서명

* 가용성 : 권한 있는자의 정보 자산 접근 필요시 언제든지 접근할 수 있도록 보장하는 것 -> 장비 이중화 구성, 서버 클러
  스터링, 디스크 RAID 구성, 백업 및 소산 관리, 재하복구센터 구축

* 인증성 또는 인증 : 사용자 인증 -> 정보자산에 접근하려는 자의 신원이 주장된 실체와 일치함을 보이는 것 ( 지식 기
  반 인증, 소유기반 인증, 존재 기반 인증) , 메시지 인증 -> 수신한 메시지가 올바른 상대방이 보낸 메시지임을 보장하는
  것 ( 메시지인증코드, 디지털 서명/전자서명)

* 책임성 : 정보자산에 접근하는자가 접근 결과에 대해서 책임지도록 보장하는 것으로 책임 추적성과 부인방지로 구분할
  수 있음 -> 책임추적성 ( 주체를 고유하게 식별하여 그 행위를 추적할 수 있도록 보장함 ) -> 감시와 로깅 , 부인방지 (
  주체가 한 행위를 나중에 부인하지 못하도록 보장함 ) -> 디지털 서명/ 전자서명

* 접근 통제 : 인증받은 사용자가 시스템, 정보등의 자원에 대하여 어떤 유형의 접근이 가능한지 결정하는 것 -> 적절한 접
  근 권한을 가진 사용자만이 정보자산등에 접근할 수 있도록 통제함
-> 주체, 객체, 접근으로 구성 됨

* 접근통제 절차 과정 3단계 : 식별 -> 인증(Authentication) -> 인가(Authorization)
* 접근 통제를 위한 요구사항 : 접근 통제의 기본 원칙
- 입력의 신뢰성
- 직무 분리
- 최소 권한 부여

* 사용자 인증
- 지식 기반 ( 패스워드, 핀 ) something you know
- 소유 기반 ( 스마트카드, OTP 발생기, 인증 토큰 ) something you have
- 존재 기반 ( 지문, 홍채, 망막, 정맥 ) something you are
- 위치 기반 (사용자 네트워크 주소, 콜백) somewhere you are
- 행위 기반 (키 스트로크, 서명, 걸음걸이 )something you do
* 다중인증
- Multi Factor 인증 -> 두가지 인증 유형을 섞은 것
* 접근통제 보안 모델
- 임의적 접근 통제 (DAC ) : 사용자 주체의 신원 정보와 자원에 대한 사용자의 접근 권한을 기반으로 접근 통제를 수행
  하는 정책 - > ACL, ACM, CL 등 ..
-> 자원의 소유자가 해당 자원에 대한 다른 사용자의 접근 권한을 임의로 지정할 수 있으므로 임의적이라는 용어를 사용
한다
-> 다양한 운영체제의 파일 시스템과 데이터베이스 시스템등에 적용된 접근통제 정책
-> 주체와 객체 각각에 대해 유연한 접근 통제가 가능함
-> 중앙 집중적인 접근 통제가 어려움, 일관성 있기 힘듬
-> 주체의 신원정보를 바탕으로 접근통제를 수행하므로 신원 정보를 위장한 불법적인 접근이 발생하면 접근통제에 심각
한 문제가 발생할 수 있음
-> 트로이 목마 공격에 취약함
- 강제적 접근통제( MAC) : 사용자에게 부여된 보안 취급인가와 자원에 부여된 보안수준을 비교하여 접근 통제를 수행
  하는 정책
-> 사용자가 임의로 다른 사용자에게 자원에 대한 접근 권한을 설정할 수 없고 관리자에 의해서만 사용자의 보안 취급 인
가와 자원의 보안 수준이 설정되어 접근통제를 수행하므로 강제적이라는 용어를 사용한다 -> BLP, Biba, Clark-
Wilson, Chinese wall 모델이 있음
-> BLP : 벨라 파듈라 모델 -> 기밀성을 보장하기 위한 정보흐름을 통제 , 최초의 수학적 모델, 높은 보안 수준에서 낮은
보안 수준으로 정보가 이동하는 것을 통제함 ( 주체 보다 보안수준이 높은 객체의 정보를 읽을 수 없고 주체보다 보안 수
준이 낮은 객체에 정보를 쓸 수 없다, No Read UP, No Write Down )
-> Biba 모델 : 관리자가 주체와 객체에 대한 무결성 수준을 부여하고 이를 비교하여 접근 통제를 수행하는 모델 -> 무결
성을 보장 -> 비인가자에 의한 정보의 위변조 방지만을 해결한 모델 -> 낮은 무결성 수준에서 높은 무결성 수준으로 정보
가 흐르는 것을 통제함 -> 주체보다 무결성 수준이 낮은 객체의 정보를 읽을 수 없고, 주체보다 무결성 수준이 높은 객체에
정보를 쓸 수 없다 ( No read Down, No Write Up)
-> Clark-Wilson 모델 : 상업적 응용에서 무결성의 3가지 목표를 모두 보장하기 위해 접근 통제를 수행함
-> Chinese Wall : 만리장성 모델, 이해관계가 충돌하는 상업적 응용에서 기밀성과 무결성을 보장하기 위해 접근 제어를
수행하는 모델

-RBAC : 역할기반 접근 통제 : 사용자에게 부여된 역할과 역할이 가지는 자원에 대한 접근 권한을 기반으로 접근통제를
수행하는 정책으로 임의적 접근 통제 정책과 구분하기 위해 비임의적 접근통제 정책이라고도 함 -> 역할에 자원에 대한
접근 권한을 부여하고 사용자에게 역할을 할당하는 방식이므로 개별 사용자마다 자원에 대한 접근 권한을 설정할 필요가
없다 -> 사용자가 많고 접근 권한이 자주 변경되는 환경에서 유리 -> 관리자에 의해서만 사용자의 역할이 부여되고 역할
이 가지는 자원에 대한 접근 권한이 설정 됨 -> 직무 분리/최소 권한 부여가 유리함 -> 임의적 접근 통제에 비해 주체와
객체 각각에 대한 접근 권한 설정의 유연성이 떨어짐

* 정보보호 정책 : 기본적으로 무엇이 수행되어야 하는가를 일목요연하게 기술한 지침과 규악으로 조직의 정보보호 목적
  과 활동에 관한 사항을 정의한 최상위 문서 -> 경영진의 의지 및 방향, 역할과 책임/대상과 범위, 관리적/기술적/물리
  적 정보보호 활동의 근거를 포함해야 함

* 표준 : 정보보호정책 달성을 위해 필요한 세부 요구사항을 구체적으로 정의한 규정으로 , 강제성을 가짐
* 지침 : 정보보호 정책에 따라 특정 시스템 또는 특정 분야별로 활동에 필요하거나 도움이되는 세부 요구사항에 대한 규
  정으로 강제성보다는 권고적이고 융통성있게 적용할 수 있는 사항
* 절차 : 정책/표준/지침을 준수하기 위해 구체적으로 어떻게 해야 하는지를 세부적으로 상세하게 설명한 문서 -> 수행해
  야할 업무들을 순서에 따라 단계적으로 설명

* 위험관리 : 조직의 자산에 대한 위험을 수용가능한 목표 위험수준 ( DoA) 으로 유지하기 위해 자산에 대한 위험을 분석
  하고 이러한 위험으로부터 자산을 보호하기 위한 비용 대비 효율적인 보호대책을 마련하는 일련의 과정

* 위험관리 5단계
- 위험관리 전략 및 계획 수립 : 조직에 적합한 위험관리 방법을 선택하고 비용과 시간, 인력 등 가용 자원을 고려하여 계
  획을 수립
- 위험 분석 : 자산,위협,취약성, 기존 보호대책 등을 분석하여 위험의 종류와 규모를 결정하는 단계
- 위험 평가 : 위험분석 결과를 이용하여 최종적으로 위험도를 평가하고 조직에서 “수용 가능한 목표 위험수준” 을 정하여
  이를 기준으로 위험의 대응 여부와 우선순위를 결정하는 단계
- 정보보호 대책 선정 : DoA를 초과하는 위험을 수용 가능한 수준으로 감소시키기 위해 위험처리 전략을 설정하고 적절
  한 통제사항을 선택함 -> 정보보호대책 명세서 작성
- 정보보호 계획 수립 : 선정한 보호대책을 구현하기 위한 계획을 수립하는 단계
* 위험 구성요소 :
- 위험 : 자산의 취약성을 이용한 위협에 의해 원하지 않는 사건이 발생하여 자산에 손실을 미칠 가능성을 위험이라고 함
- 잔여 위험 : 정보보호 대책을 구현한 후 남아 있는 위험
- 위험 -> 위험은 자산, 위협, 취약섬을 인자로한 함수적 관계로 표현 됨
- 위험의 3대 구성요소 : 자산, 위협, 취약성
- 위험의 4대 구성요소 : 자산,위협,취약성, 보호대책
* 자산 : 조직이 보호해야할 유/무형의 대상
* 위협 : 자신에 손실을 초래할 수 있는 원하지 않는 사건의 잠재적 원인
* 취약성 : 위협의 이용 대상이 되는 자산의 잠재적 속성을 취약성이라고 함 -> 위협이 발생한다고 반드시 자산에 손실이
  발생하는 것은 아님
* 보호대책 : 조직의 자산을 위험으로부터 보호하기 위한 안전 대첵 또는 통제 또는 위협을 감소시키기 위한 정보보호 대
  책

* 보호대책 유형에 따른 구분
- 기술적 보호 대책 : 보안 솔루션 도입 및 운영
- 관리적 보호 대책 : 보안 정책의 수립 및 시행 고나점
- 물리적 보호 대책 : 시설물 및 장비 보호 관점
*보호대책 시점에 따른 구분
- 예방 통제 : 발생할 수 있는 위험을 식별하여 사전에 대처하는 능동적 개념의 통제로 물리적 접근통제와 논리적 접근 통
  제로 구분할 수 있음
- 탐지 통제 : 예방 통제를 우회하여 발생하는 각종 위험을 탐지하는 통제
- 교정 통제 : 탐지된 위험에 대처하거나 감소시키는 통제
-> 위험을 파악함으로써 보안요구사항을 도출할 수 있고 보안요구사항을 충족시키는 보안대책을 선정하여 구현함으로
써 위험을 방어하고 위험을 감소시킴


* 위험 관리 전략 및 계획 수립
- 기준선 접근 방법 : 체크 리스트 형식의 표준화된 보호대책을 이용하여 보호의 기본 수준을 정하고 위험분석을 수행 ->
 시간과 비용을 절약할 수 있고 기본적인 보호 대책 선정 가능 / 조직의 특성이 고려되지 않음
- 비정형 접근방법 : 경험자의 지식을 기반으로 위험 분석을 수행 -> 작은 규모의 조직에 비용대 비효과적 / 수행자의 경
 험에 따라 중요 위험 중심으로 분석하므로 수행자의 경험이 부족한 위험 영역을 놓칠 가능성이 있음
- 상세 위험 분석 : 정형화되고 구조화된 위험 분석방법론에 따라 자산 분석, 위협 분석, 취약성 분석 등의 각 단계를 통
  해 조직의 위험들을 모두 상세하게 분석하는 접근 방법 -> 상세한 위험 분석을 통해 조직에 가장 적절한 보호대책을 수
  립할 수 있고 보안환경의 변화에 유연하게 대처 가능 / 전문적인 지식과 많은 노력이 요구 됨
- 복합 접근법 : 고위험 영역을 식별하여 상세 위험분석을 수행하고 그 외의 영역은 기준선 접근법을 사용함 -> 위험분석
  비용과 자원을 효과적으로 사용할 수 있음 / 고위험 영역을 잘못 식별하면 문제가 생김

* 위험 분석 : 자산,위협,취약성, 기존 보호대책 등을 분석하여 위험의 종류와 규모를 결정함
-> 절차는 “자산분석 ( 자산 식별 -> 자산 가치 평가 -> 자산 그룹핑 )-> 위협 평가 ( 위험 식별 -> 위협 평가 )-> 취약성
평가 ( 취약성 식별 -> 취약성 평가 -> 우려사항 평가 ) -> 기존 보안대첵 평가 -> 위험 평가 ( 위험도 평가 -> 수용 가능
한 목표 위험수준 결정 )” 로 진행 됨

* 정량적 위험 분석 방식 : 손실이나 위험의 크기를 금전적 가치로 표현하는 방식 - > 객관적/수학적
* 정성적 위험 분석 방식 : 손실이나 위험 크기를 개략적인 크기로 비교하는 방식 -> 비교적 간단하고 시간과 노력이 적게
  들지만 객관적 검증이 어렵고 평가자 개인에 따라 결과가 달라질 수 있음 -> 델파이(전문가 집단이 토론)/시나리오(어
  떠한사건도 기대대로 발생하지 않는다는 사실에 근거하여 일정 조건에서의 위협에 대한 발생 가능 결과를 추정 )/순위
  결정법 (비교 우위 순위 결정표에 의해 위험 항목들의 서술적 순위를 결정함 )/퍼지행렬법 (위험분석 요소들을 정성적
  인 언어로 표현된 값을 사용하여 기대 손실을 평가함 )

* 위험처리전략 유형 : 위험수용(현재의 위험을 받아들이고 잠재적 손실 비용을 감수함), 위험감소(위험을 감소시킬 수 있
  는 대책을 채택함), 위험전가( 보험이나 외주 등 제 3자에게 이전), 위험회피(프로세스나 사업을 수행하지 않고 포기함)

* 업무 연속성 계획 : BCP -> 인재, 천재지변, 테러등에 의한 각종 장애 및 재해로부터 업무중단이 발생할 경우 최대한
  빠른 시간 내에 업무를 복구함으로써 업무 연속성을 유지하기 위한 계획
