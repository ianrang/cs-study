* 침입탐지 시스템 : Snort -> 네트워크 트래픽을 더 많이 감시하고 분석함

- 네트워크 IDS/IPS : 네트워크 트래픽을 분석하여 공격을 탐지/차단하는 기능
* snort rule : snort rule/signature은 크게 헤더 부분과 바디 부분으로 구성되어 있음
- 헤더 : 처리 방식, 프로토콜, IP주소, 포트번호 등 처리할 패킷을 판단하는 기준을 명시
-> Rule Action : 패킷이 탐지되었을 때 처리 방식으로 8가지 유형이 있음
-> Protocols : 탐지할 프로토콜, 현재 4가지 프로토콜을 지원함 ( TCP, UDP, ICMP, IP )
-> IP Address : 출발지/목적지 IP, 출발지 정보는 공격자의 IP를 알지 못하는 경우가 대부분이기 때문에 any 로 지정
함
-> Port Numbers : 출발지/목적지 Port -> 포트 범위를 지정할 경우 콜론을 이용하여 표시 함
-> Direction Operator : -> 단방향, <> 양방향, <- 방향 지시자는 없음 !!!!

- 바디 : 패킷을 탐지하기 위한 규칙을 명시
* snort rule 헤더에서 Action 의 유형
- alert : 선택한 alert 방식을 이용해서 alert를 발생시키고 패킷을 로그에 남긴다
- log : 패킷을 로그에 남긴다
- pass : 패킷을 무시한다
- activate : alert를 발생시키고 대응하는 dynamic rule을 활성화 시킨다
- drop : 패킷을 차단하고 로그에 남긴다
- reject : drop rule과 동일하게 패킷을 차단하고 로그에 남긴 후 TCP일 경우 TCP Reset을 전송하고 UDP의 경우
   ICMP port unreachable 메시지를 전송함
- sdrop : 패킷을 차단하지만 로그에 남기지 않는다 ( slient )

- reference : rule 과 관련된 외부 취약점 정보 참조시 사용
- sid : snort rule id

   우선순위를 정의해 놓음 ( 1 높음, 2 중간, 3 낮음 )
- priority : rule의 우선순위에 대해 숫자로 표기함. priority 를 지정하면 classtype의 기본 우선순위를 덮어 씀
- rev : rule의 버전정보로 수정 횟수를 표기함. 수정을 할 때마다 1씩 증가함

* 비정상 TCP 패킷 -> SYN 과 FIN 은 동시에 설정할 수 없는 플래그임

우가 있고 FIN은 연결을 종료하는 플래그로 일부 시스템에서는 이러한 플래그 패킷에 대해 로그를 남기지 않는 특성이 있
어 많은 스캐닝 도구들이 SYN-FIN 패킷을 많이 사용 함 , 추가적으로 일반적으로 클라이언트는 1024 이상의 포트를 사
용함
-> snort rule 에서 flags : SF 형식으로 사용함
-> + 연산자가 있으면 SYN, FIN은 반드시 포함하고 다른 플래그도 설정될 수 있음
-> * 연산자가 있으면 지정한 플래그를 하나 이상 포함한 경우를 말함, 즉 SYN 또는 FIN 또는 SYN_FIN을 포함하고 다
른 flags들도 설정될 수 있음
-> ! 연산자가 있으면 지정한 플래그를 포함하지 않은 설정을 의미한다

* FIN 플래그만 설정된 비정상 TCP 패킷 -> FIN 스캔 -> flags : F
* 플래그가 설정되지 않은 비정상 TCP 패킷 ( NULL 패킷 ) -> Null스캔 -> flags : !UAPRSF

- Positive 정책 : default policy를 ACCEPT로 하고 차단할 부분만 DROP/REJCT 하는 정책

-> - -hitcount : - -rcheck 또는 - - update 옵션과 함께 사용된다. 해당 entry의 hit count가 지정한 값 이상인지

  장비, 애플리케이션 등 다양한 범위에서 발생하는 로그와 이벤트를 수집하여 빅데이터 기반의 상관관계 분석을 통해
  위협을 사전에 차단하는 통합보안관제 솔루션을 말함 -> 데이터 통합/상관관계 분석/알림/대시보드 기능이 있음
- 보안 오케스트레이션, 자동화 및 대응 시스템 ( SOAR ) : SOAR는 SOA, SIRP, TIP을 폭넓게 포함하는 개념으로 보

  유기적으로 협력할 수 있도록 지원하는 대응 플랫폼 , 보안 위협에 대한 자동화된 분석과 대응 환경을 만들어 보안 인
  력을 효율적으로 운영하면서 분석의 정확도를 높이고 대응 시간을 단축하기 위한 솔루션 -> 자동화된 통합보안관제 솔
  루션

* 루트킷의 hidden process 탐지 원리
- /proc -> 유닉스/리눅스 커널이 메모리상에 사용하고 있는 모든 자원들에 대한 정보들을 파일 형식으로 보관하는 파
  일 시스템, 프로세스등의 자원, 커널 파라미터 등에 대한 상태정보를 파일명으로 보관하고 있음 , 메모리 영역에 존재
  하는 파일 시스템으로 부팅시마다 새롭게 생성 됨
- /proc/<pid>/exe : 프로세스의 실행파일 및 경로를 확인할 수 있는 심볼릭 링크 파일
- /proc/<pid>/cmdline : 프로세스를 실행한 명령어 및 전달인자 정보를 담고 있는 파일로 프로세스가 실행된 방식을
  정확하게 파악하는데 활용 함
=> 일반적으로 루트킷은 공격자가 숨기고자 하는 프로세스가 출력되는 부분만 제거하고 출력하도록 ps 프로그램을 만들
어 타겟 시스템에 있는 정상적인 ps 프로그램과 교체하는 방식으로 동작함 -> 루트킷 탐지 프로그램은 ps 실행 결과와 /
proc 디렉터리에 있는 프로세스 정보를 비교하여 /proc 디렉터리에는 프로세스가 있지만 ps 실행 시 보이지 않는 프로세
스를 히든 프로세스로 탐지함 -> 일반적으로 공격자는 백도어를 실행한 후 실행파일 흔적을 숨기기 위한 목적으로 위 예
와 같이 실행 후 삭제하는 방식을 많이 사용함 -> 루트킷이 설치된 것으로 의심 된다면 변조/감염이 된 파일만 교체하는
것이 아니라 해당 시스템 전체를 재설치 하는 것이 좋음

* 루트킷 감지 여부 대응 방법
- RPM 명령을 이용하여 변조된 파일을 확인함
-> rpm -Uvh : 패키지파일이름.rpm : 해당 패키지 신규 설치 또는 업그레이드 수행
-> rpm -qf 파일절대경로 : 해당 파일이 어느 패키지에 포함되어 있는지 확인
-> rpm -V 패키지 이름 : V 옵션은 Verify를 의미하며 해당 패키지를 통해 설치된 파일의 무결성을 검사함 ( S: 파일 크
기 변경, M : 파일 퍼미션 변경, 5: MD5 체크섬 변경, T : 파일 수정시간 변경, U : 소유자 정보 변경, G : 소유 그룹 정보
변경, D : 장치 정보 변경, L : 심볼릭 링크 정보 변경
- strace 명령을 이용하여 변조된 파일을 확인함 -> strace 명령은 유닉스/리눅스 시스템에서 특정 프로그램의 시스
 템 콜과 시그널을 추적하는데 사용하는 디버깅 도구로 정상적인 프로그램과 변조된 프로그램의 시스템 콜을 비교하여
 변조 여부를 확인할 수 있음

* 리눅스 파일 속성 변경 : 속성은 특정 파일 자체에 대한 허가를 의미한다 . 퍼미션과 다른 개념임 ! ( 퍼미션은 사용자에
  따라서 특정권한을 부여하는 것을 의미함 )
-> lsattr 명령은 특정 파일에 대한 속성을 확인하고자 할 때 사용하고 chattr 명령은 속성을 변경하고자 할 때 사용 함
-> lsattr -i : 파일을 읽기 전용 모드로 만듬 / lsattr -a : 해당 파일에 내용 추가만 허용함 ( 파일 내용 변경 및 삭제는
불가) / lsattr -A : 해당 파일의 Access Time을 변경하지 않음
-> chattr : + 속성을 통해 속성을 설정하고 - 속성을 통해 속성을 제거 함

* 리버스 쉘 : 역으로 타겟 서버가 클라이언트/공격자로 접속해서 클라이언트가 타겟 서버의 쉘을 획득하는 방식
-> 방화벽을 우회하여 쉘을 획득하기 위함 : 일반적으로 아웃바운드가 인바운드보다 정책이 느슨하기 때문
-> 리버스 쉘을 테스트하기 위해 netcat 프로그램(다양한 네트워크 프로그램을 시뮬레이션 하기 위한 목적으로 만들어
진 프로그램 )을 많이 사용함

* 칼리 리눅스 : 보안 테스트를 위한 오픈 소스 리눅스 배포판인 백트랙의 후속작
* 메타스플로잇 : 칼리 리눅스에 내장되어 있는 모의 침투 테스트 도구
* 익스플로잇 : 소프트웨어나 하드웨어의 버그 또는 취약점을 이용하여 공격자가 의도한 동작이나 명령을 실행하도록 하
  는 코드 또는 그런 행위
* 쉘코드: 익스플로잇에 성공하면 프로그램 실행 제어권을 획득하게 되고 공격자가 의도한 명령을 담고 있는 코드를 실행
  하는데 이를 쉘 코드라고 함
* 쉘 : 원하는 명령을 해석해서 처리해주는 명령어 해석기
* CVE : 공개적으로 알려진 소프트웨어의 보안취약점을 표준화한 식별자 목록 -> 공통의 유일한 명칭으로 취약점을 식
  별하기 위해 만들어짐
* CWE : 미국 비영리 회사인 MITRE 사가 중심이 되어 소프트웨어에서 공통적으로 발생하는 약점을 체계적으로 분류한
  목록을 제공하기 위해 만들어짐 -> CWE/SANS TOP 25 는 미국 MITRE 및 SANS 연구소에서 발표하는 소프트웨어
  개발자가 가장 범하기 쉽고 위험한 25가지 소프트웨어 취약점 목록을 유형별로 분류해놓은 것

cf. 보안 약점 : 공격에 잠재적으로 활용될 수 있는 소프트웨어의 결함, 오류
cf. 보안 취약점 : 해킹 등 실제 보안 사고에 이용되는 소프트웨어의 보안 약점이나 시스템상의 보안 허점

* DBD (Drive By Download ) : 다운로드를 유도하는 공격 -> 공격자는 홈페이지를 해킹한 후 사용자 PC의 취약점을
  익스플로잇하는 악성 스크립트와 악성코드를 은닉시키고 취약한 PC 환경의 사용자가 홈페이지에 접속할 경우 자신의
  의도와는 무관하게 악성코드가 다운로드되어 설치되는 공격 기법 -> DBD 공격은 일반적으로 난독화된 악성 스크립트
  와 다수의 경유지/중계지를 거쳐 최종 유포지로 접속하여 악성코드를 다운로드하도록 유도함 -> 난독화로 인해 보안
  장비에 탐지 되기 힘들고 다수의 중계지를 거쳐 유포지에 대한 추적을 어렵게 함

-> 홈페이지를 통한 악성 코드의 유포 목적은 파밍형 악성코드를 통해 금융정보, 개인정보를 탈취하는 형태와 특정 타겟
을 대상으로 악성코드를 유포하는 워터링홀 방식의 APT 공격이 주된 목적으로 나타나고 있음
-> 방문페이지 : 웹사이트를 방문했을 때 현재 브라우저가 보여주는 페이지
-> 경유페이지 : 악성 스크립트가 삽입되어 있는 방문페이지
-> 중계페이지 : 경유 페이지에서 유포 페이지로 연결해주는 징검다리 역할을 하는 페이지
-> 유포페이지 : 실제 익스플로잇 및 악성코드가 저장되어 있고 이를 유포하는 페이지

* 홈페이지 유포 악성코드 탐지 기술 : 정적 분석기술 -> 점검대상 홈페이지 웹 프로그램 소스코드를 분석하여 알려진 악성
  코드 경유지,중계지,유포지 또는 악성 스크립트 문자열을 포함하는 악성스크립트가 있는지를 탐지하는 방식 -> 알려진
  탐지 패턴으로 점검하기 때문에 점검 속도는 빠르지만 알려지지 않은 신종 악성코드에 대해서는 탐지하지 못하는 한계
  가 있음 / 동적 분석기술 -> 가상의 이용자 PC환경을 구성하고 다양한 취약점에 노출시킨 후 점검 대상 홈페이지에 접
  속하여 비정상적인 레지스트리 변경, 악성코드 다운로드, C&C 서버 접속 등의 악성행위를 분석하는 기법 -> 이용자 환
  경에대한 행위분석이라고도하며 탐지 패턴을 이용하지 않기 때문에 신종 악성코드 탐지가 가능하나 상대적으로 분석 속
  도가 느린 단점이 있음

* 난독화 : 프로그램 코드를 읽기 어렵게 만드는 기술로 난독화의 대상에 따라 소스코드 난독화( 프로그램 소스코드를 알
  아보기 힘들게 만드는 기술 )와 바이너리 난독화(컴파일 후 생성된 바이너리를 역공학을 통해 분석하기 어렵게 변조함 )
  로 구분 됨 -> 패턴/시그니처 보안장비의 탐지를 우회할 목적으로 사용 함


* 디페이스 공격 : 웹 사이트의 화면을 해커가 원하는 화면으로 바꾸고 해킹 성공을 알리는 공격으로 시각적으로 자신의 존
  재감을 알리고 정치적인 메시지나 자신의 실력을 과시하는 목적으로 사용 함

* 악성코드 이해하기 : 악성 행위를 위해 개발된 컴퓨터에서 동작하는 모든 소프트웨어 = Malware

- 드롭퍼 : 다운로드와 유사하게 새로운 악성코드를 생성하지만 다운로더가 외부에서 악성코더를 다운 받는 것에 비해 드
  롭퍼의 경우 자신이 내부에 포함되어 있는 데이터를 이용하여 악성코드를 생성한다는 차이점이 있음 ( 인젝터 : 드롭퍼

  는 형태의 악성 코드 )
- 키로거 : 컴퓨터가 입력하는 정보를 기록하는 것으로 그 중에서도 주로 키보드를 통한 메시지를 중간에 가로채서 기록
  하는 형태
- 백도어 : 감염된 사용자 PC에 특정 포트를 열어두어 정상적인 인증과정 없이 원격접속을 통해 직접 조작하는 형태로 사
  용자 몰래 특정 파일을 삭제하거나 파일이나 정보를 빼가는 등의 행위를 함
- 트로이목마 : 정상적인 소프트웨어의 형태를 띠지만 악의적인 행위를 포함하고 있는 악성코드를 말한다. 주로 일반 프
  로그램으로 가장하여 사용자가 다운로드해 실행하기를 기다리는 형태로 전파됨
- 랜섬웨어 : 사용자 파일을 암호화 시켜 일정 시간 안에 일정 금액을 지불하면 암호를 풀어주는 방식으로 사용자에게 금
  전적인 요구를 하는 악성코드를 말함
- 애드웨어 : 상업용 광고 목적으로 만들어진 악성코드로 웹 브라우저 시작 페이지 변경, 팝업 윈도우 생성 등의 동작을 함
- 스파이웨어 : 사용자 PC에 사전 동의 없이 설치되어 컴퓨터 정보와 개인정보를 수집하는 악성코드

- 루트킷 : 악의적인 행동을 하는 프로그램을 숨기기 위한 목적으로 시작된 악성코드로 현재는 프로세스나 파일 등의 흔

  격을 유발시킬 수 있는 취약점 -> NTP 서버의 monlist 명령어(최근 접속한 최대 600개의 접속 호스트를 응답받을 수
  있음 ) 사용
=> 대응책 : 취약한 NTP 데몬 서버 버전을 업그레이드 함, 버전 업그레이드가 어려운 경우 monlist 기능을 해제함 (
ntp.conf 파일에 disable monitor 추가, ntpdc -n -c monlist [ip정보] 명령어를 통해 적절하게 조치되었는지 확인
할 수있음, 보안 장비등을 통해서도 차단할 수있다.

* ntpdate - q “NTP 서버” -> NTP 서버의 날짜/시간 정보 확인
* ntpdate “NTP 서버” -> NTP 서버의 날짜/시간 정보를 현재 서버에 설정함

* SSDP 분산 서비스 거부 취약점 : SSDP -> 네트워크 상의 서비스나 정보를 검색하는 프로토콜로 프린터, 스캐너, IP
  카메라 등 다양한 IoT 기기의 네트워크 탐색 용x도로 사용되며 UDP 1900 번을 사용한다.
-> SSDP 프로토콜을 악용하여 대규모 증폭,반사 형태의 디도스 공격을 실행한 사례가 보고 되고 있음
