[패스워드 크래킹]
• 사전 공격/사전 대입 공격 ( Dictionary Attack )
  -> 패스워드로 자주 사용되는 사전에 있는 단어, 키보드 자판의 일
련순을 미리 사전파일로 생성 후 하나씩 대입하여 패스워드 일치 여부
를 확인함

• 무차별 공격/무작위 대입 공격 ( Brute Force Attack )
-> 패스워드에 사용될 수 있는 문자열 범위를 지정하고 범위 내에서
생성가능한 모든패스워드를 생성하여 이를 하나씩 대입, 패스워드 일
치 여부를 확인하는 패스워드 크래킹 기법 ( 일반적으로 사전 대입 공
격 이후 무차별 공격을 수행 )

•   혼합 공격 ( Hybrid Attack )
->    사전 공격 + 무차별 공격/무작위 대입 공격

• 레인보우 테이블 공격
-> 해시테이블과 함수의 반복 수행을 통해 일치하는 해시값을 통해서
패스워드를 찾아내는 방식
-> 레인보우 테이블 : 일정 수의 패스워드와 해시로 이루어진 체인을
무수히 만들어 놓은 테이블

* PCB(Process Control Block) : 프로세스 제어 블록
 - 프로세스 상태 : ready, running, blocked
 - 프로세스 번호 : 프로세스 색별 번호

* 프로세스 특징
 - 시스템 부팅을 담당하는 boot 프로세스를 제외한 모든 프로세스
는 부모 프로세스를 가진다.
 - 자식 프로세스가 살아 있는 상태에서 부모 프로세스가 종료하게 되
면 자식 프로세스는 고아 프로세스가 됨. 이 경우 대리모 프로세스 (
init proccess -> pid :1 ) 가 부모 역할을 한다

 - 프로세스 종료시에는 자신의 종료 상태 정보( pid, exit
code, cpu time) 를 부모 프로세스에 반환해야 정상적으로 소멸할
수 있다.

- 좀비 프로세스 : 프로세스 수행을 종료 했지만 부모 프로세스가
   종료 상태정보를 확인하지 않아 소멸하지 않고 남아 있는 상태의
   프로세스를 좀비 프로세스라고 한다
 -> 좀비 프로세스가 많아지면 더 이상 프로세스 생성이 불가능 하
다
 -> Z, defunct 로 표기
 -> kill 시그널을 보내도 소멸되지 않음. 시스템 리부팅을 통해 커
널 정보를 초기화 해야함

* 시그널 종류
 1) SIGINT (2) : ctrl + c 를 사용해   포그라운드 프로세스
그룹 내 모든 프로세스에 전달함. 디폴트로 종료

*   PAM(장착형 인증 모듈) - Pluggable Authentication

- 리눅스 시스템 내에서 사용되는 각종 애플리케이션 인증을 위해 제공

Type Control   module-pam module-arguments

-> type 은 pam 모듈의 종류를 의미함       ( account / auth /
password / session) 으로 구분 됨

-> control 은 각 모듈 실행 후 성공 또는 실패에 따른 pam 라이
브러리의 행동을 결정함 ( requisite -> 인증 실패시 즉시 인증

실행하지만 최종 결과는 인증 실패 / sufficient -> 이전에 있
는 요청들이 실패하더라도 여기서 성공하면 pam 인증 성공 )

-> module-Path 는 pam에서 사용할 실제 모듈 파일이 위치한
경로ㅡㄹ 의미함 ( lib/security에 *.so 형식으로 존재)
-> module-arguments 는 모듈에 전달되는 인수를 의미함

*  주요 파일 소유자 및 권한 설정
- /etc/passwd : 사용자 정보를 담고 있는 파일 . root 소유의
  644 이하의 권한을 설정
- /etc/shadow : 사용자의 암호화된 패스워드를 담고 있는 파
  일. root 소유의 400 이하의 권한을 설정
- /etc/hosts : IP와 호스트의 이름을 매핑하는데 사용되는 파일
  로 외부의 불법적인 접근에 의한 조작시 악의적인 시스템에 접근할
  가능성이 있으므로 root 소유의 600 이하의 권한을 설정 ( 금융
  위원회 기준으로는 644 )
- /etc/(x)inetd.conf 파일 : inetd 데몬에 의한 설정 파일로
  외부의 불법적인 접근에 의해 악의적인 프로그램을 등록하고 root
  권한으로 실행할 가능성이 있으므로 root 소유의 600 이하의 권한
  을 설정한다
- /etc/syslog.conf : syslogd 데몬에 대한 설정파일로 주요
  로그파일에 대한 설정 정보를 가지고 있음. 외부의 침입에 의해 변
  경되는 경우 로그가 정상적으로 기록되지 않아 침입자의 흔적이나 시
  스템 오류사항을 정확히 분석할 수 없으므로 root 소유의 644 이
  하의 권한을 설정함
- /etc/services : 서비스 관리 ( 서비스별포트와 프로토콜) 정
  보를 담고 있는 파일로 불법적인 접근에 의해 변경되는 경우 정상적
  인 서비스를 제한하거나 허용되지 않는 서비스를 등록하는 등의 위험
  이 있으므로 root 644 권한을 설정
