> 출제기준 2과목 1-2-3 "유무선 네트워크 서비스의 개요와 종류별 특징" 과 1-2-4 "네트워크 도구(ping, arp, rarp, traceroute, netstat, tcpdump 등) 의 이해와 활용" 을 다룬다.

## 0. 들어가며 — 도구는 "어느 계층의 어느 헤더를 읽는가"

네트워크 진단 명령은 단순 암기가 아니라 "이 도구가 어느 계층(2/3/4) 의 어느 프로토콜을 사용하며, 어느 헤더를 읽거나 조작하는가" 의 관점으로 이해한다. 무선 네트워크는 1·2 계층이 유선과 다르므로 보안 메커니즘(WEP·WPA·WPA2·WPA3) 과 인증(PSK·EAP·802.1X) 가 별도로 정의된다.

| 블루투스 / RFID | 1-2-3 | 1830 ~ 1894 | (없음 — 외부 보강) |

## 1. 네트워크 관리·진단 도구

### 1-1. ping — ICMP Echo Request/Reply 기반 도달성 진단

종단 노드 간에 네트워크 상태를 관리하기 위한 명령어. ICMP Echo Request(Type 8) 와 Echo Reply(Type 0) 를 이용하여 대상 시스템에 대한 접근성과 해당 구간의 네트워크 속도·품질을 검사한다. (ICMP 메시지 타입은 [02 단원 §3-3](02.network-protocols-and-addressing.md) 참조)

| 플랫폼 | 명령 예시 | 옵션 의미 |
|---|---|---|
| **Windows** | `ping -n 5 -l 128 www.google.com` | `-n` 횟수, `-l` 패킷 크기 |
| **Linux** | `ping -c 5 -s 128 www.google.com` | `-c` 횟수, `-s` 패킷 크기 |

**동작 절차**
1. 로컬 호스트가 ICMP Echo Request 타입을 지정하여 대상 시스템으로 ICMP 패킷 전송
2. 대상 시스템으로부터 Echo Reply 패킷을 수신하여 접근성 확인

### 1-2. traceroute (Linux) / tracert (Windows) — TTL 기반 경로 추적

종단 노드 사이의 여러 중계 노드 각 구간의 네트워크 상태를 측정하기 위한 명령어. **TTL 필드를 1 부터 점차 증가시키면서** 패킷을 보내고 각 라우터의 ICMP Time Exceeded 응답으로 경로를 역추적한다.

| 도구 | 사용 패킷 | 최종 목적지 응답 |
|---|---|---|
| **traceroute** (Linux) | TTL=1 부터 증가시킨 **UDP** 패킷 | 닫힌 UDP 포트 → ICMP `Destination Unreachable` 반환 |
| **tracert** (Windows) | TTL=1 부터 증가시킨 **ICMP Echo Request** 패킷 | ICMP `Echo Reply` 반환 |

**동작 절차**
1. 라우터가 패킷 수신 시 TTL 을 1 감소
2. TTL = 0 이 되면 해당 패킷을 폐기
3. ICMP `Time Exceeded` 패킷을 최초 출발지로 반환
4. 최초 출발지가 TTL 값을 계속 증가시켜 다음 라우터까지 도달
5. 최종 목적지에 도달하면 도달 알림 패킷이 반환되어 네트워크 상태 판단 종료

> 보안상의 이유로 라우터에서 ICMP 응답을 차단하는 경우가 많아 traceroute 결과에 `*` 표시가 자주 나타난다.

### 1-3. netstat — 시스템 네트워크 상태정보

시스템의 네트워크 관련 다양한 상태정보를 관리할 수 있는 명령어. 소켓 상태정보·시스템 라우팅 정보·인터페이스 정보·프로토콜별 통계정보를 확인할 수 있다.

| 옵션 | 동작 |
|---|---|
| (없음) | 모든 연결된 소켓 상태정보 |
| `-a` | 모든 소켓 상태정보 |
| `-i` | 네트워크 인터페이스 정보 |
| `-r` | 시스템 라우팅 테이블 정보 |
| `-s` | 프로토콜별 통계정보 |
| `-t` | TCP 소켓 정보 (`-u` 는 UDP) |
| `-n` | 네트워크 주소를 숫자 형식으로 출력 |
| `-p` | 소켓의 프로세스명·PID 정보 |

**netstat 소켓 상태 출력 필드 (`netstat -t` 등)**

| 필드 | 의미 |
|---|---|
| Proto | 사용한 프로토콜의 종류 |
| Recv-Q | 원격 호스트로부터 수신 버퍼에 저장된 데이터 크기 |
| Send-Q | 원격 호스트로 전송하기 위해 송신 버퍼에 저장한 데이터 크기 |
| Local Address | 로컬 호스트의 소켓 주소 |
| Foreign Address | 원격 호스트의 소켓 주소 |
| State | 소켓 상태 |

**netstat 라우팅 테이블 출력 필드 (`netstat -r` / `netstat -rn`)**

| 필드 | 의미 |
|---|---|
| Destination | 목적지 호스트 또는 네트워크 |
| Gateway | 1차 경유지 주소 |
| Genmask | 목적지 식별을 위한 서브넷 마스크 |
| Flags | 해당 경로의 상태 플래그 |
| **Metric** | 라우팅 경로의 효율성(우선순위). **클수록 비용↑ 우선순위↓** |
| Iface | 사용할 네트워크 인터페이스 |

> Metric 등 라우팅 테이블 검색 우선순위는 [02 단원 §2-6](02.network-protocols-and-addressing.md) 에서 다루었다.

### 1-4. ifconfig (Linux/Unix) / ipconfig (Windows) — 인터페이스 설정

| 도구 | 동작 |
|---|---|
| **ifconfig** | 유닉스/리눅스에서 네트워크 인터페이스 설정정보 조회·변경. `ifconfig [인터페이스] promisc` 로 무차별 모드 전환, `-promisc` 로 해제 |
| **ipconfig** | 윈도우에서 네트워크 인터페이스 설정정보 조회. `ipconfig /all` 로 상세 정보, `ipconfig /displaydns` 로 DNS 캐시 조회, `ipconfig /flushdns` 로 DNS 캐시 삭제 |

> *원본 표기 보정*: 통합자료 L1822 의 `ifconfig /all` 은 윈도우 `ipconfig` 의 옵션이므로 `ipconfig /all` 로 보정했다.

### 1-5. tcpdump — 패킷 스니퍼·분석

네트워크 인터페이스를 거치는 패킷의 내용을 출력해주는 프로그램. 자신의 컴퓨터로 들어오는 모든 패킷의 내용을 도청하는 스니핑 도구의 일종이며, 공격자 추적 및 공격 유형 분석을 위한 패킷 분석에 활용된다.

```
tcpdump -i eth0           # eth0 인터페이스의 트래픽 모니터링
tcpdump -i eth0 host 1.2.3.4
tcpdump -i eth0 port 80
```

스니핑 도구로서의 동작은 [04 단원 §스니핑](04.scanning-sniffing-spoofing.md) 에서 자세히 다룬다.

### 1-6. arp / nslookup / dig / whois (보강)

출제기준 1-2-4 에 명시된 도구이지만 통합자료의 본 단원 위치에는 표가 없는 영역. ARP 명령어는 [02 단원 §1-6](02.network-protocols-and-addressing.md) 에서 다루었다.

| 도구 | 용도 |
|---|---|
| `arp -a` / `-d` / `-s [IP] [MAC]` | ARP Cache 조회 / 삭제 / 정적 설정 |
| `nslookup` / `dig` | DNS 레코드 조회. dig 는 nslookup 의 후속 도구로 유닉스·리눅스에 기본 설치 |
| `whois [도메인 또는 IP]` | 도메인 등록정보·소유정보·네임서버·IP 대역 관리 대행자 정보 조회 |

DNS 레코드 타입(A·AAAA·MX·NS·SOA·TXT·PTR·AXFR·IXFR) 과 DNS 공격(스푸핑·캐시 포이즈닝)·DNS 도구의 자세한 활용은 출제기준 3과목(어플리케이션 보안) §DNS 영역에서 다룬다.

### 1-7. 도구 정리표

| 도구 | 사용 계층/프로토콜 | 핵심 용도 |
|---|---|---|
| ping | 3 (ICMP) | 도달성 진단 (Echo Request/Reply) |
| traceroute / tracert | 3 (UDP+ICMP / ICMP) | TTL 기반 경로 추적 |
| netstat | 4 / 3 | 소켓·라우팅·통계 |
| ifconfig / ipconfig | 2/3 | 인터페이스 설정 |
| tcpdump | 2 ~ 7 (전 계층 패킷) | 스니핑·분석 |
| arp | 2/3 (ARP) | ARP Cache 조작 |
| nslookup / dig | 7 (DNS) | DNS 레코드 조회 |
| whois | 7 (TCP/43) | 도메인·IP 등록정보 조회 |

### 시험 포인트

- ping 옵션 — Windows `-n -l` / Linux `-c -s`
- traceroute(UDP) vs tracert(ICMP) 의 패킷 종류 차이 — 둘 다 TTL 1부터 증가
- traceroute 의 `*` 출력 = 라우터 ICMP 차단으로 인한 응답 없음
- netstat 옵션 — `-a`(모든 소켓) `-r`(라우팅) `-s`(통계) `-n`(숫자) `-p`(PID)
- Metric 이 클수록 우선순위 낮음
- ipconfig `/displaydns` `/flushdns` 의 용도
- ifconfig `[인터페이스] promisc` 로 NIC 무차별 모드 전환

## 2. 블루투스 (Bluetooth)

### 2-1. 정의·링크 형태

서로 짧은 거리에 있을 때 전화기·노트북·컴퓨터 등 서로 다른 기능을 가진 장치를 연결하기 위해 설계된 무선 LAN 기술. **마스터(Master) - 슬레이브(Slave) 방식** 으로 블루투스 링크를 설정한다.[^ieee80215]

| 링크 형태 | 설명 |
|---|---|
| **피코넷(Piconet)** | 슬레이브를 연결하여 네트워크를 구성한 형태. 한 대의 마스터는 슬레이브를 **7대까지** 구성할 수 있음 |
| **스캐터넷(Scatternet)** | 계층적이고 규모가 큰 네트워크 구성. **피코넷이 여러 개 모인 형태** |

### 2-2. 블루투스 4대 공격

| 공격 | 설명 |
|---|---|
| **블루프린팅(BluePrinting)** | 블루투스 공격 장치의 검색 활동을 의미 (장치 식별 단계) |
| **블루스나핑(BlueSnarfing)** | 블루투스의 취약점을 이용하여 장비의 임의 파일에 접근하는 공격 |
| **블루버깅(BlueBugging)** | 블루투스 장비 간의 취약한 연결 관리를 악용한 공격 — 장비 제어 |
| **블루재킹(BlueJacking)** | 블루투스를 이용해 스팸처럼 명함을 익명으로 퍼트림 |

### 시험 포인트

- 피코넷 — 마스터 1대 + 슬레이브 **7대** 까지 / 스캐터넷 — 피코넷 여러 개의 결합
- 블루투스 4대 공격의 차이 — Printing(검색) / Snarfing(파일 탈취) / Bugging(장비 제어) / Jacking(스팸)

## 3. RFID (Radio-Frequency Identification)

### 3-1. 정의·동작 방식

무선 주파수를 이용해 ID 를 식별하는 방식. 태그가 송신하는 전파의 에너지원을 얻는 동작 방법에 따라 두 가지로 구분된다.

| 분류 | 에너지원 |
|---|---|
| **수동형(Passive)** | 리더기로부터 받은 전파 에너지를 사용 (자체 전원 없음) |
| **능동형(Active)** | 자체 배터리로 전파 송신 |

### 3-2. RFID 공격

| 공격 | 설명 |
|---|---|
| **도청(Eavesdropping)** | 공격자가 리더기를 갖고 태그를 스캐닝하는 적극적 공격 + 리더기와 태그 간 통신을 도청하는 수동적 공격 |
| **트래픽 분석** | 특정 태그에서 리더기와 태그 간의 트래픽 양을 분석하여 정보 추정 |
| **위조(Spoofing)** | 데이터 항목을 지우거나 대신할 수 있는 방법을 통해 잘못된 데이터가 교환되도록 함 |
| **서비스 거부(DoS)** | 수많은 질의를 리더기·태그에 보내 응답 자원을 소진시킴 |

### 3-3. RFID 보안 메커니즘

| 보안 기법 | 설명 |
|---|---|
| **Kill 명령어** | 태그에 Kill 명령을 전송하면 태그가 영구적으로 비활성화. 태그 내용은 영원히 재활용 불가 |
| **Sleep / Wake 명령어** | 태그의 기능을 잠시 동안 정지했다가 안전한 장소에서 다시 동작하도록 하는 명령 |
| **블로커 태그(Blocker Tag) 기법** | 블로커 태그라고 불리는 전용 IC 태그를 소비자가 소지함으로써 가까이 있는 IC 태그의 ID 를 읽을 수 없게 만듦 |
| **Faraday Cage** | 금속성 박막으로 태그를 막아 무선 신호의 전달을 방해 |
| **Jamming** | 강한 방해 신호를 보내어 불법적인 리더기의 접근을 막는 방식 |
| **해시 락(Hash Lock) 기법** | One-way 해시 함수를 기반으로 하는 접근제어 메커니즘 |
| **XOR 기반 원타임 패드(OTP) 기법** | XOR 과 같은 간단한 연산으로 태그 데이터를 인코딩 |

### 시험 포인트

- 수동형(에너지 외부 의존) vs 능동형(자체 배터리) RFID
- RFID 4대 공격 — 도청 / 트래픽 분석 / 위조 / 서비스 거부
- RFID 7대 보안 메커니즘 — Kill / Sleep·Wake / 블로커 태그 / Faraday Cage / Jamming / 해시 락 / XOR OTP
- Faraday Cage 와 Jamming 의 차이 — Cage 는 차폐(passive), Jamming 은 방해 신호 송신(active)

## 4. 무선랜 (IEEE 802.11) 일반

### 4-1. 정의·핵심 용어

무선 신호 전달 방식을 이용하여 두 대 이상의 장치를 연결하는 기술. 근거리 지역에서 이동하면서도 지속적으로 네트워크에 접근할 수 있다.[^ieee80211]

| 용어 | 의미 |
|---|---|
| **SSID** (Service Set Identifier) | 무선랜을 식별하기 위한 32byte 이름. 서로 다른 무선랜 식별 |
| **BSS** (Basic Service Set) | 가장 기본적인 망 구성 단위 (특정 무선 AP 와 그 AP 에 연결된 클라이언트 집합) |
| **BSSID** (Basic Service Set Identifier) | BSS 망을 식별하기 위한 ID — 일반적으로 무선 AP 의 48bit MAC 주소 사용 |
| **ESS** (Extended Service Set) | 여러 BSS 에 의해 구성된 네트워크. 무선 AP 를 모두 엮는 논리적 집합 |
| **ESSID** (Extended Service Set Identifier) | ESS 네트워크를 식별하기 위한 ID. 복수의 무선 AP 를 설치한 환경에서 사용 |
| **DS** (Distribution System) | 무선 AP 가 연결된 유선 네트워크 (무선 AP 이후에 반드시 유선을 거쳐 통신) |
| **Beacon 프레임** | 무선 AP 가 자신이 관리하는 무선랜의 존재를 정기적으로 알리는 브로드캐스트 프레임 |
| **무선 브리지** | 2개 이상의 무선랜을 연결하는 장비. 물리적으로 떨어진 2개의 무선랜에 각각 위치하여 동작 |

### 4-2. 무선랜 환경 분류

| 환경 | 운영 주체 |
|---|---|
| **상용 무선랜** | 이동통신사가 고객 서비스용으로 구축 |
| **공중 무선랜** | 호텔·카페 등 서비스 업종에서 불특정 다수의 고객 편의를 위해 구축 |
| **사설 무선랜** | 일반 사용자가 무선 공유기를 통해서 구축 |
| **기업 무선랜** | 기업이 내부 업무용으로 구축 |

### 4-3. 무선랜 특성 (전파 특성)

| 특성 | 의미 |
|---|---|
| **감쇠(Attenuation)** | 신호는 모든 방향으로 퍼지기 때문에 전자기 신호의 강도가 급격하게 줄어듦 |
| **간섭(Interference)** | 같은 주파수 대역을 사용하는 다른 송신자의 신호를 받음 |
| **다중경로 전달** | 전기적 파동이 장애물에 반사되어 같은 송신자에게서 하나 이상의 신호 수신 |
| **오류** | 오류와 오류 감지가 유선보다 무선에서 더욱 심각한 문제 |

### 4-4. IEEE 802.11 표준 변천

| 표준 | 보안·암호화 도입 시점 |
|---|---|
| **IEEE 802.11** (1997) | 최초의 무선랜 표준. 인증절차 없이 접속, 평문 전송 |
| **IEEE 802.11b** (1999) | 무선전송 데이터 암호화와 인증을 제공하기 위한 **WEP** 규정 |
| **IEEE 802.11i** (2004) | **WPA, WPA2** 를 규정하여 보다 강화된 암호화·인증 제공 |

> 외부 보강 — 무선랜 PHY 표준은 위와 별개로 a/b/g/n/ac/ax(Wi-Fi 6)/be(Wi-Fi 7) 가 추가되어 왔다. 보안 측면에서는 WPA3(2018, IEEE 802.11i 갱신) 가 SAE(Simultaneous Authentication of Equals, 일명 Dragonfly) 키 교환을 도입했다.[^wpa3]

### 4-5. 무선랜 보안 취약점

| 분류 | 취약점 |
|---|---|
| **물리적** | 외부 노출된 무선 AP 의 도난·파손 / 리셋버튼을 통한 장비 초기화 / 전원 케이블 분리로 인한 장애 / LAN 케이블 절체 |
| **기술적** | **도청** (브로드캐스트 특성으로 도청·스니핑에 취약) / 서비스 거부 (대량의 무선 패킷 전송으로 무선랜 무력화) / **불법 AP** (불법적인 AP 설치로 접속 유도) |
| **관리적** | 장비 관리 미흡 / 사용자의 보안 의식 부족 / 전파 출력 관리 미흡 (기관 외부로 전파 유출) |

### 시험 포인트

- SSID = 무선랜 이름 / BSSID = AP 의 MAC 주소 / ESSID = 다수 BSS 묶음 / DS = AP 가 연결된 유선망
- Beacon 프레임 — AP 가 자신을 알리는 브로드캐스트
- IEEE 802.11(평문) → 802.11b(WEP) → 802.11i(WPA/WPA2) 의 변천
- 무선랜 4대 특성 — 감쇠 / 간섭 / 다중경로 / 오류
- 무선랜 보안 취약점 3분류 — 물리적 / 기술적 / 관리적

## 5. 무선전송 데이터 암호 기술

### 5-1. WEP (Wired Equivalent Privacy)

초기 무선랜 보안기술로 IEEE 802.11b 표준에서부터 적용. 데이터 암호화와 사용자 인증 두 가지 기능을 모두 제공.

| 구성 요소 | 설명 |
|---|---|
| **WEP 암호키** | 고정된 공유키 + 무작위로 생성한 24bit **초기벡터(IV)** 를 조합한 키 |
| **RC4 알고리즘** | WEP 암호키의 키 스트림을 생성하기 위한 알고리즘 |
| **공유키 인증** | 서로 같은 공유키를 갖는 사용자들이 인증하고 통신하는 방식 |
| **CRC-32 알고리즘** | 평문의 무결성을 보장하기 위한 알고리즘 |

#### 5-1-1. WEP 암호화 문제점

- 짧은 초기벡터(24bit) → IV 값이 재사용될 가능성
- 불완전한 RC4 알고리즘 → 암호키 노출 문제
- 짧은 길이의 암호키 → 공격 가능성
- 암호키 노출 시 무선전송 데이터의 노출 위험

#### 5-1-2. WEP 인증 문제점

- **단방향 인증** : 악의적으로 설치된 불법 AP 인지 알 수 없어 피해 발생
- **고정된 공유키** : 동일한 WEP 공유키를 사용하기 때문에 외부 유출 시 문제

### 5-2. WPA (Wi-Fi Protected Access)

WEP 의 취약성을 보완하기 위한 기술로 IEEE 802.11i 표준에서부터 적용. WPA 는 암호 알고리즘으로 **RC4-TKIP** 를 사용한다.

| 구성 요소 | 설명 |
|---|---|
| **길어진 초기벡터** | WEP 의 24bit → **48bit** 로 확장 |
| **TKIP** (Temporal Key Integrity Protocol) | 공유 비밀키를 동적으로 생성하여 무선 구간 패킷을 암호화 |
| **키 믹싱(Key Mixing)** | 별도의 키 생성 과정을 거쳐 각 패킷마다 별도의 암호키를 적용하는 함수 |
| **MIC** (Message Integrity Code) | CRC-32 알고리즘보다 안전한 메시지 무결성 검사 알고리즘 |

#### 5-2-1. WPA 암호화 문제점

- 불완전한 RC4 알고리즘 — 여전히 취약한 RC4 사용
- 키 관리 — 키 관리 방법을 제공하지 않으며 키 크랙 공격에 대한 취약성
- 소프트웨어 — 소프트웨어적 처리에 따른 암·복호화 시간 지연 등 성능 문제

### 5-3. WPA2 — AES-CCMP

WPA 의 후속으로 IEEE 802.11i 의 완전 구현. 암호 알고리즘으로 **AES-CCMP** (AES + Counter Mode with CBC-MAC) 를 사용하여 RC4 의존성을 제거.[^ieee80211i]

| 구성 요소 | 설명 |
|---|---|
| **AES-CCMP** | AES 기반의 블록 암호화 + 메시지 인증 코드 결합 모드 — 검증된 보안성 |
| **TKIP 와 유사한 키 관리** | 패킷 번호를 이용해 재전송 공격 방지 |

### 5-4. WPA3 (외부 보강)

2018년 Wi-Fi Alliance 가 발표한 후속 표준. **SAE(Simultaneous Authentication of Equals, Dragonfly)** 키 교환을 도입하여 PSK 의 사전 공격 취약성을 완화.[^wpa3]

| 강화 영역 | 설명 |
|---|---|
| **SAE 키 교환** | 4-way Handshake 의 사전 공격 취약점 완화 |
| **개별 데이터 암호화** | 공중 네트워크에서도 클라이언트별 트래픽 암호화 |
| **192bit 보안 모드** | 정부·금융·국방용 강화 모드 (CNSA 호환) |

### 시험 포인트

- WEP — 24bit IV + RC4 + CRC-32 + 공유키 단방향 인증 → 다수의 취약점
- WPA — TKIP(RC4 기반, 48bit IV, 키 믹싱) + MIC → 여전히 RC4
- WPA2 — **AES-CCMP** 사용 → RC4 의존 제거 (정답 빈도 매우 높음)
- WPA3 — SAE 키 교환으로 PSK 사전 공격 완화

## 6. 무선랜 접속 인증 기술

### 6-1. WPA / WPA2 - 개인 인증 방식 (PSK)

PSK 인증 방식은 **인증 서버가 설치되지 않은 소규모 망**에서 사용하는 방식. 무선 AP 와 단말기가 자신과 동일한 공유키를 가지고 있는지 4-way Handshake 과정을 통해 확인하여 인증을 수행한다.

#### 6-1-1. PSK 사전 공격 절차

PSK 의 핵심 취약점 — 추측 가능한 공유키일 경우 사전 공격(Dictionary Attack) 으로 손쉽게 크랙 가능.

1. 공격자가 4-way Handshake 과정을 스니핑
2. 사전 파일에 있는 단어를 이용하여 PSK 를 추측·생성
3. 생성한 PSK 와 스니핑한 무선 패킷의 파라미터를 이용하여 PTK(Pairwise Transient Key) 생성
4. 생성한 PTK 를 이용하여 MIC 값을 생성한 후 스니핑한 MIC 값과 비교
5. 일치할 경우 추측한 패스워드를 접속·인증 패스워드로 사용
6. 일치하지 않을 경우 다른 단어를 이용하여 반복

### 6-2. WPA - 기업 인증 방식 (IEEE 802.1x / EAP)

802.1X 표준에서 제공하는 EAP 인증 기능을 이용하여 사용자 인증 방법으로 사용.[^ieee8021x]

| 구성 요소 | 설명 |
|---|---|
| **EAP** (Extensible Authentication Protocol) | 어떤 링크에도 접속이 가능한 단순한 캡슐화 프로토콜. EAP 자체는 실제 인증 프로토콜이 아니지만 다양한 종류의 인증 방법(EAP-MD5, EAP-TLS, EAP-TTLS, PEAP 등) 을 전송하는 역할 수행[^rfc3748] |
| **IEEE 802.1X** | 포트 기반 네트워크 접근 제어 표준. 인증되지 않은 장치는 네트워크 포트를 통과할 수 없도록 차단 |
| **RADIUS 서버** | 인증·인가·과금(AAA) 을 담당하는 인증 서버. EAP 메시지를 처리[^rfc2865] |

| EAP 방식 | 인증 방식 |
|---|---|
| **EAP-MD5** | MD5 챌린지 — 단방향, 보안 약함 |
| **EAP-TLS** | 클라이언트·서버 모두 인증서 사용 — 가장 강력한 상호 인증[^rfc5216] |
| **EAP-TTLS** | 서버는 인증서, 클라이언트는 ID/패스워드 (TLS 터널 내부에서 인증)[^rfc5281] |
| **PEAP** | 서버는 인증서, 클라이언트는 MS-CHAPv2 등 사용 (TLS 터널 내부)[^peap-draft] |

### 6-3. SSID 설정 및 MAC 주소 인증 (보조 메커니즘)

| 인증 기술 | 장점 | 단점 |
|---|---|---|
| **SSID 설정을 통한 접속 제한** | SSID 를 모르는 사용자는 무선랜에 접속할 수 없음 | 무선 데이터 분석 도구(예: airodump-ng) 를 이용해 손쉽게 SSID 알아낼 수 있음 |
| **MAC 주소 인증** | 무선랜 접속을 허용할 단말의 MAC 주소를 무선 AP 에 사전 등록 → 등록된 단말기만 접속 가능 | 인증된 단말의 MAC 주소로 위조하여 접속할 수 있음 |

### 6-4. 무선랜 보안 대책 정리

| 환경 | 권고 |
|---|---|
| **기업 네트워크** | 무선 단말 인증 — WPA2-기업의 RADIUS 서버를 활용한 EAP 기반 인증 / 전송데이터 암호화 — 검증된 WPA2 보안규격의 AES-CCMP 방식 |
| **개인·SOHO 사업자** | 사설망 인증 — WPA2-개인의 PSK 인증을 설정한 후 안전한 인증 패스워드 이용 / 전송데이터 암호화 — WPA2 AES-CCMP / 무선랜 사업자의 인증서버 통한 단말 인증 사용 |
| **무선 AP 물리·관리** | SSID 변경 및 브로드캐스트 금지 (초기 SSID 변경, 숨김 설정) / 무선 AP 의 기본 패스워드 변경 (추측 어려운 패스워드, 주기적 변경) / 무선 AP 의 물리적 접근 제한 (보호 케이스, 리셋 버튼 차단) |

### 시험 포인트

- PSK = 소규모, EAP/802.1X = 기업
- PSK 사전 공격 절차 — 4-way Handshake 스니핑 → PSK 추측 → PTK 생성 → MIC 비교
- EAP 4대 방식 — MD5(약함) / TLS(상호 인증서) / TTLS(서버 인증서 + 클라이언트 ID/PW) / PEAP(MS-CHAPv2)
- 802.1X = 포트 기반 접근 제어 / RADIUS = AAA 서버
- SSID 숨김·MAC 필터링은 보조 메커니즘으로 우회 가능

## 7. 무선 응용 프로토콜 — WAP / WTLS

### 7-1. WAP (Wireless Application Protocol)

모바일 사용자에게 무선전화기·기타 인터넷·웹을 포함하는 정보 서비스를 제공하는 무선 단말기를 위한 표준.[^wapforum]

- 종단-대-종단 보안으로 **WAP2** 가 있음
- 모든 기존 무선 네트워크 기술 및 IP·XML·HTML·HTTP 등 인터넷 기술과 가능한 한 호환되도록 설계

### 7-2. WAP 구성 요소

| 구성 요소 | 역할 |
|---|---|
| **WDP** (Wireless Datagram Protocol) | 상위 계층 WAP 프로토콜을 모바일 노드와 WAP 게이트웨이 사이에 적합하게 변형 |
| **WTP** (Wireless Transaction Protocol) | WAP 브라우저 같은 사용자 에이전트와 응용 서버 사이에 브라우징·전자상거래 트랜잭션 동작을 위한 요청·응답 전달 및 트랜잭션 관리 |
| **WSP** (Wireless Session Protocol) | 응용 프로그램에 두 가지 세션 서비스를 위한 인터페이스 지원 |
| **WML** (Wireless Markup Language) | 제한된 통신 용량·화면 크기·사용자 입력 기능을 가진 장비에서 콘텐츠와 양식을 표현 |
| **WAE** (Wireless Application Environment) | 모바일 전화·버퍼·PDA 같은 무선 장비의 응용 프레임워크를 규격화 |
| **WTLS** (Wireless Transport Layer Security) | 무선 응용 프로토콜에서 전송 계층 보안을 위한 TLS 기반 보안 프로토콜. SSL/TLS 와 같이 두 개의 계층으로 구성 |

### 시험 포인트

- WAP = 모바일 단말기용 표준, **WAP2** = 종단-대-종단 보안
- WAP 핵심 6 컴포넌트 — WDP / WTP / WSP / WML / WAE / **WTLS**
- WTLS = 무선 환경에서 TLS 와 동등한 역할

## 8. 무선랜 크래킹 도구

무선랜 접속·인증 패스워드 크래킹에 사용되는 도구. 주로 WEP/WPA/WPA2-PSK 인증 패스워드를 크래킹할 수 있다.

### 8-1. aircrack-ng 도구 모음

| 도구 | 역할 |
|---|---|
| **airmon-ng** | 무선랜 인터페이스를 모니터 모드로 전환시키는 프로그램 |
| **airodump-ng** | 무선 패킷을 수집하여 저장하는 프로그램 |
| **aireplay-ng** | 무선 네트워크에 패킷을 삽입해주는 프로그램 |
| **airdecap-ng** | 암호화된 패킷 덤프를 복호화해주는 프로그램 |
| **aircrack-ng** | 패스워드를 크랙하여 출력해주는 프로그램 |

> *원본 표기 보정*: 통합자료 L2598·L2602·L2600 의 `airdump-ng`, `airecap-ng`, `airplay-ng` 는 OCR 결함 또는 약자 표기로, 표준 도구 이름인 `airodump-ng`, `airdecap-ng`, `aireplay-ng` 로 보정했다.[^aircrack]

### 8-2. iwconfig — 무선랜 인터페이스 모드

무선랜 인터페이스 정보를 확인하거나 환경 설정을 위해 사용하는 명령어.

| 모드 | 동작 |
|---|---|
| **관리(Managed) 모드** | 자신에게 전송되는 무선 패킷만을 수신 |
| **모니터(Monitor) 모드** | (유선 NIC 의 promisc 모드와 동일) 주변의 모든 무선 패킷을 수신 |

### 시험 포인트

- aircrack-ng 5종 도구 — airmon(모니터 모드 전환) / airodump(패킷 수집) / aireplay(패킷 삽입) / airdecap(복호화) / aircrack(크랙)
- iwconfig 의 관리 모드 vs 모니터 모드 — 모니터 모드 = 무선의 promisc

## 단원 요약

1. **진단 도구** : ping(ICMP Echo) / traceroute(UDP+ICMP, Linux) vs tracert(ICMP, Windows) / netstat(`-a -r -s -n -p`) / ifconfig·ipconfig / tcpdump(전 계층 스니핑) / arp / nslookup·dig / whois.
2. **블루투스** : 마스터-슬레이브, 피코넷(7대까지) / 스캐터넷(피코넷 결합). 4대 공격 — Printing(검색) / Snarfing(파일 탈취) / Bugging(장비 제어) / Jacking(스팸).
3. **RFID** : 수동형 vs 능동형. 4대 공격(도청·트래픽 분석·위조·DoS), 7대 보안(Kill / Sleep·Wake / 블로커 태그 / Faraday Cage / Jamming / 해시 락 / XOR OTP).
4. **무선랜 일반** : SSID(이름) / BSSID(AP MAC) / ESS·ESSID / DS(유선) / Beacon(AP 광고). 표준 변천 — 802.11(평문) → 802.11b(WEP) → 802.11i(WPA·WPA2). 4대 특성(감쇠·간섭·다중경로·오류). 보안 취약점 3분류(물리·기술·관리).
5. **암호 기술** : WEP(IV 24bit + RC4 + CRC-32, 단방향 인증) → WPA(IV 48bit + TKIP + MIC, RC4 잔존) → WPA2(**AES-CCMP**) → WPA3(**SAE** 키 교환).
6. **인증 기술** : 개인 PSK(사전 공격 취약, 4-way Handshake 스니핑) vs 기업 EAP/802.1X(RADIUS + EAP-TLS/TTLS/PEAP). SSID 숨김·MAC 필터링은 보조.
7. **WAP** : 모바일 응용 표준. WDP / WTP / WSP / WML / WAE / **WTLS**. WAP2 = 종단-대-종단 보안.
8. **크래킹 도구** : airmon → airodump → aireplay → airdecap → aircrack. iwconfig 의 모니터 모드 = 무선 promisc.
