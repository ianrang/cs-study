# 01. 네트워크 일반 — OSI / TCP-IP / 네트워크 장비

> 출제기준 2과목 1-1 "네트워크 개념 이해" 와 1-2 "네트워크의 활용" 을 다룬다.
> OSI 7 계층 모델 → TCP/IP 4 계층 모델 → 네트워크 종류 → 네트워크 장비(NIC/Hub/Repeater/Bridge/Switch/Router/Gateway/VLAN) → 네트워크 공유 프로토콜(NetBIOS/NetBEUI/P2P) 순으로 정리한다.

## 0. 들어가며 — 계층 모델은 왜 학습의 출발점인가

네트워크 보안 공부는 "어떤 계층에서 어떤 헤더 정보가 노출되며, 어떤 장비가 그 계층을 책임지는가" 를 외우는 것에서 시작한다.

| 단원 영역 | 출제기준 매핑 | 통합자료 매핑(line) | 2과목.txt 매핑(line) |
|---|---|---|---|
| OSI / TCP-IP / 캡슐화 / 다중화 | 1-1-1, 1-1-2 | 1394 ~ 1494 | 1 ~ 13, 21 ~ 102, 124 ~ 127 |
| 네트워크 카드(NIC) / LLC·MAC / IEEE 802 | 1-1-1, 1-2-1 | 1472 ~ 1494 | 21 ~ 30 |
| 네트워크 장비(Hub·Repeater·Bridge·Switch·Router·Gateway·VLAN·L4) | 1-2-1 | 1495 ~ 1565 | 14 ~ 19, 39 ~ 80 |
| 네트워크 종류(Ethernet·LAN·CAN·PAN·HAN·SDN 등) | 1-1-2 | (보강) | (보강) |
| 네트워크 공유(NetBIOS·NetBEUI·P2P) | 1-2-2 | (보강) | (보강) |

학습 순서는 (1) 계층 구조 → (2) 각 계층의 데이터 단위·주소·프로토콜 → (3) 계층별 장비 → (4) 네트워크 종류·공유 형태 순서가 자연스럽다.

## 1. OSI 7 Layer 모델

### 1-1. 정의

OSI(Open System Interconnection) 7 계층 모델은 국제표준화기구(ISO) 가 제정한 개방형 시스템 간 상호연결 모델로, 분산된 이기종 시스템 간의 네트워크 상호호환을 위한 표준 아키텍처를 정의한다. 핵심 설계 원칙은 **분할 정복** — 하나의 큰 프로토콜로 설계하지 않고 기능에 따라 7개 계층으로 분리하여 각각 표준화한다.[^iso7498]

> *원본 표기 보정*: 2과목.txt L9 의 `Phyical`, L12 의 `프로토코롤` 은 OCR 결함으로 표준 표기인 `Physical` 과 `프로토콜` 로 보정했다. 본 단원에서는 이후 보정된 표기로 통일한다.

### 1-2. 7개 계층 정리

| 계층 | 계층명 | 데이터 단위 | 주요 프로토콜·기능 | 대표 장비 |
|---|---|---|---|---|
| 7 | 응용(Application) | Data | HTTP, FTP, SMTP, DNS 등 사용자 인터페이스 담당 | (서버·클라이언트 프로그램) |
| 6 | 표현(Presentation) | Data | 인코딩/디코딩, 압축/압축해제, 암호화/복호화 | — |
| 5 | 세션(Session) | Data | 어플리케이션 간 논리적 연결인 세션의 생성·관리·종료 | — |
| 4 | 전송(Transport) | Segment | TCP, UDP, SCTP, SPX (흐름·오류·혼잡 제어, 연결 제어) | L4 스위치(SLB/Failover) |
| 3 | 네트워크(Network) | Packet | IP, IPX (라우팅, 논리 주소 — IP 주소) | 라우터(Router) |
| 2 | 데이터 링크(Data Link) | Frame | Ethernet, TokenRing, FDDI, X.25, Frame Relay (인접 노드 간 신뢰성 있는 프레임 전송, 흐름·오류·회선 제어) | 스위치(Switch), 브리지(Bridge) |
| 1 | 물리(Physical) | bit | 전기적·전자적 연결 명세 (주소 정보 없음 → 모든 노드에 전달) | 허브(Hub), 리피터(Repeater) |

### 1-3. 데이터 링크 계층의 흐름·오류·회선 제어

- **흐름제어** : 송신 노드가 수신 노드의 처리 속도를 고려하여 전송을 제어
  - **정지-대기(Stop and Wait)** : 프레임 전송 후 확인 응답을 받을 때까지 대기
  - **슬라이딩 윈도우(Sliding Window)** : 응답을 받기 전에 수신 가능한 범위 내에서 여러 프레임을 전송
- **오류제어** : 전송 중 오류·손실 해결
  - **후진 오류 수정(BEC, Backward Error Correction)** : 데이터 전송 시 오류 검출 부가 정보를 함께 전송. 오류 발견 시 재전송 요청을 위한 절차를 ARQ(Automatic Repeat Request) 라 함
  - **전진 오류 수정(FEC, Forward Error Correction)** : 데이터 전송 시 오류 검출·수정까지 가능한 부가 정보를 함께 전송 (재전송 불필요)
- **회선제어**
  - 회선 구성 방식 : 점-대-점(Point-to-Point), 다중점(Multipoint)
  - 전송 방식 : 단방향(Simplex), 반이중(Half Duplex), 전이중(Full Duplex)

### 1-4. 캡슐화·역캡슐화

- **캡슐화(Encapsulation)** : 송신 측에서 상위 계층 데이터가 하위 계층으로 보내질 때, 하위 계층 프로토콜이 자신의 기능 수행에 필요한 헤더(필요 시 트레일러) 를 추가
- **역캡슐화(Decapsulation)** : 수신 측에서 하위 계층 데이터가 상위 계층으로 보내질 때, 해당 계층의 헤더 정보를 확인한 후 제거

### 1-5. 다중화·역다중화

- **다중화(Multiplexing)** : 하나의 기능을 여러 영역에서 동시에 사용. 상위 계층의 여러 프로토콜이 하위 계층의 단일 프로토콜을 이용해 데이터를 전달하는 방식. 헤더에 프로토콜 식별자 정보 추가.
- **역다중화(Demultiplexing)** : 공유하는 기능(매체) 으로부터 개별 영역으로 분할. 하위 계층의 프로토콜이 여러 상위 계층 프로토콜 중 하나를 식별하여 데이터를 전달.

### 시험 포인트

- 7 계층 이름과 데이터 단위(bit / frame / packet / segment / Data) 를 순서대로 암기
- 각 계층의 대표 프로토콜과 대표 장비 매칭 — 1·2·3·4계층은 출제 빈도 매우 높음
- ARQ ↔ BEC, 전이중 ↔ Full Duplex, 슬라이딩 윈도우 ↔ 흐름제어 등의 매칭
- 캡슐화/역캡슐화 방향(상위→하위 / 하위→상위) 과 다중화/역다중화 차이

## 2. TCP/IP 4 계층 모델

### 2-1. 정의 및 OSI 와의 매핑

TCP/IP 모델은 인터넷에서 실제로 사용되는 4 계층 모델로, OSI 7 계층의 일부를 통합한 형태이다.

| TCP/IP 계층 | OSI 매핑 | 데이터 단위 / 주소 |
|---|---|---|
| 4. 응용(Application) | OSI 5·6·7 | Data |
| 3. 전송(Transport) | OSI 4 | Segment / Port 주소 (16bit) |
| 2. 인터넷(Internet) | OSI 3 | Packet / IP 주소 (IPv4 32bit, IPv6 128bit) |
| 1. 네트워크 인터페이스(Network Interface) | OSI 1·2 | Frame / MAC 주소 (48bit) |

### 2-2. 계층별 역할

| 계층 | 설명 |
|---|---|
| **네트워크 인터페이스 계층** | 인접한 노드 간(node-to-node delivery) 신뢰성 있는 데이터 전송 담당. node 식별을 위해 MAC 주소(48bit) 사용 — 상위 24bit 는 제조사 식별코드(OUI), 하위 24bit 는 제조사가 할당한 일련번호 |
| **인터넷 계층** | 호스트 간 라우팅 담당. 호스트 식별을 위한 논리 주소로 IPv4(32bit) 또는 IPv6(128bit) 사용. 주요 프로토콜 — IP, ICMP, IGMP, ARP, RARP |
| **전송 계층** | 프로세스 간 신뢰성 있는 데이터 전송 담당. 프로세스 식별을 위한 16bit 포트 주소 사용. 주요 프로토콜 — TCP, UDP, SCTP |
| **응용 계층** | 네트워크 서버/클라이언트 프로그램 담당, 사용자와의 인터페이스 |

### 2-3. 응용 계층 주요 프로토콜과 well-known 포트

| 서비스 | 포트 / 프로토콜 |
|---|---|
| HTTP (웹) | 80/tcp |
| HTTPS (웹) | 443/tcp |
| FTP | 20/tcp(데이터 채널), 21/tcp(제어 채널) |
| SSH / SFTP | 22/tcp |
| Telnet | 23/tcp |
| SMTP (메일 전송) | 25/tcp |
| POP3 (메일 수신) | 110/tcp |
| IMAP (메일 수신·복사) | 143/tcp |
| DNS | 53/tcp, 53/udp |
| DHCP | 67/udp(서버), 68/udp(클라이언트) |
| TFTP | 69/udp |
| SNMP | 161/udp(에이전트), 162/udp(매니저) |

### 2-4. 네트워크 인터페이스 계층 — LLC / MAC / IEEE 802

IEEE 802 표준은 LAN 상의 데이터 링크 계층을 LLC(Logical Link Control) 와 MAC(Media Access Control) 두 부계층으로 구분한다.[^ieee802]

| 부계층 | 역할 |
|---|---|
| **LLC** | 상위 계층(네트워크 계층) 으로 안전하고 정상적인 데이터를 전달하기 위해 오류 검출 및 상위 프로토콜에 대한 타입을 확인. 네트워크 계층과의 연결을 담당 |
| **MAC** | 신호변환을 통해 물리 계층과 연결하고 MAC 주소를 통해 주소를 확인하여 정상적으로 데이터가 송수신되고 있는지 확인. 물리 계층과의 연결을 담당 |

LAN 환경 프로토콜 — Ethernet, TokenRing, FDDI
WAN 환경 프로토콜 — X.25, Frame Relay, PPP, SLIP

### 시험 포인트

- TCP/IP 4 계층의 데이터 단위·주소 길이 매칭 (MAC 48bit / IPv4 32bit / IPv6 128bit / Port 16bit)
- MAC 주소의 OUI(상위 24bit, 제조사 식별) 와 일련번호(하위 24bit) 분할
- IEEE 802 표준에서 LLC ↔ 네트워크 계층 연결, MAC ↔ 물리 계층 연결
- 응용 계층 well-known 포트 — 특히 FTP(20/21), SSH(22), SMTP(25), DNS(53/tcp·udp), HTTP(80), POP3(110), IMAP(143), SNMP(161/162)

## 3. 네트워크 종류

출제기준 1-1-2 는 다음 네트워크 종류를 명시한다 — Ethernet / LAN / Intranet / Extranet / Internet / CAN / PAN / HAN / SDN. 통합자료에 명시된 분류 표가 없으므로 **외부 보강** 에 해당하는 영역이며 1차 출처 각주를 단다.

### 3-1. 분류

| 종류 | 설명 |
|---|---|
| **Ethernet** | IEEE 802.3 표준 기반의 LAN 기술. CSMA/CD(현재는 스위치 환경에서 사용되지 않음) 매체 접근 제어. 가장 보편적인 유선 LAN 데이터 링크 기술[^ieee8023] |
| **LAN** (Local Area Network) | 동일 건물·캠퍼스 등 좁은 지역 내의 네트워크. 일반적으로 사설 소유 |
| **Intranet** | 기업·기관 내부의 사설 네트워크. 외부 인터넷과 분리되거나 제한적으로 연결됨 |
| **Extranet** | Intranet 을 협력사·고객사 등 외부의 인가된 사용자에게 확장한 네트워크 |
| **Internet** | TCP/IP 기반의 전 세계적 공중망 |
| **CAN** (Campus Area Network) | 대학·기업 캠퍼스 등 여러 건물에 걸친 LAN 의 집합. LAN 보다 넓고 MAN/WAN 보다 좁음 (IETF·IEEE 가 부여한 단일 표준 정의는 없으며 업계 관행 용어로 사용됨) |
| **PAN** (Personal Area Network) | 개인 주변 수 미터 범위의 네트워크. 무선 PAN 의 대표 표준은 IEEE 802.15 계열 — Bluetooth(IEEE 802.15.1), Zigbee(IEEE 802.15.4)[^ieee802154]. 유선 PAN 으로는 USB(USB-IF 표준) 가 사용됨 |
| **HAN** (Home Area Network) | 가정 내 가전·IoT 기기를 연결하는 네트워크. NIST IR 7628 은 스마트 그리드 사이버보안 가이드에서 HAN 을 보안 도메인 중 하나로 분류·운용 사례를 다룬다[^nist-han] |
| **SDN** (Software-Defined Network) | 네트워크의 제어 평면(Control Plane) 과 데이터 평면(Data Plane) 을 분리하여 중앙 컨트롤러가 소프트웨어로 네트워크를 제어하는 아키텍처. OpenFlow 가 대표적인 남향 인터페이스 프로토콜[^onf-sdn][^rfc7426] |

### 시험 포인트

- LAN < CAN < MAN < WAN 의 범위 확장 순서
- PAN 의 대표 기술(Bluetooth/Zigbee) 과 IEEE 802.15 계열 매칭
- SDN 의 핵심 — Control Plane 과 Data Plane 의 분리, 중앙 컨트롤러 기반 제어, OpenFlow 프로토콜
- Intranet vs Extranet — 내부 한정 vs 외부 인가 사용자 확장

## 4. 네트워크 장비

각 계층에서 동작하는 대표 장비를 계층별로 매핑한다. 1·2·3·4 계층 장비는 출제 빈도가 매우 높다.

### 4-1. 계층별 장비 매핑

| 계층 | 장비 | 핵심 기능 |
|---|---|---|
| 1 (물리) | Hub, Repeater | 신호 재생·중계, 주소 정보 없음 |
| 2 (데이터 링크) | Bridge, Switch, NIC | MAC 주소 기반 프레임 전송 |
| 3 (네트워크) | Router | IP 주소 기반 라우팅 |
| 4 (전송) | L4 스위치 | 포트 기반 부하 분산(SLB), 장애 조치(Failover) |
| 7 (응용) | L7 스위치, Gateway | 어플리케이션 컨텐츠 분석, 이기종 프로토콜 변환 |

### 4-2. NIC (Network Interface Card)

회선에 데이터를 송수신하는 역할을 하는 네트워크 장비. 동작 모드는 두 가지:

| 동작 모드 | 동작 |
|---|---|
| **Normal 모드** | 자신이 목적지가 아닌 패킷은 모두 폐기 (NIC 에서 걸러짐) |
| **Promiscuous(Promisc) 모드** | 자신이 목적지가 아닌 패킷도 모두 수신 → 더미 허브 환경에서 스니핑 가능 |

스니핑 도구 — Wireshark, tcpdump

### 4-3. 허브(Hub) 와 리피터(Repeater)

- **허브** : 들어오는 신호를 연결된 모든 포트로 전달하는 중계 장치. 더미 허브 환경에서는 NIC 를 Promisc 모드로 설정하면 스니핑 가능
- **리피터** : 감쇠된 전송 신호를 새롭게 생성하여 다시 전달해주는 재생 중계 장치

### 4-4. 브리지(Bridge)

물리적으로 떨어진 동일한 LAN 을 연결해주는 장비. 데이터 링크 계층에서 동작.

### 4-5. 스위치(Switch)

유입된 프레임의 MAC 주소의 포트로만 프레임을 전달해주는 네트워크 장비. 내부적으로 **MAC Address Table** 을 가지고 있어서 스위치 포트에 연결된 노드의 MAC 정보를 담는다 → 목적지로만 패킷 전송 가능 → 스니핑 불가능. 스니핑하려면 **스위치 재밍 / ARP 스푸핑 / ARP 리다이렉트 / ICMP 리다이렉트 / SPAN(Port Mirroring)** 을 사용해야 함.

#### 4-5-1. Fail Safe vs Fail Secure

| 정책 | 동작 | 적용 |
|---|---|---|
| **Fail Safe** | 장애 발생 시 모든 기능을 허용 | 가용성을 중시하는 장비 (대부분의 네트워크 장비) |
| **Fail Secure** | 장애 발생 시 모든 기능을 차단 | 보안성을 중시하는 정책 |

> *원본 표기 보정*: 2과목.txt L46 의 `장애ㅏㄱ 발생하면` 은 OCR 결함으로 `장애가 발생하면` 으로 보정했다. 스위치 재밍이 가능한 이유는 스위치가 Fail Safe 정책을 따르기 때문 — MAC Address Table 의 버퍼를 오버플로우시켜 스위치가 장애 상태에 빠지면 더미 허브처럼 모든 포트로 패킷을 전달하기 시작한다.

#### 4-5-2. 스위칭 방법

| 방법 | 동작 |
|---|---|
| **Store-and-forward** | 들어오는 프레임을 전부 받아들인 다음에 처리를 시작 |
| **Cut-Through** | 들어오는 프레임의 처음 48bit(목적지 MAC) 만을 확인하여 처리를 시작 |
| **Fragment-Free** | 위 두 방식의 결합. 처음 512bit 를 확인하여 처리를 시작 |

#### 4-5-3. 스위치 기능

| 기능 | 동작 |
|---|---|
| **Learning** | 프레임의 출발지 MAC 주소를 테이블에 생성하여 MAC 주소를 식별 |
| **Flooding** | 프레임의 MAC 주소가 테이블에 없을 경우 모든 포트로 프레임을 전송 |
| **Forwarding** | 테이블을 참조하여 전송할 프레임의 목적지 MAC 주소의 포트로 프레임을 전송 |
| **Filtering** | 목적지 외에는 프레임을 전송하지 않음 |
| **Aging** | 테이블에 저장된 MAC 주소를 유지하는 시간 |

### 4-6. 라우터(Router)

라우팅을 담당하는 장비로 최적의 경로를 선정해서 패킷을 포워딩한다. 데이터 링크 계층의 브로드캐스트와 멀티캐스트를 포워딩하지 않으며, 서로 다른 VLAN 간 통신을 가능하게 하고 기본적인 보안 기능과 QoS 관련 기능을 지원한다.

#### 4-6-1. 라우팅 방식

| 방식 | 설명 |
|---|---|
| **정적 라우팅** | 관리자가 직접 라우팅 테이블을 설정 |
| **동적 라우팅** | 라우팅 프로토콜을 이용하여 자동으로 경로를 탐색하고 패킷을 전달 |

#### 4-6-2. 라우팅 프로토콜

AS(Autonomous System) 는 하나의 관리 도메인에 속해 있는 라우터의 집합이다.

| 분류 | 알고리즘 / 프로토콜 |
|---|---|
| **IGP** (자율 시스템 내부 라우팅) | 거리 벡터 — RIP, RIPv2, IGRP / 링크 상태 — IS-IS, OSPF / 하이브리드 — EIGRP |
| **EGP** (자율 시스템 간 라우팅) | 경로 벡터 — BGP |

#### 4-6-3. 라우팅 테이블

| 용어 | 의미 |
|---|---|
| **Genmask** | 목적지 호스트, 목적지 네트워크, 기본 게이트웨이를 식별하기 위한 마스크 |
| **Flags** | 해당 경로에 대한 상태정보 플래그 |

**IP 라우팅 규칙**
1. 목적지 주소가 자신과 동일한 네트워크에 있다면 직접 전송
2. 동일한 네트워크에 속하지 않는다면 1차 경유지 주소(게이트웨이) 를 라우팅 테이블을 참조하여 찾음
3. 목적지가 자신일 경우 상위 계층으로 데이터를 전달

**라우팅 테이블 검색 방식**
1. IP 패킷의 목적지 IP 와 라우팅 테이블의 netmask/genmask 를 bit AND(&) 연산
2. 연산 수행 후 라우팅 테이블의 destination 필드와 비교
3. 일치하는 경로를 선택하여 패킷을 전송

**라우팅 테이블 검색 우선순위**
1. 먼저 목적지 호스트 주소와 일치하는 경로를 찾음
2. 일치하는 경로가 없으면 목적지 네트워크 주소와 일치하는 경로를 찾음
3. 일치하는 경로가 없으면 default gateway 로 보냄

**Metric** : 라우팅 경로의 효율성(우선순위) 을 나타내는 값. 값이 클수록 비용이 크다 → 메트릭이 클수록 우선순위가 낮다.

### 4-7. 게이트웨이(Gateway)

서로 다른 프로토콜·아키텍처를 사용하는 네트워크를 연결해주는 장비. 응용 계층까지 동작하여 프로토콜 변환을 수행할 수 있음.

### 4-8. L4 스위치

전송 계층에서 동작하며 서버 트래픽 부하분산(SLB, Server Load Balancing) 과 Failover(장애조치) 기능을 제공.

### 4-9. VLAN (Virtual LAN)

물리적인 선이 아닌 논리적으로 브로드캐스트 도메인을 나누기 위해 사용하는 기술. 관리자가 서로 다른 논리적 그룹에 대하여 서로 다른 보안 정책을 사용할 수 있게 함. (상세 분류 — Port/MAC/Network/Protocol 기반 — 은 [08 단원 §VLAN](08.security-solutions-and-monitoring.md) 에서 다룸)

### 시험 포인트

- 계층별 장비 매칭 — 특히 1계층(허브·리피터), 2계층(스위치·브리지), 3계층(라우터), 4계층(L4 스위치)
- NIC Promisc vs Normal 모드 — 스니핑 가능 조건
- Fail Safe(가용성) vs Fail Secure(보안성) 정책 차이
- 스위칭 방법 3종(Store-and-forward / Cut-Through(48bit) / Fragment-Free(512bit)) 및 각 비트 수치
- 스위치 기능 5종(Learning / Flooding / Forwarding / Filtering / Aging) 의 동작
- IGP/EGP 분류와 프로토콜 매칭 — 거리벡터(RIP) / 링크상태(OSPF·IS-IS) / 하이브리드(EIGRP) / 경로벡터(BGP)
- 라우팅 테이블 검색 우선순위 (호스트 주소 → 네트워크 주소 → default gateway)
- 메트릭이 클수록 우선순위가 낮다는 점

## 5. 네트워크 공유 프로토콜

출제기준 1-2-2 가 명시한 공유 프로토콜은 NetBIOS, NetBEUI, P2P 다. 통합자료에 분류 표가 없는 영역으로 외부 보강한다.

### 5-1. NetBIOS (Network Basic Input/Output System)

PC 가 LAN 환경에서 다른 컴퓨터와 통신하기 위해 사용하는 API 로, 네임 서비스·세션 서비스·데이터그램 서비스를 제공한다. 현재는 TCP/IP 위에서 동작하는 NetBIOS over TCP/IP(NBT) 형태로 사용된다.[^rfc1001][^rfc1002]

| 서비스 | 포트 |
|---|---|
| NetBIOS Name Service | 137/udp |
| NetBIOS Datagram Service | 138/udp |
| NetBIOS Session Service | 139/tcp |

> 보안 측면 — NetBIOS Name Service(137) 는 NBNS(NetBIOS Name Server) 스푸핑 공격에 취약하며, NetBIOS Session(139) 는 윈도우 공유 폴더 공격(Null Session, IPC$ 등) 의 진입점이다.

### 5-2. NetBEUI (NetBIOS Extended User Interface)

NetBIOS 의 확장 인터페이스로, IBM·Microsoft 가 소규모 LAN 용으로 설계한 비라우팅(non-routable) 프로토콜. 라우팅을 지원하지 않으므로 LAN 내부에서만 사용 가능하며 현재는 거의 사용되지 않는다.[^ms-netbeui]

### 5-3. P2P (Peer-to-Peer)

서버-클라이언트 구조 없이 모든 노드가 동등한 자격으로 자원을 공유하는 분산 네트워크 모델. 파일 공유(BitTorrent), 분산 컴퓨팅, 블록체인 등에서 사용된다.

| 보안 위협 | 설명 |
|---|---|
| 악성코드·랜섬웨어 유포 | P2P 사이트를 통해 다운로드한 파일이 감염원이 될 수 있음 |
| 정보 유출 | 공유 폴더가 잘못 설정되면 민감 정보가 외부로 노출 |
| 봇넷 C&C | 분산형 봇넷이 P2P 통신으로 명령을 전파(중앙 C&C 서버 없음) |

### 시험 포인트

- NetBIOS 의 3개 서비스와 포트(137/udp · 138/udp · 139/tcp)
- NetBEUI 가 비라우팅 프로토콜이라는 점
- P2P 의 위협 범주 — 악성코드 유포, 정보 유출, 분산형 봇넷 C&C

## 단원 요약

1. **OSI 7 계층** : Application / Presentation / Session / Transport / Network / Data Link / Physical. 데이터 단위는 Data / Data / Data / Segment / Packet / Frame / bit.
2. **TCP/IP 4 계층** : 응용 / 전송 / 인터넷 / 네트워크 인터페이스. 주소 길이 — MAC 48bit, IPv4 32bit, IPv6 128bit, Port 16bit.
3. **데이터 링크 부계층** : LLC(상위 네트워크 계층 연결) / MAC(하위 물리 계층 연결). IEEE 802 표준이 LAN 의 데이터 링크 계층을 두 부계층으로 분할.
4. **네트워크 종류** : Ethernet/LAN/Intranet/Extranet/Internet 외에 출제기준 명시 — CAN(캠퍼스), PAN(개인, Bluetooth/Zigbee), HAN(가정), SDN(Control/Data Plane 분리, OpenFlow).
5. **계층별 장비** : 1계층 Hub·Repeater / 2계층 Bridge·Switch·NIC / 3계층 Router / 4계층 L4 스위치 / 7계층 L7·Gateway.
6. **스위치** : MAC Address Table 보유 → 목적지 포트로만 전송. Fail Safe(기본) → 장애 시 허브처럼 동작 → 스위치 재밍 가능. 스위칭 방법(Store-and-forward / Cut-Through 48bit / Fragment-Free 512bit). 5대 기능(Learning / Flooding / Forwarding / Filtering / Aging).
7. **라우팅** : 정적 vs 동적, IGP(RIP/OSPF/IS-IS/EIGRP) vs EGP(BGP), 검색 우선순위(호스트 → 네트워크 → default gateway), 메트릭이 클수록 우선순위 낮음.
8. **네트워크 공유** : NetBIOS(137/138/139), NetBEUI(비라우팅), P2P(분산 봇넷·악성코드 유포).
