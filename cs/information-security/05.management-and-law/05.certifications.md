# 5과목 05 — 정보보호 인증제도

> 출제기준 매핑: 1.4 정보보호 인증제도 이해 — 정보보호 관리체계 인증의 이해 / 정보보호 제품인증의 이해

## §0. 들어가며

본 단원은 한국과 국제의 정보보호 인증제도를 다룬다. 한국에서 가장 출제 빈도가 높은 인증은 **ISMS-P (정보보호 및 개인정보보호 관리체계 인증)** 이며, 정보보호 제품의 보안성 평가는 **공통평가기준 (Common Criteria, ISO/IEC 15408)** 과 **KCMVP (암호모듈 검증)** 으로 이루어진다. 추가로 **PIA (개인정보 영향평가)** 는 개인정보 처리 시스템에 대한 사전적 평가 제도이다.

본 단원은 [정보통신망법] §47 (정보보호 관리체계 인증) [^infonet-act], [개인정보보호법] §32-2 (개인정보보호 인증) 및 §33 (개인정보 영향평가) [^pipa], 「정보보호 및 개인정보보호 관리체계 인증 등에 관한 고시」 [^isms-p-cert], ISO/IEC 27001:2022 [^iso27001], ISO/IEC 15408 시리즈 [^iso15408], KISA 인증제도 안내 [^kisa-cert] 를 1차 출처로 한다.

학습 순서:
1. ISMS-P 인증제도 — 개요·법적 근거 (§1)
2. ISMS-P 인증기준 — 3개 영역·통제 구조 (§2)
3. ISMS·ISMS-P 의무대상·심사 절차 (§3·§4)
4. ISO/IEC 27001 국제 ISMS 표준 (§5)
5. 공통평가기준 (CC, ISO/IEC 15408) — EAL 7단계 (§6)
6. KCMVP 암호모듈 검증 (§7)
7. GS·CC·기타 제품 인증 비교 (§8)
8. 개인정보 영향평가 (PIA) (§9)

## §1. ISMS-P 인증제도

> **ISMS-P (Information Security Management System – Personal information)** : 정보보호 및 개인정보보호 관리체계 인증. 정보보호 및 개인정보보호를 위한 일련의 조치와 활동이 인증기준에 적합함을 **한국인터넷진흥원 (KISA) 또는 인증기관** 이 증명하는 제도.[^isms-def]

### §1-1. 법적 근거

ISMS-P 는 두 법령에 근거를 둔 통합 인증이다.

| 인증 | 법적 근거 | 소관 부처 |
|---|---|---|
| ISMS (정보보호 관리체계) | [정보통신망 이용촉진 및 정보보호 등에 관한 법률] §47 [^infonet-act] | 과학기술정보통신부 |
| 개인정보보호 관리체계 | [개인정보 보호법] §32-2 [^pipa] | 개인정보보호위원회 |
| 통합 운영 | 「정보보호 및 개인정보보호 관리체계 인증 등에 관한 고시」 [^isms-p-cert] | KISA (인증기관) |

ISMS-P 는 **2018년 11월** ISMS 와 PIMS (개인정보보호 관리체계) 가 통합되어 출범하였다. 통합 이전에는 ISMS 와 PIMS 가 분리 운영되었다.

### §1-2. ISMS vs ISMS-P 구분

조직은 다음 두 인증 중 선택할 수 있다.

| 구분 | ISMS | ISMS-P |
|---|---|---|
| 영문 | Information Security Management System | Information Security Management System – Personal information |
| 적용 범위 | 정보보호 영역만 | 정보보호 + 개인정보보호 (개인정보 처리 단계 포함) |
| 인증기준 영역 | 2개 영역 | 3개 영역 (개인정보 영역 추가) |
| 적합 대상 | 개인정보 처리가 적은 조직 | 개인정보 처리가 많은 조직 |

### §1-3. 시험 포인트

- ISMS-P 의 법적 근거 = **정통망법 §47 + 개보법 §32-2**.
- 인증기관 = **한국인터넷진흥원 (KISA)** 또는 지정 인증기관.
- ISMS-P 통합 출범 = **2018년 11월**.
- ISMS = 2개 영역 / ISMS-P = 3개 영역 (개인정보 영역 추가).

## §2. ISMS-P 인증기준 구조

ISMS-P 인증기준 [^isms-p-cert] 은 **3개 영역·21개 분야·102개 인증기준** 으로 구성된다 (작성 시점 기준).

> *주의*: 인증기준 영역·분야·통제항목 수는 KISA 의 정기 개정으로 변동될 수 있다. 작성 시점 (2026.04.30) 의 KISA ISMS-P 인증제도 안내 [^kisa-cert] 를 직접 확인하여 사용해야 한다. 본 문서는 2023.10.31 일부 개정 (시행 2024.5.1) 기준을 반영한다.

### §2-1. 3개 영역

| 영역 | 분야 수 | 인증기준 수 | ISMS 적용 | ISMS-P 적용 |
|---|---|---|---|---|
| **1. 관리체계 수립 및 운영** | 4 | 16 | ✓ | ✓ |
| **2. 보호대책 요구사항** | 12 | 64 | ✓ | ✓ |
| **3. 개인정보 처리 단계별 요구사항** | 5 | 22 | – | ✓ |
| **합계 (ISMS-P)** | **21** | **102** | – | 102개 |
| **합계 (ISMS, 영역 1+2)** | 16 | 80 | 80개 | – |

> *주의*: 본 표의 분야 수·인증기준 수는 2023.10.31 개정안 기준이며, ISMS 인증 (영역 1+2) = 16분야 80개, ISMS-P 인증 (영역 1+2+3) = 21분야 102개로 계수된다. 시험 응시 시점에 KISA 공식 발표 수치를 직접 확인할 것.

| 1.2 위험 관리 | 정보자산 식별, 현황 분석, 위험 평가, 보호대책 선정 |
| 1.3 관리체계 운영 | 보호대책 구현, 보호대책 공유, 운영현황 관리 |

### §2-4. 영역 3 — 개인정보 처리 단계별 요구사항 (ISMS-P 만)

| 3.2 개인정보 보유 및 이용 시 보호조치 | 개인정보 현황관리, 개인정보 품질보장, 개인정보 표시제한 및 이용 시 보호조치, 이용자 단말기 접근 보호, 개인정보 목적 외 이용 및 제공 |

| 3.4 개인정보 파기 시 보호조치 | 개인정보 파기, 처리목적 달성 후 보유 시 조치, 휴면 이용자 관리 |
| 3.5 정보주체 권리보호 | 개인정보처리방침 공개, 정보주체 권리보장, 이용내역 통지 |

### §2-5. 시험 포인트

- ISMS-P 인증기준 = **3개 영역**.
- 영역 1 = 관리체계 수립·운영 / 영역 2 = 보호대책 / 영역 3 = 개인정보 처리 단계별.
- ISMS = 영역 1 + 영역 2 / ISMS-P = 영역 1 + 영역 2 + 영역 3.
- 영역 3 의 분야 수 = **5** (3.1~3.5).
- 인증기준 수의 정확한 수치는 KISA 공식 자료에서 작성 시점 직접 확인.

## §3. ISMS·ISMS-P 의무대상

[정보통신망법] §47 ②항 [^infonet-act] 과 그 시행령은 **일정 규모 이상의 정보통신서비스 제공자** 에게 ISMS 인증을 의무화하고 있다.

### §3-1. ISMS 의무인증 대상

[정보통신망법 시행령] §49 [^infonet-act] 의 의무인증 대상 (작성 시점 현행본 기준):

| 분류 | 기준 |
|---|---|
| ISP (정보통신망 서비스 제공자) | 전기통신사업법에 따른 기간통신사업자로서 사업장 내 정보통신망 보유 |
| IDC (집적정보통신시설 사업자) | 정보통신서비스 부문 매출 100억 원 이상 |
| 일정 규모 이상 정보통신서비스 제공자 | 전년도 매출 또는 이용자 수 일정 기준 이상 — 시행령에서 구체적 수치 |
| 의료기관·고등교육법상 학교 | 매출·재학생 수 일정 기준 이상 |

> *주의*: 시행령의 매출·이용자 수 수치는 정기 개정으로 변동된다 (예: 정보통신서비스 부문 전년도 매출 100억 원 이상, 일평균 이용자 100만 명 이상 — 이는 과거 기준이며 현행 수치는 [정보통신망법 시행령] §49 직접 확인). 시험 응시 시점의 시행령을 국가법령정보센터에서 확인하여 답안에 사용할 것.

### §3-2. ISMS-P 인증

ISMS-P 는 **자율 인증** 이다. 다만 개인정보 처리량이 많은 조직 (포털·전자상거래·금융 등) 은 ISMS-P 를 선택하는 경우가 일반적이며, 일부 공공·금융 분야에서는 별도 규정으로 ISMS-P 를 요구한다.

### §3-3. 비의무대상의 자율 인증

의무대상이 아닌 조직도 자율적으로 ISMS·ISMS-P 인증을 신청할 수 있으며, 이 경우 동일한 인증기준이 적용된다.

### §3-4. 시험 포인트

- ISMS 의무인증 = **정보통신망법 §47 ②항** + 시행령 §49.
- 의무인증 대상 = ISP·IDC·일정 규모 이상 정보통신서비스 제공자·일정 규모 이상 의료기관·학교.
- ISMS-P = **자율 인증** (의무 대상 별도 규정 제외).
- 의무인증 미이행 시 과태료 ([정보통신망법] §76).

## §4. 인증 절차

ISMS-P 인증 절차는 「정보보호 및 개인정보보호 관리체계 인증 등에 관한 고시」 [^isms-p-cert] 에 따라 다음과 같이 진행된다.

### §4-1. 인증 6단계

| 단계 | 활동 | 산출물 |
|---|---|---|
| 1. 신청 | 조직이 KISA·인증기관에 인증 신청 | 신청서, 운영명세서, ISMS-P 운영 증빙 |
| 2. 계약 및 예비점검 | 인증기관 선정·계약, 예비점검 (선택) | 인증 계약서 |
| 3. 심사 (인증심사) | 인증기관의 심사원이 현장 심사 — 결함 식별 | 심사 보고서 |
| 4. 결함 보완 | 식별된 결함을 조직이 보완 (보완 조치 기간 일반적 30~40일) | 보완조치 결과 |
| 5. 인증위원회 심의 | 인증위원회가 심사 결과·보완조치 검토 | 인증위원회 심의 결과 |
| 6. 인증서 발급 | 인증위원회 의결 후 인증서 발급 | 인증서 |

### §4-2. 인증 유효기간 및 사후관리

| 항목 | 내용 |
|---|---|
| 인증 유효기간 | **3년** |
| 사후관리 심사 | **연 1회** (1·2차 사후관리) |
| 갱신 심사 | 유효기간 만료 전 갱신 신청 → 재심사 |
| 인증 취소 | 중대한 보안사고·허위 사실·결함 미보완 시 |

### §4-3. 심사원 자격

ISMS-P 심사원은 KISA 가 양성·인증한다.

| 등급 | 자격 요건 (개략) |
|---|---|
| 심사원보 | 정보보호 또는 개인정보보호 분야 1년 이상 + KISA 양성과정 수료 |
| 심사원 | 심사원보로서 일정 횟수 이상의 심사 참여 + 평가 통과 |
| 선임심사원 | 심사원으로서 일정 횟수 이상의 심사 참여 + 평가 통과 |

> *주의*: 심사원 자격 요건의 구체적 수치는 KISA 「정보보호 및 개인정보보호 관리체계 인증심사원 자격 운영지침」 [^kisa-cert] 에서 작성 시점에 확인.

### §4-4. 시험 포인트

- 인증 유효기간 = **3년**.
- 사후관리 심사 = **연 1회**.
- 갱신 심사는 만료 전 신청.
- 인증위원회 심의 후 인증서 발급.

## §5. ISO/IEC 27001 — 국제 ISMS 표준

> **ISO/IEC 27001** : 정보보호 관리체계 (ISMS) 의 요구사항을 정의한 **국제 표준**.[^iso27001]

### §5-1. ISO/IEC 27001 의 변천

| 버전 | 발행 연도 | 주요 변경 |
|---|---|---|
| ISO/IEC 27001:2005 | 2005 | 최초 ISO 표준화 (BS 7799-2 기반) |
| ISO/IEC 27001:2013 | 2013.10 | HLS (High Level Structure) 도입, 통제 114개 |
| **ISO/IEC 27001:2022** | **2022.10** | Annex A 통제 재편성 (93개), 4개 카테고리 |

### §5-2. ISO/IEC 27001:2022 구조

| 절 | 제목 | PDCA 매핑 |
|---|---|---|
| §0~§3 | 서문·범위·인용·용어 | – |
| §4 | 조직의 환경 (Context of the organization) | Plan |
| §5 | 리더십 (Leadership) | Plan |
| §6 | 계획 (Planning) | Plan |
| §7 | 지원 (Support) | Do |
| §8 | 운영 (Operation) | Do |
| §9 | 성과평가 (Performance evaluation) | Check |
| §10 | 개선 (Improvement) | Act |

ISO/IEC 27001:2022 §6.1.3 은 **Annex A 의 93개 통제** (ISO/IEC 27002:2022 의 통제 [^iso27002]) 를 SoA 로 작성할 것을 요구한다 (5과목 02 §8 참조).

### §5-3. ISMS-P vs ISO/IEC 27001 비교

| 비교 항목 | ISMS-P | ISO/IEC 27001:2022 |
|---|---|---|
| 인증 주체 | KISA·국내 인증기관 | 국제 인증기관 (KAB·UKAS 등 인정) |
| 적용 국가 | 한국 | 전 세계 |
| 법적 근거 | 정통망법 §47, 개보법 §32-2 | 자율 (계약·요구) |
| 인증기준 | 3개 영역 21분야 102개 인증기준 (작성 시점 기준) | 7개 절 §4~§10 + Annex A 93개 통제 |
| 개인정보보호 통합 | 통합 (영역 3) | 별도 (ISO/IEC 27701 PIMS) |
| 유효기간 | 3년 | 3년 |

### §5-4. ISO/IEC 27001:2013 vs 27001:2022 변경

| 비교 항목 | 27001:2013 | 27001:2022 |
|---|---|---|
| 본문 절 수 | 7개 절 (§4~§10) | 7개 절 (§4~§10) |
| Annex A 카테고리 수 | 14개 | 4개 |
| Annex A 통제 수 | 114개 | 93개 |
| 신규 통제 | – | 11개 신설 (Threat Intelligence, Cloud, Data Masking 등) |

### §5-5. 시험 포인트

- ISO/IEC 27001:2022 = **2022.10 발행**.
- Annex A 통제 = **93개** (4개 카테고리: 조직·인적·물리·기술).
- ISO 27001:2013 = 114개 (14개 카테고리). 27001:2022 = 93개 (4개 카테고리).
- ISMS-P 와 ISO/IEC 27001 은 **별개 인증** — 한국·국제 별개 체계.
- ISO/IEC 27701 = PIMS (개인정보보호 관리체계 — 27001 확장).

## §6. 공통평가기준 — Common Criteria (CC)

> **공통평가기준 (CC, Common Criteria for Information Technology Security Evaluation, ISO/IEC 15408)** : 정보보호 제품의 보안성을 평가하기 위한 **국제 표준**.[^iso15408]

### §6-1. CC 의 역사

CC 는 미국 TCSEC (1985), 유럽 ITSEC (1991), 캐나다 CTCPEC 의 통합으로 1998 년 출범했으며, 2009 년 ISO/IEC 15408 로 표준화되었다.

| 표준 | 발행 |
|---|---|
| TCSEC (Orange Book) | 미국 DoD 5200.28-STD, 1985 |
| ITSEC | 유럽 EU, 1991 |
| CTCPEC | 캐나다, 1993 |
| Common Criteria v2.x → v3.x | 1998~ |
| ISO/IEC 15408 (CC) | 2009 (현행: 15408-1·2·3·4·5, 2022 개정) [^iso15408] |

### §6-2. CC 의 구조 — ISO/IEC 15408 시리즈

| 표준 | 제목 |
|---|---|
| ISO/IEC 15408-1 | 개요·일반 모델 |
| ISO/IEC 15408-2 | 보안 기능 컴포넌트 |
| ISO/IEC 15408-3 | 보안 보증 컴포넌트 |
| ISO/IEC 15408-4 | 평가 방법 (CEM 통합) |
| ISO/IEC 15408-5 | 사전정의 보안 요구사항 패키지 |

### §6-3. CC 평가의 핵심 개념

| 용어 | 정의 |
|---|---|
| TOE (Target of Evaluation) | 평가 대상 제품 또는 시스템 |
| PP (Protection Profile) | 보호 프로파일 — 특정 제품 유형에 대한 표준 보안 요구사항 |
| ST (Security Target) | 보안 목표서 — 특정 TOE 의 보안 요구사항 명세 |
| SFR (Security Functional Requirements) | 보안 기능 요구사항 |
| SAR (Security Assurance Requirements) | 보안 보증 요구사항 |
| EAL (Evaluation Assurance Level) | 평가 보증 수준 (1~7) |

### §6-4. EAL 7단계

EAL 은 평가의 엄격성·신뢰성 수준을 1~7 단계로 나눈다.

| EAL | 의미 | 검증 수준 |
|---|---|---|
| EAL1 | 기능 시험 (Functionally Tested) | 최소한의 기능 시험 |
| EAL2 | 구조 시험 (Structurally Tested) | 설계 정보 검토 + 시험 |
| EAL3 | 체계적 시험 및 점검 (Methodically Tested and Checked) | 개발 환경 검토 추가 |
| EAL4 | 체계적 설계·시험·검토 (Methodically Designed, Tested and Reviewed) | 상용 제품의 표준 수준 — **상호인정 협정 (CCRA) 의 일반적 상한** |
| EAL5 | 준 정형적 설계 및 시험 (Semiformally Designed and Tested) | 준 정형 명세 |
| EAL6 | 준 정형적 검증 설계 및 시험 (Semiformally Verified Design and Tested) | 준 정형 검증 |
| EAL7 | 정형적 검증 설계 및 시험 (Formally Verified Design and Tested) | 정형 명세·검증 — 최고 보증 수준 |

> *상호인정*: CCRA (Common Criteria Recognition Arrangement) 회원국 간 인정 범위는 일반적으로 **EAL2 까지** 이며, 협정 부속서에 따라 cPP (Collaborative Protection Profile) 기반은 별도 인정. 작성 시점에 CCRA 공식 자료 [^ccra] 에서 현행 인정 범위 확인.

### §6-5. 한국 CC 인증

한국에서는 IT 보안인증사무국 (ITSCC) 이 운영하는 **K-CC (한국 공통평가기준 인증제도)** 가 있으며, 국가용 보안제품에 대해 별도 인증 (KCMVP·국정원 보안적합성 검증 등) 을 운영한다.

| 인증 | 운영기관 | 대상 |
|---|---|---|
| K-CC | IT 보안인증사무국 (ITSCC), 국가정보원 | 국가·공공기관 도입 보안제품 |
| KCMVP | KISA, 국가정보원 | 암호모듈 |
| 국가용 보안제품 적합성 검증 | 국가정보원 | 국가·공공기관 도입 |

### §6-6. 시험 포인트

- CC = **ISO/IEC 15408**.
- EAL = **1~7 단계**.
- EAL4 = 상용 제품 표준 / EAL7 = 정형 검증 (최고 수준).
- TOE = 평가 대상, PP = 보호 프로파일, ST = 보안 목표서.
- CCRA 일반적 상호인정 = **EAL2** 까지.

## §7. KCMVP — 암호모듈 검증

> **KCMVP (Korea Cryptographic Module Validation Program)** : 국가·공공기관에서 사용되는 **암호모듈** 의 안전성과 구현 적합성을 검증하기 위한 한국의 인증제도.[^kcmvp]

### §7-1. KCMVP 의 법적 근거

KCMVP 는 [전자정부법] §56 [^egov-act] 및 「전자정부법 시행령」, 「국가정보 보안기본지침」 (국가정보원) 에 근거한다. 국가·공공기관은 검증된 암호모듈 (KCMVP 인증) 을 의무 사용해야 한다.

### §7-2. KCMVP 검증 대상

| 검증 대상 | 사례 |
|---|---|
| 암호 알고리즘 | SEED, ARIA, LEA, HIGHT, LSH, HAS-160 (4과목 05·06 cross-ref) |
| 암호모듈 구현 | 소프트웨어 / 하드웨어 / 펌웨어 형태의 암호 구현 |
| 키 관리 | 키 생성·분배·저장·폐기 |

### §7-3. KCMVP 검증 수준

KCMVP 검증 수준은 **L1·L2·L3·L4** 4단계이며, 단계가 올라갈수록 물리적·논리적 보호 요구사항이 강화된다 (FIPS 140-3 [^fips140-3] 와 유사한 구조).

| 수준 | 보호 수준 | 적용 대상 |
|---|---|---|
| L1 | 최소 보안 (소프트웨어 구현 가능) | 일반 시스템 |
| L2 | 역할 기반 인증, 변조 흔적 (Tamper Evident) | 일반 시스템·주요 정보 |
| L3 | 신원 기반 인증, 변조 저항 (Tamper Resistant) | 중요 정보 시스템 |
| L4 | 변조 탐지·자동 응답, 환경 공격 방어 | 최고 등급 보안 |

### §7-4. KCMVP vs FIPS 140-3 비교

| 비교 항목 | KCMVP | FIPS 140-3 |
|---|---|---|
| 운영 국가 | 한국 | 미국 |
| 운영기관 | KISA, 국가정보원 | NIST, CCCS |
| 인증 알고리즘 | SEED·ARIA·LEA 등 국산 + AES·SHA 등 국제 | AES·SHA 등 NIST 표준 |
| 검증 수준 | L1~L4 | Level 1~4 |
| 법적 근거 | 전자정부법 §56 | FISMA |

KCMVP 인증과 FIPS 140-3 인증은 **별개 체계** 이며, 한 모듈이 두 인증을 모두 받을 수도 있다.

### §7-5. 시험 포인트

- KCMVP = 암호모듈 검증 (한국).
- 국가·공공기관은 KCMVP 검증된 모듈 사용 **의무** ([전자정부법] §56).
- KCMVP 검증 수준 = **L1~L4**.
- KCMVP 와 FIPS 140-3 = 별개 체계.
- 검증 알고리즘에 SEED·ARIA·LEA·LSH 등 국산 알고리즘 포함.

## §8. 기타 정보보호 인증·평가

### §8-1. 인증·평가 비교표

| 인증 | 대상 | 운영기관 | 표준·법적 근거 |
|---|---|---|---|
| ISMS-P | 정보보호 + 개인정보보호 관리체계 | KISA | 정통망법 §47, 개보법 §32-2 |
| ISO/IEC 27001 | 정보보호 관리체계 (국제) | 국제 인증기관 | ISO/IEC 27001:2022 |
| ISO/IEC 27701 (PIMS) | 개인정보보호 관리체계 (국제) | 국제 인증기관 | ISO/IEC 27701:2019 |
| CC (K-CC) | 정보보호 제품 보안성 | ITSCC, 국가정보원 | ISO/IEC 15408 |
| KCMVP | 암호모듈 | KISA, 국가정보원 | 전자정부법 §56 |
| GS 인증 | 소프트웨어 품질 (보안 외 기능·신뢰성·효율성) | TTA | 산업표준화법 |
| PIA | 개인정보 처리시스템 영향평가 | 한국인터넷진흥원 등 평가기관 | 개보법 §33 |
| 보안적합성 검증 | 국가·공공기관 도입 보안제품 | 국가정보원 | 국가정보 보안기본지침 |

### §8-2. GS 인증 vs CC 인증 차이

| 비교 항목 | GS (Good Software) 인증 | CC 인증 |
|---|---|---|
| 평가 대상 | 소프트웨어 (보안 + 기능·신뢰성·효율성·이식성) | 정보보호 제품 |
| 평가 기준 | ISO/IEC 25051 (소프트웨어 품질 요구사항·시험) | ISO/IEC 15408 (보안 요구사항) |
| 운영기관 | 한국정보통신기술협회 (TTA) | ITSCC (한국), 인정 평가기관 (해외) |
| 보증 수준 | 등급 없음 (적합/부적합) | EAL 1~7 |

### §8-3. 시험 포인트

- ISMS-P / ISO 27001 / CC / KCMVP / PIA / GS 의 대상·기관·법적 근거 구분.
- ISO 27701 = PIMS = ISO 27001 의 개인정보보호 확장.
- GS 인증은 **소프트웨어 품질** 인증 (보안 외 영역 포함).

## §9. PIA — 개인정보 영향평가

> **PIA (Privacy Impact Assessment, 개인정보 영향평가)** : 개인정보를 처리하는 정보시스템의 도입·변경 시 **개인정보의 침해 위험성을 사전에 분석·평가** 하여 침해 위험을 최소화하기 위한 제도.[^pipa]

### §9-1. 법적 근거

[개인정보보호법] §33 [^pipa] 은 일정 규모 이상의 개인정보 처리시스템 도입·변경 시 PIA 의무를 부과한다.

### §9-2. PIA 의무대상 (공공기관)

[개인정보보호법 시행령] §35 [^pipa] 의 의무대상 (공공기관 중심):

| 분류 | 기준 |
|---|---|
| 5만 명 이상의 민감정보·고유식별정보 처리 | – |
| 50만 명 이상의 개인정보와 연계되는 시스템 | – |
| 100만 명 이상의 개인정보를 처리하는 시스템 | – |

> *주의*: 위 수치는 시행령 §35 의 현행본 기준이며, 정기 개정으로 변동될 수 있다. 작성 시점에 국가법령정보센터에서 직접 확인.

민간 기업은 PIA 가 의무는 아니나, 자율적으로 수행할 수 있다.

### §9-3. PIA 절차

KISA 「개인정보 영향평가 수행 안내서」 [^kisa-pia] 의 표준 절차:

| 단계 | 활동 |
|---|---|
| 1. 사전 준비 | 평가팀 구성, 평가 범위·일정 결정 |
| 2. 영향평가 수행 | 개인정보 흐름 분석, 침해 요인 분석, 위험 평가, 개선 계획 수립 |
| 3. 이행 점검 | 개선사항 이행 모니터링 |

### §9-4. 시험 포인트

- PIA 법적 근거 = **개보법 §33**.
- PIA 의무대상 = **공공기관** (시행령 §35 의 일정 규모 이상). 민간은 자율.
- PIA 의 핵심 산출물 = 개인정보 흐름도, 위험 평가, 개선 계획.

## §10. 단원 요약

| 영역 | 핵심 |
|---|---|
| ISMS-P | 정통망법 §47 + 개보법 §32-2 통합 인증 (2018.11 출범, KISA 인증) |
| ISMS-P 인증기준 | 3개 영역 — 관리체계 / 보호대책 / 개인정보 처리 단계별 |
| 인증 유효기간 | 3년 / 사후관리 연 1회 |
| ISO 27001 | 국제 ISMS 표준 (2022.10 개정, Annex A 93개 통제) |
| CC | ISO/IEC 15408, EAL 1~7 단계, CCRA 일반 인정 = EAL2 |
| KCMVP | 한국 암호모듈 검증 ([전자정부법] §56), L1~L4 |
| PIA | [개보법] §33, 공공기관 의무 (시행령 §35 기준) |

## §11. 시험 함정 정리

| 함정 | 정답 |
|---|---|
| ISMS-P 는 정보보호 관리체계만 인증 | **틀림**. 정보보호 + 개인정보보호 통합 인증 |
| ISMS-P 의 법적 근거는 개인정보보호법만 | **틀림**. 정통망법 §47 + 개보법 §32-2 |
| ISMS 의무인증 = 모든 정보통신서비스 제공자 | **틀림**. **일정 규모 이상** (시행령 §49) |
| 인증 유효기간은 5년 | **틀림**. **3년** |
| 사후관리 심사는 격년 | **틀림**. **연 1회** |
| ISO/IEC 27001:2022 통제 수는 114개 | **틀림**. **93개**. 27001:2013 = 114개 |
| ISO/IEC 27701 = ISMS | **틀림**. **PIMS** (개인정보보호 관리체계) |
| EAL 은 1~5 단계 | **틀림**. **1~7 단계** |
| CC 의 표준 번호는 ISO/IEC 27001 | **틀림**. **ISO/IEC 15408** |
| KCMVP 와 FIPS 140-3 은 동일 인증 | **틀림**. **별개 체계** (한국·미국) |
| KCMVP 검증 수준은 1~3 | **틀림**. **L1~L4** |
| PIA 는 모든 기업의 의무 | **틀림**. **공공기관** 의 일정 규모 이상 의무 (개보법 §33). 민간은 자율 |
| GS 인증 = 보안 인증 | **틀림**. GS = **소프트웨어 품질** 인증 (보안 + 기능·신뢰성·효율성 등 종합) |
| ISMS 와 ISMS-P 는 동일 인증 | **틀림**. ISMS = 영역 1+2 / ISMS-P = 영역 1+2+3 |
| CC 의 평가 대상은 ST | **틀림**. **TOE** (Target of Evaluation). ST 는 보안 목표서 |
