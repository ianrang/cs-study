# 5과목 04 — 침해사고 대응 및 디지털 포렌식

> 출제기준 매핑: 1.3 정보보호 대책 구현 및 사고대응 (5) — 정보보호 사고대응 (디지털 포렌식 등)

## §0. 들어가며

본 단원은 침해사고 발생 전·중·후의 체계적 대응과 디지털 포렌식의 원칙·절차를 다룬다. 침해사고는 발생 자체를 완전히 막을 수 없으므로 (잔여 위험), 효과적인 **대응 절차** 와 사고 후 **법적 증거 수집** 능력이 정보보호의 마지막 방어선이다.

본 단원은 NIST SP 800-61 Rev. 2 (Computer Security Incident Handling Guide) [^nist800-61], NIST SP 800-86 (Guide to Integrating Forensic Techniques into Incident Response) [^nist800-86], ISO/IEC 27035-1:2023 (Incident management — Principles) [^iso27035-1], ISO/IEC 27037:2012 (Digital Evidence Identification and Collection) [^iso27037], ISO/IEC 27042:2015 (Digital Evidence Analysis and Interpretation) [^iso27042], 그리고 한국 [정보통신망법] §48-3 (침해사고의 신고) [^infonet-act], [정보통신기반보호법] §13 (침해사고의 통지) [^cii-act] 을 1차 출처로 한다.

학습 순서:
1. 침해사고의 정의·법적 근거 (§1)
2. 침해사고 대응 절차 — 한국 7단계·NIST 4단계 (§2·§3)
3. 침해사고 신고 의무 — 정통망법·기반보호법 (§4)
4. 디지털 포렌식 — 정의·분야·원칙 (§5·§6·§7)
5. 디지털 포렌식 5단계 절차 (§8)
6. 안티 포렌식 (§9)
7. 사회공학 (§10)

## §1. 침해사고의 정의

> **침해사고 (Information Security Incident)** : [정보통신망 이용촉진 및 정보보호 등에 관한 법률] §2 제1항 제7호 [^infonet-act] 의 정의에 따라 "해킹·컴퓨터바이러스·논리폭탄·메일폭탄·서비스 거부 또는 고출력 전자기파 등의 방법으로 정보통신망 또는 이와 관련된 정보시스템을 공격하는 행위로 인하여 발생한 사태."

### §1-1. 한국 법령상 침해사고 정의

| 법령 | 조문 | 정의 |
|---|---|---|
| 정보통신망법 §2 ①항 7호 [^infonet-act] | 침해사고 | 해킹·바이러스·논리폭탄·메일폭탄·DoS·전자기파 등 전자적 침해행위로 인한 사태 |
| 정보통신기반보호법 §2 [^cii-act] | 전자적 침해행위 | 해킹·바이러스·논리폭탄 등으로 정보통신기반시설을 공격하여 정보·시스템에 영향을 미치는 행위 |
| 개인정보보호법 §34 [^pipa] | 개인정보 유출 등 | 개인정보가 분실·도난·유출·위변조·훼손되는 사고 |

### §1-2. NIST SP 800-61 침해사고 분류

NIST SP 800-61 Rev. 2 §2.1 [^nist800-61] 은 침해사고를 다음 카테고리로 분류한다.

| 카테고리 | 사례 |
|---|---|
| 외부·이동매체 (External/Removable Media) | 감염된 USB·CD 를 통한 악성코드 침투 |
| 침투 (Attrition) | 무차별 공격 — DDoS·BoF·BAT·Brute-force |
| 웹 (Web) | 웹 어플리케이션을 통한 공격 — XSS·SQLi |
| 이메일 (Email) | 피싱·악성첨부파일·악성링크 |
| 부적절한 사용 (Improper Usage) | 정책 위반·내부자 악의 |
| 분실·도난 장비 (Loss/Theft) | 노트북·이동매체 분실 |
| 기타 | 위 분류에 속하지 않는 사고 |

### §1-3. 시험 포인트

- 한국 법령상 침해사고 정의는 [정보통신망법] §2 ①항 **7호** 에 명시.
- "전자적 침해행위" 용어는 [정보통신기반보호법] §2 에서 사용.
- 개인정보 유출 사고는 [개인정보보호법] §34 별도 조항.

## §2. 침해사고 대응 절차 — 한국 7단계

KISA 침해사고 대응 매뉴얼 [^kisa-incident] 과 5과목.txt [^isms-def] 의 표준 절차는 다음 7단계로 구성된다.

### §2-1. 7단계 절차

| 단계 | 활동 | 산출물 |
|---|---|---|
| 1. **사고 전 준비** | 정책·조직·도구·교육·훈련 | 사고대응 정책서, CIRT 조직, 비상연락망 |
| 2. **사고 탐지** | 보안장비·로그·신고 등으로 사고 인지 | 사고 인지 보고 |
| 3. **초기 대응** | 사고 사실 확인·증거 보존·임시 조치 | 초기 대응 보고서 |
| 4. **대응 전략 체계화** | 사고 영향 평가·대응 우선순위·조직 동원 결정 | 대응 전략서 |
| 5. **사고 조사** | 원인 분석·피해 범위 파악·증거 수집 | 사고 조사 보고서 |
| 6. **보고서 작성** | 사고 경위·원인·대응·재발방지 대책 | 침해사고 종합 보고서 |
| 7. **해결** | 시스템 복구·개선 조치·재발 방지·교훈 도출 (Lessons Learned) | 시정조치 결과서 |

### §2-2. 단계별 핵심 활동

| 단계 | 핵심 활동 | 시간 민감도 |
|---|---|---|
| 사고 전 준비 | CIRT 구성, 도구 (포렌식·로그분석·통신) 준비, 훈련 | 평시 |
| 사고 탐지 | 알람 검증, 사고 vs 오탐 구분, 신고 접수 | 즉시 |
| 초기 대응 | **증거 보존 우선** — 임의 종료·재부팅 금지 | 1시간 이내 권장 |
| 대응 전략 체계화 | 영향 분석·격리 vs 모니터링 결정·외부 협력 결정 | 24시간 이내 |
| 사고 조사 | 침해 경로·범위·잔존 위협 파악 | 사고 규모에 따라 |
| 보고서 작성 | 경영진·당국 보고 자료 | 법정 신고 의무에 따라 |
| 해결 | 시스템 복구·취약점 제거·정책 개선 | 잔존 위협 제거 후 |

## §3. NIST SP 800-61 Rev. 2 — 4단계 절차

NIST SP 800-61 Rev. 2 §3 [^nist800-61] 는 침해사고 대응을 4단계 사이클로 정의한다. 한국 7단계와 본질적으로 동일하나, 상위 단계로 단순화되어 있다.

### §3-1. NIST 4단계

| 단계 | 한국 7단계 매핑 | 활동 |
|---|---|---|
| **Preparation** (준비) | 1단계 | 정책·조직·도구·교육·훈련. 평시 활동 |
| **Detection and Analysis** (탐지 및 분석) | 2·3·5단계 | 사고 탐지·검증·우선순위·범위 파악 |
| **Containment, Eradication, and Recovery** (격리·제거·복구) | 4·7단계 | 사고 격리·악성코드 제거·시스템 복구 |
| **Post-Incident Activity** (사후 활동) | 6·7단계 | 보고서·교훈·정책 개선·증거 보관 |

### §3-2. NIST Containment 전략 — 단기 vs 장기

NIST SP 800-61 Rev. 2 §3.3.1 [^nist800-61] 은 격리 전략을 다음 두 측면으로 구분한다.

| 격리 유형 | 목적 | 사례 |
|---|---|---|
| 단기 격리 (Short-term Containment) | 즉시 피해 확산 차단 | 네트워크 분리·계정 잠금·서비스 차단 |
| 장기 격리 (Long-term Containment) | 시스템 복구 전까지 운영 유지 | 임시 패치·우회 경로·격리 환경 운영 |

### §3-3. 사후 활동 — 교훈 도출

NIST SP 800-61 Rev. 2 §3.4.1 [^nist800-61] 은 사후 활동의 핵심으로 **Lessons Learned 회의** 를 명시한다. 회의는 다음 질문에 답해야 한다:

- 정확히 무슨 일이 일어났는가? 시간순으로 정리.
- 직원과 관리진은 사고에 얼마나 잘 대응했는가? 정책·절차가 따르졌는가?
- 다음 번에 다른 행동을 해야 할 것은 무엇인가?
- 시정 조치는 무엇이며 누가 책임지는가?
- 향후 유사 사고를 어떻게 방지할 것인가?
- 추가 도구·자원이 필요한가?

### §3-4. 시험 포인트

- NIST 4단계 = **Preparation → Detection&Analysis → Containment, Eradication&Recovery → Post-Incident Activity**.
- 한국 7단계 = **사고 전 준비 → 탐지 → 초기대응 → 전략 체계화 → 조사 → 보고서 → 해결**.
- 초기 대응의 최우선 = **증거 보존** (임의 재부팅·종료 금지).
- Containment = 격리. Eradication = 악성코드 제거. Recovery = 복구.
- Post-Incident Activity 의 핵심 = **Lessons Learned**.

## §4. 침해사고 신고 의무

### §4-1. 정보통신망법 §48-3 — 침해사고 신고

[정보통신망 이용촉진 및 정보보호 등에 관한 법률] §48-3 [^infonet-act] 은 정보통신서비스 제공자에게 침해사고 발생 시 신고 의무를 부과한다.

| 항목 | 내용 |
|---|---|
| 의무자 | 정보통신서비스 제공자 |
| 신고 사유 | 침해사고 발생 사실의 인지 |
| 신고 대상 | 과학기술정보통신부 장관 또는 한국인터넷진흥원 (KISA) |
| 신고 방법 | KISA 인터넷침해사고대응센터 (KrCERT/CC) 신고 채널 |
| 신고 시기 | "지체 없이" — 시행령 별도 명시 사항 (조문 본문에 구체적 시간 미명시 — 작성 시점에 시행령 직접 확인 필요) |
| 비신고 시 처벌 | 정보통신망법 §76 의 과태료 부과 사유에 해당할 수 있음 — 처분 기준은 시행령 |

### §4-2. 정보통신기반보호법 §13 — 침해사고 통지

[정보통신기반 보호법] §13 [^cii-act] 은 주요정보통신기반시설 관리기관에 침해사고 통지 의무를 부과한다.

| 항목 | 내용 |
|---|---|
| 의무자 | 주요정보통신기반시설 관리기관 |
| 통지 사유 | 전자적 침해행위 발생 인지 |
| 통지 대상 | 관계 중앙행정기관의 장, 수사기관, KISA |
| 통지 시기 | "지체 없이" |

### §4-3. 개인정보보호법 §34 — 개인정보 유출 통지·신고

[개인정보보호법] §34 [^pipa] 는 개인정보 유출 사고 시 통지·신고 의무를 부과한다 (5과목 08 §9 에서 상세 다룸).

| 항목 | 내용 |
|---|---|
| 의무자 | 개인정보처리자 |
| 통지 대상 | **정보주체** (유출된 개인정보의 주인) |
| 신고 대상 | **개인정보보호위원회** 또는 한국인터넷진흥원 |
| 통지·신고 시기 | "지체 없이" — 1,000명 이상 또는 민감정보·고유식별정보 유출 시 신고 의무 |
| 통지 내용 | 유출 항목·시점·경위·피해 최소화 방법·구제 절차·담당부서 연락처 |

> *주의*: 위 시기·기준은 작성 시점 (2026.04.30) 의 [개인정보보호법] 및 시행령에 따른다. 법령은 정기 개정되므로 시험 응시 시점에 국가법령정보센터에서 현행본 직접 확인이 필요하다.

### §4-4. 침해사고 신고 vs 개인정보 유출 신고 비교

| 비교 항목 | 침해사고 신고 (정통망법 §48-3) | 개인정보 유출 신고 (개보법 §34) |
|---|---|---|
| 대상 사고 | 해킹·DDoS·악성코드 등 정보통신망 침해 | 개인정보의 분실·도난·유출·위변조·훼손 |
| 신고 대상 | 과기정통부·KISA | 개인정보보호위원회 또는 KISA |
| 통지 대상 | (없음 — 신고만) | **정보주체** 통지 의무 |
| 시간 기준 | 지체 없이 | 지체 없이 (특정 규모 이상 신고) |
| 법적 근거 | 정보통신망법 §48-3 | 개인정보보호법 §34 |

### §4-5. 시험 포인트

- 침해사고 신고 의무 = **정통망법 §48-3** (KISA 신고).
- 개인정보 유출 통지 의무 = **개보법 §34** (정보주체 + 위원회 신고).
- 정보통신기반시설의 침해 통지 = **기반보호법 §13**.
- 개인정보 유출 시에는 **정보주체에게 직접 통지** 의무가 추가됨.

## §5. 디지털 포렌식의 정의와 분야

> **디지털 포렌식 (Digital Forensics)** : 침해사고 발생 시 법적으로 유효한 증거를 디지털 기기로부터 **수집·분석·보관·제출** 하는 등의 일련의 과정을 다루는 과학 분야.[^isms-def]

ISO/IEC 27037:2012 [^iso27037] 는 디지털 포렌식을 "디지털 증거의 식별·수집·획득·보존" 으로 정의한다. 본 분야는 단순한 기술 활동이 아니라 **법정 증거능력** 을 보장하기 위한 엄격한 과학적·법적 절차이다.

### §5-1. 디지털 포렌식의 분야

| 분야 | 대상 | 핵심 기술 |
|---|---|---|
| **디스크 포렌식** | HDD·SSD·이동매체 | 이미지 획득 (DD·EnCase·FTK), 파일 시스템 분석, 삭제 파일 복원 |
| **휘발성 데이터 포렌식 (라이브 포렌식)** | RAM·실행 중 프로세스 | 메모리 덤프, 프로세스·네트워크 연결 분석 |
| **네트워크 포렌식** | 네트워크 트래픽 | 패킷 캡처 (Wireshark), 트래픽 분석, 통신 흐름 재구성 |
| **이메일 포렌식** | 이메일 서버·클라이언트 | 헤더 분석, 첨부파일 추출, 발신자 추적 |
| **웹 포렌식** | 브라우저 로그·캐시·쿠키 | 브라우저 사용 이력 분석, 캐시 복원 |
| **모바일 포렌식** | 스마트폰·태블릿 | 물리적·논리적 추출, 앱 데이터 분석 |
| **멀티미디어 포렌식** | CCTV·블랙박스·카메라 | 영상 분석, 화질 개선, 위변조 검증 |
| **데이터베이스 포렌식** | DB 서버 | 트랜잭션 로그, 변경 이력, 삭제 데이터 |

### §5-2. 휘발성 vs 비휘발성 — 수집 우선순위

증거 수집의 우선순위는 **휘발성이 높은 순** 으로 진행한다. RFC 3227 (Guidelines for Evidence Collection and Archiving) [^rfc3227] 의 Order of Volatility:

| 순위 | 데이터 유형 | 수명 |
|---|---|---|
| 1 | 레지스터, 캐시 | 나노초 |
| 2 | 라우팅 테이블, ARP 캐시, 프로세스 테이블, 커널 통계 | 마이크로초~밀리초 |
| 3 | 메모리 (RAM) | 시스템 종료 시 손실 |
| 4 | 임시 파일 시스템 (tmpfs, /tmp) | 재부팅 시 손실 |
| 5 | 디스크 | 영구 (단 와이핑·덮어쓰기로 손실) |
| 6 | 원격 로깅·모니터링 데이터 | 보존 정책에 따라 |
| 7 | 물리 구성·네트워크 토폴로지 | 영구 |
| 8 | 보관용 매체 | 영구 |

> *원칙*: 휘발성이 높은 데이터를 먼저 수집해야 한다. 디스크 이미지보다 메모리 덤프가 우선.

### §5-3. 시험 포인트

- 디지털 포렌식의 핵심 목적 = **법정 증거능력 보장**.
- 휘발성 우선 수집 (RFC 3227) — RAM 이 디스크보다 우선.
- 라이브 포렌식 = **휘발성 데이터** 포렌식.
- CCTV·블랙박스 = **멀티미디어 포렌식**.

## §6. 디지털 포렌식 5대 원칙

> 5과목.txt [^isms-def] 의 표준 5대 원칙. 모든 한국 디지털 증거 수집 절차의 기본이다.

### §6-1. 5대 원칙

| 원칙 | 정의 |
|---|---|
| **정당성의 원칙** (Legitimacy) | 증거 수집·분석 절차가 적법해야 한다. 영장 없이 수집한 증거는 증거능력을 상실 |
| **재현성의 원칙** (Reproducibility) | 동일한 조건에서 동일한 결과가 도출되어야 한다 |
| **신속성의 원칙** (Promptness) | 휘발성 데이터의 손실을 막기 위해 신속히 수행 |
| **연계보관성의 원칙** (Chain of Custody) | 법정 제출까지 증거물 관리 주체들 간의 **연속적인 승계 내역** 을 기록함으로써 디지털 증거가 최초 수집된 상태로 유지되어야 함 |
| **무결성의 원칙** (Integrity) | 수집된 증거가 변경되지 않았음을 입증 (해시값 비교) |

> *원본 표기 보정*: 5과목.txt line 199 의 "무결서의 원칙" 은 "무결성의 원칙" 의 OCR 오류이다.

### §6-2. 5대 원칙의 위반 시 결과

| 원칙 위반 | 결과 |
|---|---|
| 정당성 위반 | 증거능력 상실 (위법수집 증거배제) — 형사소송법 §308-2 |
| 재현성 위반 | 분석 결과 신뢰성 의심 |
| 신속성 위반 | 휘발성 데이터 손실 — 증거 가치 저하 |
| 연계보관성 위반 | 증거 변조·오염 가능성 — 증거능력 상실 |
| 무결성 위반 | 증거 변경 — 증거능력 상실 |

### §6-3. 연계보관성 (Chain of Custody) 의 기록 항목

ISO/IEC 27037:2012 §6.4 [^iso27037] 와 NIST SP 800-86 §6.2 [^nist800-86] 는 연계보관성 기록에 다음을 요구한다.

- 수집 일시 / 장소
- 수집자 (이름·소속)
- 수집 방법 / 도구
- 증거의 식별 정보 (시리얼 번호·해시값)
- 보관·이동의 모든 변경 이력 (수령자·일시·장소)
- 변경의 사유

### §6-4. 시험 포인트

- 5대 원칙 = **정당성·재현성·신속성·연계보관성·무결성**.
- 위법 수집 증거 = **정당성 위반** → 증거능력 상실 (형사소송법 §308-2).
- 해시값 비교 = **무결성 입증**.
- Chain of Custody = **연계보관성**.

## §7. 한국 법령상 디지털 증거의 증거능력

### §7-1. 형사소송법 — 위법수집 증거배제 원칙

[형사소송법] §308-2 [^crim-proc] 는 "적법한 절차에 따르지 아니하고 수집한 증거는 증거로 할 수 없다" 고 규정한다. 디지털 증거도 동일하게 적용되어 정당성의 원칙 위반 시 증거능력이 부정된다.

### §7-2. 디지털 증거의 진정성 입증

대법원 판례 [^supreme-court-ev] 에 따르면 디지털 증거의 진정성은 다음 요건의 종합적 판단으로 입증된다.

- 무결성 (해시값 일치)
- 동일성 (원본과 사본의 동일성)
- 신뢰성 (수집·분석 절차의 신뢰성)
- 원본 또는 그와 동일성·무결성이 보장된 사본

### §7-3. 시험 포인트

- 위법수집 증거배제 = 형사소송법 **§308-2**.
- 디지털 증거의 진정성 입증 = 무결성 + 동일성 + 신뢰성.

## §8. 디지털 포렌식 5단계 절차

### §8-1. 5단계 절차

| 단계 | 활동 |
|---|---|
| **1. 사전 준비** | 도구·교육·법적 권한 확보. 평시 활동 |
| **2. 증거 수집** | 현장 도착·식별·획득·해시 산출 |
| **3. 보관 및 이송** | 봉인·운반·보관소 입고. 연계보관성 기록 |
| **4. 조사 및 분석** | 사본에 대해 분석. 원본은 원본대로 보존 |
| **5. 보고서 작성** | 객관적·재현 가능한 보고서. 분석 도구·방법 명시 |

### §8-2. ISO/IEC 27037 vs ISO/IEC 27042 분담

ISO/IEC 27037:2012 [^iso27037] 는 1·2·3 단계 (식별·수집·획득·보존), ISO/IEC 27042:2015 [^iso27042] 는 4·5 단계 (분석·해석) 를 다룬다.

| 단계 | 표준 |
|---|---|
| 식별·수집·획득·보존 | ISO/IEC 27037:2012 |
| 분석·해석 | ISO/IEC 27042:2015 |
| 사고 조사 원칙 | ISO/IEC 27043:2015 |

### §8-3. 증거 수집의 원칙 — 원본 보존

- **원본 우선 보존**: 원본 매체에 직접 접근·분석 금지.
- **이미지 사본 분석**: 비트 단위 (bit-by-bit) 사본 (이미지 파일) 을 만들어 분석.
- **해시값 검증**: 원본 → 이미지 변환 시·분석 전후로 해시값 (SHA-256 등) 으로 무결성 검증.
- **쓰기 방지 (Write Blocker) 사용**: 원본 매체는 쓰기 방지 장치를 거쳐 접근.

### §8-4. 시험 포인트

- 5단계 = **사전 준비 → 증거 수집 → 보관 및 이송 → 조사 및 분석 → 보고서 작성**.
- 분석은 **사본** 에 대해 수행 (원본 보존).
- ISO/IEC 27037 = 수집·보존 / 27042 = 분석·해석.
- 쓰기 방지 (Write Blocker) = 원본 보존을 위한 필수 장치.

## §9. 안티 포렌식

> **안티 포렌식 (Anti-Forensics)** : 디지털 데이터를 조작·삭제·난독화하여 디지털 포렌식을 방해하는 행위.[^isms-def]

### §9-1. 안티 포렌식 기법

| 분류 | 기법 | 설명 |
|---|---|---|
| **데이터 은폐** | 데이터 암호화 | 증거 데이터를 암호화하여 분석 방해 |
| | 스테가노그래피 | 기밀정보를 미디어 파일 (이미지·오디오) 안에 숨기는 기술 |
| | 파일 위장 | 확장자 변경, 매직 넘버 조작 |
| **데이터 파괴** | 디스크 와이핑 | 디스크 전체를 무작위 데이터로 덮어쓰기 (DoD 5220.22-M, NIST SP 800-88 [^nist800-88]) |
| | 디스크 디가우징 | 강한 자기장으로 디스크 자기 신호 소거 (HDD 한정) |
| | 물리적 파괴 | 드릴·파쇄·소각 |

### §9-2. NIST SP 800-88 — 매체 정보 위생

NIST SP 800-88 Rev. 1 (Guidelines for Media Sanitization) [^nist800-88] 는 매체 정보 위생을 3 가지 수준으로 분류한다.

| 수준 | 방법 | 회복 가능성 |
|---|---|---|
| Clear | 일반 소거 (포맷·삭제) | 일반 도구로 회복 가능 |
| Purge | 와이핑·디가우징 | 전문 도구로도 회복 어려움 |
| Destroy | 물리적 파괴 | 회복 불가능 |

### §9-3. 시험 포인트

- 안티 포렌식 = **은폐 + 파괴**.
- 스테가노그래피 = 미디어 안에 정보 숨김 (암호화 ≠ 스테가노).
- 디가우징은 **HDD 에만 적용 가능** (SSD 는 자기 매체가 아님).
- NIST SP 800-88 등급 = **Clear < Purge < Destroy**.

## §10. 사회공학 — 침해사고의 인적 측면

> **사회공학 (Social Engineering)** : 사람들 사이의 신뢰관계를 바탕으로 사람들을 속여 정상적인 보안 절차를 무너뜨리고 비기술적인 수단으로 정보를 탈취하는 행위.[^isms-def]

### §10-1. 사회공학 공격 유형

| 공격 | 설명 | 매체 |
|---|---|---|
| 피싱 (Phishing) | 정상 기관을 가장한 이메일·웹사이트로 정보 탈취 | 이메일·웹 |
| 스피어 피싱 (Spear Phishing) | 특정 개인·조직을 표적으로 한 정교한 피싱 | 이메일 |
| 보이스 피싱 (Vishing) | 전화로 정보 탈취 | 전화 |
| 스미싱 (Smishing) | SMS·MMS 로 정보 탈취 | 문자 |
| **파밍 (Pharming)** | 희생자가 정상적인 도메인 주소를 입력하여 접속해도 **가짜 사이트로 접속이 유도** 되어 개인정보·금융정보 등을 탈취하는 형태의 공격 | DNS·hosts 파일 변조 |
| 베이팅 (Baiting) | USB·이동매체에 악성코드를 심고 호기심 유발 | 물리적 매체 |
| 프리텍스팅 (Pretexting) | 가짜 시나리오로 신뢰 획득 | 전화·대면 |
| 어깨 너머 보기 (Shoulder Surfing) | 입력 화면을 어깨 너머로 관찰 | 물리적 |
| 휴지통 뒤지기 (Dumpster Diving) | 폐기 문서·매체에서 정보 수집 | 물리적 |

### §10-2. 파밍 vs 피싱

| 비교 항목 | 피싱 | 파밍 |
|---|---|---|
| 공격 방법 | 가짜 URL 을 메일·SMS 로 유포 | DNS·hosts 파일 변조로 정상 URL 입력 시에도 가짜 사이트 |
| 사용자 인지 | 가짜 URL 인지 가능 | **정상 URL 입력이므로 인지 어려움** |
| 대응 | URL 확인, 출처 확인 | DNS 보안 (DNSSEC), 호스트 파일 검증 |

### §10-3. 시험 포인트

- 파밍 = **DNS·hosts 변조**로 정상 URL 입력 시에도 가짜 사이트 (URL 변경 X).
- 피싱 = 가짜 URL 을 메일·SMS 로 유포 (URL 변경 O).
- 베이팅 = USB 미끼.
- 휴지통 뒤지기 (Dumpster Diving) = 폐기물 수집 — 종이 매체 파쇄로 대응.

## §12. 시험 함정 정리

| 함정 | 정답 |
|---|---|
| 침해사고는 [개인정보보호법] 에 정의되어 있다 | **틀림**. [정통망법] §2 ①항 7호 정의 |
| NIST SP 800-61 절차는 5단계이다 | **틀림**. 4단계 (Preparation·Detection&Analysis·Containment Eradication&Recovery·Post-Incident Activity) |
| 한국 침해사고 대응 절차는 5단계이다 | **틀림**. 7단계 |
| 침해사고 발생 시 즉시 시스템을 재부팅 | **틀림**. **증거 보존 우선** — 임의 종료·재부팅 금지 |
| 휘발성 데이터는 디스크보다 나중에 수집 | **틀림**. **휘발성 우선** (RFC 3227) — RAM 먼저 |
| 디지털 포렌식 5대 원칙은 정당성·재현성·신속성·증거력 | **틀림**. 정당성·재현성·신속성·연계보관성·**무결성** |
| 분석은 원본 매체에서 수행 | **틀림**. **사본** (이미지) 분석. 원본 보존 |
| 디가우징은 SSD 에도 효과적 | **틀림**. **HDD 한정** (자기 매체) |
| 스테가노그래피 = 암호화 | **틀림**. 스테가노 = 미디어 안에 **숨김** (암호화는 별개) |
| 파밍 = 가짜 URL 유포 | **틀림**. 파밍 = DNS·hosts 변조 (정상 URL 입력 시에도 가짜 접속) |
| 침해사고 신고 = 개인정보보호위원회 | **틀림**. 침해사고 신고 = KISA·과기정통부 ([정통망법] §48-3). 개인정보 유출 신고 = 위원회 ([개보법] §34) |
| 위법수집 증거배제는 형사소송법 §307 | **틀림**. **§308-2** |
