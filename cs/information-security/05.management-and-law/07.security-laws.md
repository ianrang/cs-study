# 5과목 07 — 정보보호 관련 법제

> 출제기준 매핑: 2.2 정보보호 관련 법제 — 정보보호 관련 법제 용어의 정의 / 정보통신망 이용촉진 및 정보보호 등에 관한 법률의 이해

## §0. 들어가며

본 단원은 한국의 정보보호 관련 법제 중 **개인정보보호법을 제외한** 4 가지 핵심 법률을 다룬다. 개인정보보호법은 별도 단원 (08) 에서 상세히 다룬다.

본 단원의 4 가지 법령:
1. **[정보통신망 이용촉진 및 정보보호 등에 관한 법률]** (정통망법) — 정보통신서비스 제공자 일반 규제
2. **[정보통신기반 보호법]** — 주요정보통신기반시설 보호
3. **[전자서명법]** — 전자서명·인정 전자서명 (2020 전부개정)
4. **[정보통신산업진흥법]** + **[통신비밀보호법]** (보충)

본 단원의 모든 1차 출처는 **국가법령정보센터 (https://www.law.go.kr/) 현행본 (2026.04.30 기준)** 이다. 시험 응시 시점에 법령 개정 여부를 직접 확인해야 한다.

학습 순서:
1. 정보보호 법제 용어의 정의 (§1)
2. 정보통신망법 — 목적·체계·핵심 조문 (§2)
3. 정보통신기반 보호법 (§3)
4. 전자서명법 (§4)
5. 정보통신산업진흥법·통신비밀보호법 (§5)

## §1. 정보보호 법제 용어의 정의

각 법령에서 사용되는 핵심 용어는 해당 법령의 **§2 (정의)** 에 명시되어 있다. 출제기준 2.2 (정보보호 관련 법제 용어의 정의) 에서 용어의 정확한 정의·구분이 출제 영역으로 명시된다.

### §1-1. [정보통신망법] §2 의 핵심 용어

[정보통신망 이용촉진 및 정보보호 등에 관한 법률] §2 [^infonet-act]:

| 용어 | 정의 (요약) |
|---|---|
| **정보통신망** | 「전기통신사업법」 제2조제2호에 따른 전기통신설비를 이용하거나 전기통신설비와 컴퓨터 및 컴퓨터의 이용기술을 활용하여 정보를 수집·가공·저장·검색·송신 또는 수신하는 정보통신체제 |
| **정보통신서비스** | 「전기통신사업법」 제2조제6호에 따른 전기통신역무와 이를 이용하여 정보를 제공하거나 정보의 제공을 매개하는 것 |
| **정보통신서비스 제공자** | 「전기통신사업법」 제2조제8호에 따른 전기통신사업자와 영리를 목적으로 전기통신사업자의 전기통신역무를 이용하여 정보를 제공하거나 정보의 제공을 매개하는 자 |
| **이용자** | 정보통신서비스 제공자가 제공하는 정보통신서비스를 이용하는 자 |
| **전자우편** | 컴퓨터 등 정보처리능력을 갖춘 장치를 이용하여 부호·문자·음성·화상 또는 영상 등을 송신·수신하는 정보통신체제 |
| **침해사고** | 해킹·컴퓨터바이러스·논리폭탄·메일폭탄·서비스 거부 또는 고출력 전자기파 등의 방법으로 정보통신망 또는 이와 관련된 정보시스템을 공격하는 행위로 인하여 발생한 사태 |

### §1-2. [정보통신기반 보호법] §2 의 핵심 용어

[정보통신기반 보호법] §2 [^cii-act]:

| 용어 | 정의 (요약) |
|---|---|
| **정보통신기반시설** | 국가안전보장·행정·국방·치안·금융·통신·운송·에너지 등의 업무와 관련된 전자적 제어·관리시스템 및 「정보통신망법」 §2 ①항 1호에 따른 정보통신망 |
| **주요정보통신기반시설** | 정보통신기반시설 중 §8 에 따라 지정된 시설 |
| **전자적 침해행위** | 다음 행위로 정보통신기반시설을 공격하여 정보를 유출·변조·훼손하거나 시스템·서비스의 정상적 운영을 방해하는 행위. (해킹·컴퓨터바이러스·서비스 거부·전자기파 등) |

### §1-3. [전자서명법] §2 의 핵심 용어 — 2020 전부개정 기준

[전자서명법] §2 [^esign-act] (2020.6.9 전부개정 법률 제17354호 / 2020.12.10 시행):

| 용어 | 정의 (요약) |
|---|---|
| **전자문서** | 정보처리시스템에 의하여 전자적 형태로 작성·송신·수신·저장된 정보 |
| **전자서명** | 다음 사항을 나타내기 위하여 전자문서에 첨부되거나 논리적으로 결합된 전자적 형태의 정보. (가) 서명자의 신원, (나) 서명자가 해당 전자문서에 서명하였다는 사실 |
| **전자서명생성정보** | 전자서명을 생성하기 위하여 이용하는 전자적 정보 (사설키 등) |
| **전자서명검증정보** | 전자서명을 검증하기 위하여 이용하는 전자적 정보 (공개키 등) |
| **전자서명수단** | 전자서명생성정보가 들어 있는 정보처리장치 또는 그러한 장치 (HSM·스마트카드 등) |
| **전자서명인증** | 전자서명생성정보가 가입자에게 유일하게 속한다는 사실을 확인·증명하는 행위 |
| **전자서명인증사업자** | 전자서명인증 업무를 하는 자 |
| **전자서명인증서비스** | 전자서명인증사업자가 가입자에게 제공하는 전자서명인증과 관련된 서비스 |

> *주의*: 2020 전부개정으로 **공인인증서 (구 전자서명법) 제도는 폐지** 되었으며, 다양한 전자서명 수단이 동등한 법적 효력을 가지게 되었다. 4과목 04 §6 (한국 PKI 운영) 과 cross-ref.

### §1-4. 시험 포인트

- 정보통신망 정의 = [정통망법] §2 ①항 1호 (전기통신설비 + 컴퓨터·이용기술).
- 정보통신서비스 제공자 = [정통망법] §2 ①항 3호 (전기통신사업자 + 영리 목적 매개자).
- 침해사고 = [정통망법] §2 ①항 7호 (해킹·바이러스·DoS 등).
- 주요정보통신기반시설 = [기반보호법] §2 + §8 에 따라 지정.
- 전자서명 정의 = [전자서명법] §2 (서명자 신원 + 서명 사실).

## §2. 정보통신망 이용촉진 및 정보보호 등에 관한 법률 (정통망법)

### §2-1. 법령 개요

| 항목 | 내용 |
|---|---|
| 정식 명칭 | 정보통신망 이용촉진 및 정보보호 등에 관한 법률 |
| 약칭 | 정보통신망법 (정통망법) |
| 최초 제정 | 1986년 (당시 「전산망 보급확장과 이용촉진에 관한 법률」) |
| 현행 명칭 변경 | 2001년 (정보통신망 이용촉진 및 정보보호 등에 관한 법률) |
| 소관 부처 | 과학기술정보통신부 / 방송통신위원회 (분야별) |
| 2020년 개정 | 데이터 3법 (2020.2.4) — 개인정보보호 조항 다수가 [개인정보보호법] 으로 이관 (2020.8.5 시행) |

### §2-2. 법의 목적 — §1

> "이 법은 정보통신망의 이용을 촉진하고 정보통신서비스를 이용하는 자의 개인정보를 보호함과 아울러 정보통신망을 건전하고 안전하게 이용할 수 있는 환경을 조성하여 국민생활의 향상과 공공복리의 증진에 이바지함을 목적으로 한다." [^infonet-act]

### §2-3. 정통망법의 구조 (장 단위)

| 장 | 제목 | 핵심 조문 |
|---|---|---|
| 제1장 | 총칙 | §1 목적, §2 정의 |
| 제2장 | 정보통신망의 이용촉진 | §3~§13 (이용촉진 시책·기술 개발) |
| 제3장 | (삭제, 2020 데이터 3법) | (구 개인정보 보호 — [개인정보보호법]으로 이관) |
| 제4장 | 정보통신망에서의 이용자 보호 등 | §44, §44-2, §44-7 (불법정보 유통 금지 등) |
| 제5장 | 정보통신망의 안정성 확보 등 | §45~§48-4 (정보보호조치·CISO·ISMS·침해사고 신고 등) |
| 제6장 | 보칙 | §51~§57 |
| 제7장 | 벌칙 | §70~§76 |

### §2-4. 핵심 조문 1 — §45 정보보호조치

[정통망법] §45 [^infonet-act] 는 정보통신서비스 제공자에게 다음 정보보호조치를 요구한다.

| 항 | 내용 |
|---|---|
| §45 ①항 | 정보통신서비스 제공자는 이용자의 개인정보 등이 분실·도난·누출·위조·변조 또는 훼손되지 아니하도록 안전성 확보에 필요한 보호조치를 강구할 것 |
| §45 ②항 | 일정 규모 이상의 정보통신서비스 제공자는 정보보호 지침에 따른 안전성 확보 |
| §45 ③항 | 과학기술정보통신부장관은 정보보호 지침을 정하여 고시 |

### §2-5. 핵심 조문 2 — §45-3 정보보호 최고책임자 (CISO)

[정통망법] §45-3 [^infonet-act] 는 CISO 지정 의무를 부과한다 (5과목 01 §7-1 cross-ref).

| 항 | 내용 |
|---|---|
| §45-3 ①항 | 일정 규모 이상의 정보통신서비스 제공자는 정보보호 최고책임자를 지정하고 과학기술정보통신부장관에게 신고 |
| §45-3 ②항 | CISO 의 자격·업무 |
| §45-3 ③항 | 일정 규모 이상의 사업자는 CISO 가 다른 직무 겸직 제한 |

### §2-6. 핵심 조문 3 — §47 정보보호 관리체계 인증 (ISMS)

[정통망법] §47 [^infonet-act] 은 ISMS 인증 제도를 규정한다 (5과목 05 §1·§3 cross-ref).

| 항 | 내용 |
|---|---|
| §47 ①항 | 과학기술정보통신부장관은 ISMS 인증 제도 운영 |
| §47 ②항 | 일정 규모 이상의 정보통신서비스 제공자는 ISMS 인증 의무 |
| §47 ③항 | 인증의 유효기간 = 3년 |
| §47 ④항~⑥항 | 인증기관·인증심사원·결과 통지 등 |

### §2-7. 핵심 조문 4 — §48 정보통신망 침해행위 등의 금지

[정통망법] §48 [^infonet-act] 의 3대 금지행위 (5과목 06 §5 cross-ref):

| 항 | 금지행위 |
|---|---|
| §48 ①항 | 무단 침입 (해킹) |
| §48 ②항 | 악성프로그램 전달·유포 |
| §48 ③항 | DoS 등 정보통신망 장애 유발 |

### §2-8. 핵심 조문 5 — §48-3 침해사고 신고

[정통망법] §48-3 [^infonet-act] 은 침해사고 신고 의무를 부과한다 (5과목 04 §4-1 cross-ref).

| 항 | 내용 |
|---|---|
| §48-3 ①항 | 정보통신서비스 제공자는 침해사고 발생 사실을 인지한 경우 **지체 없이** KISA 또는 과기정통부에 신고 |
| §48-3 ②항 | 신고를 받은 기관은 침해사고 분석·대응 지원 |

### §2-9. 정통망법 벌칙 요약

| 죄명 | 조문 | 형량 |
|---|---|---|
| 사실 적시 명예훼손 | §70 ①항 | 3년 이하 / 3천만원 이하 |
| 거짓 사실 적시 명예훼손 | §70 ②항 | 7년 이하 / 5천만원 이하 / 자격정지 10년 |
| 악성프로그램 전달·유포 (§48 ②항) | §70-2 | 7년 이하 / 7천만원 이하 |
| 정보통신망 장애 유발 (§48 ③항) | §71 ①항 10호 | 5년 이하 / 5천만원 이하 |
| 무단 침입 (§48 ①항) | §72 ①항 1호 | 3년 이하 / 3천만원 이하 |
| 음란물·반복적 공포·불안 유발 (§44-7 ①항 1·3호) | §74 | 1년 이하 / 1천만원 이하 |
| 과태료 (지정·신고·보고 의무 위반 등) | §76 | 사안별 차등 |

> *주의*: 위 형량은 작성 시점 (2026.04.30) 의 [정통망법] 현행본 기준이며, 정기 개정으로 변경될 수 있다. 시험 응시 시점에 국가법령정보센터에서 직접 확인 필요.

### §2-10. 시험 포인트

- 정통망법 목적 = **이용촉진 + 이용자 개인정보 보호 + 안전한 환경**.
- 데이터 3법 (2020.2.4) 으로 **개인정보 조항 대거 [개인정보보호법] 이관** (2020.8.5 시행).
- §45 = 정보보호조치 / §45-3 = CISO / §47 = ISMS / §48 = 3대 금지행위 / §48-3 = 침해사고 신고.
- ISMS 인증 유효기간 = **3년** ([정통망법] §47 ③항).
- §70 명예훼손 (사실 3년 / 거짓 7년) 은 [형법] §307 보다 가중.

## §3. 정보통신기반 보호법

### §3-1. 법령 개요

| 항목 | 내용 |
|---|---|
| 정식 명칭 | 정보통신기반 보호법 |
| 약칭 | 기반보호법 |
| 제정 | 2001.1.26 제정 (법률 제6383호) / 2001.7.1 시행 |
| 소관 부처 | 과학기술정보통신부, 국가정보원 (분야별) |

### §3-2. 법의 목적 — §1

> "이 법은 전자적 침해행위에 대비하여 주요정보통신기반시설의 보호에 관한 대책을 수립·시행함으로써 동 시설을 안정적으로 운용하도록 하여 국가의 안전과 국민생활의 안정을 보장함을 목적으로 한다." [^cii-act]

### §3-3. 핵심 개념

| 개념 | 정의 (요약) |
|---|---|
| 주요정보통신기반시설 | 국가안전보장·행정·국방·치안·금융·통신·운송·에너지 등 분야의 핵심 시설 중 §8 에 따라 지정된 시설 |
| 전자적 침해행위 | 해킹·바이러스·DoS·전자기파 등 전자적 수단의 침해 |

### §3-4. 핵심 조문 1 — §8 주요정보통신기반시설의 지정

[정보통신기반 보호법] §8 [^cii-act] 은 다음 절차로 주요정보통신기반시설을 지정한다.

| 단계 | 활동 |
|---|---|
| 1. 추천 또는 신청 | 관계 중앙행정기관의 장이 추천 / 시설 관리기관이 신청 |
| 2. 심의·평가 | 정보통신기반보호위원회의 심의 |
| 3. 지정 | 관계 중앙행정기관의 장의 지정 + 고시 |
| 4. 정기 점검 | 지정 후에도 정기적으로 점검 |

### §3-5. 핵심 조문 2 — §9 취약점 분석·평가

[기반보호법] §9 [^cii-act] 는 주요정보통신기반시설 관리기관의 장이 정기적으로 **취약점 분석·평가** 를 수행할 것을 요구한다.

| 항 | 내용 |
|---|---|
| §9 ①항 | 정기적으로 취약점을 분석·평가하고 결과를 관계 중앙행정기관의 장에게 통보 |
| §9 ②항 | 분석·평가의 방법·절차 등은 시행령 |
| §9 ③항 | 분석·평가 수행기관 — KISA, 한국전자통신연구원, 정보보호 전문서비스 기업 등 |

### §3-6. 핵심 조문 3 — §10 보호대책 수립

[기반보호법] §10 [^cii-act] 은 취약점 분석·평가 결과를 토대로 **주요정보통신기반시설 보호대책** 을 수립할 것을 요구한다.

| 단계 | 활동 |
|---|---|
| 1. 보호대책 수립 | 관리기관의 장이 시설별 보호대책 수립 |
| 2. 정부 보고 | 관계 중앙행정기관의 장에게 제출 |
| 3. 정부 종합 | 관계 중앙행정기관의 장이 분야별 보호계획 수립 |
| 4. 종합대책 | 정부가 주요정보통신기반시설 보호 종합계획 수립·시행 |

### §3-7. 핵심 조문 4 — §12 침해 금지 + §13 침해사고 통지

[기반보호법] §12 [^cii-act] (5과목 06 §5-5 cross-ref): 주요정보통신기반시설 침해 금지 — 처벌 §28 (10년 이하).

[기반보호법] §13 [^cii-act] (5과목 04 §4-2 cross-ref): 침해사고 발생 시 관리기관의 장은 **지체 없이** 관계 중앙행정기관의 장·수사기관·KISA에 통지.

### §3-8. 정통망법 vs 기반보호법 비교

| 비교 항목 | 정통망법 | 기반보호법 |
|---|---|---|
| 보호 대상 | 일반 정보통신망·정보통신서비스 제공자 | 주요정보통신기반시설 (지정된 핵심 시설) |
| 침해 처벌 | §72 (3년 이하 / 3천만원) | §28 (10년 이하 / 1억원) — **가중** |
| 침해사고 신고 | §48-3 (지체 없이 KISA·과기정통부) | §13 (지체 없이 관계 중앙행정기관·수사기관·KISA) |
| 취약점 분석·평가 | §45 (자율) | §9 (의무) |
| 의무 강도 | 자율 + 일정 규모 이상 의무 | 지정 시 의무 |

### §3-9. 시험 포인트

- 기반보호법 = **주요정보통신기반시설** 보호 (일반 정보통신망 = 정통망법).
- 주요정보통신기반시설 지정 = **§8** (관계 중앙행정기관의 장이 지정·고시).
- 취약점 분석·평가 = **§9** (정기 의무).
- 침해사고 통지 = **§13**.
- 침해 처벌 = **§28** (10년 이하 — 정통망법보다 가중).

## §4. 전자서명법

### §4-1. 법령 개요와 2020 전부개정

| 항목 | 내용 |
|---|---|
| 정식 명칭 | 전자서명법 |
| 최초 제정 | 1999.7.1 (구 전자서명법 — **공인인증서** 제도 도입) |
| **2020 전부개정** | **2020.6.9 전부개정** (법률 제17354호) / **2020.12.10 시행** |
| 소관 부처 | 과학기술정보통신부 |

### §4-2. 2020 전부개정의 핵심 변화

| 변화 | 구 전자서명법 (1999~2020.12) | 신 전자서명법 (2020.12~ 현행) |
|---|---|---|
| 공인인증서 제도 | **존재** (공인인증기관·공인인증서) | **폐지** (다양한 전자서명 수단을 동등하게 인정) |
| 인증사업자 자격 | 정부가 지정한 공인인증기관 | 시장 경쟁 — 평가·인정제 |
| 전자서명 효력 | 공인전자서명만 우월적 효력 | 모든 전자서명이 본인 서명·날인과 동등한 법적 효력 |
| 평가·인정 | 정부 지정 | 평가기관의 평가 + 과기정통부 인정 |

### §4-3. 핵심 조문 1 — §1 목적

> "이 법은 전자문서 및 전자거래의 안전성과 신뢰성을 확보하고 그 이용을 활성화하기 위하여 전자서명에 관한 기본적인 사항을 정함으로써 국가사회의 정보화를 촉진하고 국민생활의 편익을 증진함을 목적으로 한다." [^esign-act]

### §4-4. 핵심 조문 2 — §3 전자서명의 효력

[전자서명법] §3 [^esign-act]:

| 항 | 내용 |
|---|---|
| §3 ①항 | **전자서명은 서명·서명날인 또는 기명날인의 효력을 가진다.** 다만, 다른 법률에 특별한 규정이 있는 경우는 제외 |
| §3 ②항 | 법령의 규정에 따라 서명·서명날인 또는 기명날인이 필요한 경우 전자서명이 있을 때에는 그 서명·서명날인 또는 기명날인이 있는 것으로 본다 |

> 이는 **모든 전자서명이 손글씨 서명·날인과 동등한 법적 효력** 을 가진다는 핵심 조항이다. 구 법의 "공인전자서명" 우월적 효력은 폐지되었다.

### §4-5. 핵심 조문 3 — 전자서명인증사업자

[전자서명법] §6~§14 [^esign-act] 은 전자서명인증사업자 제도를 규정한다.

| 조문 | 내용 |
|---|---|
| §6 (인증사업자 등록) | 전자서명인증 업무를 하려는 자는 과기정통부장관에게 등록 |
| §7 (인증업무준칙 등) | 인증사업자의 인증업무 수행 기준 |
| §8 (전자서명인증서비스 가입자에 대한 보호조치) | 가입자 정보의 안전성·정확성 |
| §9~§13 (운영·중단·승계 등) | 인증사업자의 운영 절차 |
| §14 (인증서의 폐지·해지) | 인증서 폐지 사유 — 4과목 04 §4-3 (CRL·OCSP) cross-ref |

### §4-6. 핵심 조문 4 — §15 평가·인정

[전자서명법] §15 [^esign-act] 는 전자서명인증사업자의 운영 평가·인정 제도를 규정한다.

| 단계 | 활동 |
|---|---|
| 1. 평가 신청 | 인증사업자가 평가기관에 평가 신청 |
| 2. 평가 수행 | 평가기관이 인증업무 운영 평가 |
| 3. 인정 신청 | 평가 통과 사업자가 과기정통부장관에 인정 신청 |
| 4. 인정 | 과기정통부장관의 인정 (인정 사실 공고) |

### §4-7. 인정 전자서명

> **인정 전자서명** : [전자서명법] §15 의 평가·인정을 받은 전자서명인증사업자가 발급한 인증서 기반의 전자서명. 구 공인전자서명을 대체하는 개념.

| 비교 항목 | 구 공인전자서명 | 신 인정 전자서명 |
|---|---|---|
| 법적 근거 | 구 [전자서명법] §3 ②항·§5 | 신 [전자서명법] §15 |
| 우월적 효력 | **존재** | **폐지** (모든 전자서명 동등) |
| 인정 방식 | 정부 지정 (공인인증기관) | 평가·인정 (시장 경쟁) |
| 운영 기관 | 과거 KISA·금융결제원·NICE 평가정보 등 | 다수 사업자 (공동인증서·금융인증서·간편인증 등) |

### §4-8. 4과목 04 §6 한국 PKI 운영과 cross-ref

4과목 04 단원 §6 (한국 PKI 운영) 은 디지털서명·인증서 표준 측면을 다루었다. 본 §4 는 같은 내용을 **법령** 측면에서 다룬다.

| 측면 | 4과목 04 §6 | 5과목 07 §4 |
|---|---|---|
| 디지털서명 알고리즘 | RSA·ECDSA·EdDSA (FIPS 186-5) | (해당 없음) |
| 인증서 구조 | X.509 v3 (RFC 5280) | (해당 없음) |
| 한국 법령 시점 | (간단 언급) | 2020.6.9 전부개정 / 2020.12.10 시행 상세 |
| 인정 전자서명 | (간단 언급) | §15 평가·인정 절차 상세 |

### §4-9. 시험 포인트

- 전자서명법 전부개정 = **2020.6.9 전부개정** (법률 제17354호) / **2020.12.10 시행**.
- 공인인증서 제도 = **폐지** (구 법) → 인정 전자서명 (신 법).
- §3 ①항 = **모든 전자서명이 서명·날인과 동등한 효력**.
- §15 = 전자서명인증사업자의 평가·인정 절차.
- 인정 전자서명 ≠ 공인전자서명 (개념 자체가 다름).

- 합법적 감청 (§5) = **법원 허가** 필수.

## §7. 시험 함정 정리

| 함정 | 정답 |
|---|---|
| 정통망법은 모든 정보통신서비스 제공자에 ISMS 의무 | **틀림**. **일정 규모 이상** ([정통망법] §47 ②항 + 시행령 §49) |
| 정통망법 §45-3 = ISMS | **틀림**. §45-3 = **CISO** 지정. ISMS = §47 |
| 정통망법 §48 ③항 = 악성코드 | **틀림**. ③항 = **DoS** 등 정보통신망 장애. ②항 = 악성프로그램 |
| 기반보호법은 모든 정보통신망에 적용 | **틀림**. **주요정보통신기반시설** (지정된 시설) 한정 |
| 주요정보통신기반시설 지정 권한 = 대통령 | **틀림**. **관계 중앙행정기관의 장** ([기반보호법] §8) |
| 기반보호법 침해 처벌 = 정통망법과 동일 | **틀림**. **§28 — 10년 이하** (정통망법 §72 = 3년 이하 보다 가중) |
| 전자서명법 전부개정 시행일 = 2020.6.9 | **틀림**. **2020.6.9 = 전부개정 공포일** / **2020.12.10 = 시행일** |
| 공인인증서 = 인정 전자서명 | **틀림**. 공인인증서는 **2020 전부개정으로 폐지**. 인정 전자서명은 신 법 §15 의 평가·인정 받은 전자서명 |
| 전자서명법 §3 ①항 = 공인전자서명만 효력 | **틀림**. **모든 전자서명** 이 서명·날인과 동등한 효력 (구 법의 우월적 효력 폐지) |
