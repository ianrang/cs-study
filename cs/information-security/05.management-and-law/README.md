## 4. 작성 원칙 (5과목 적용)

[외부자료 검증 체크리스트](../docs/외부자료-검증체크리스트.md) §3 을 따른다. 5과목의 특수성을 반영한 추가 사항:

1. **외부 1차 출처 100%**: 본 5과목은 통합자료 매핑이 부재하므로 모든 사실 진술은 1차 출처 각주 [^N] 로 명시한다. 위키피디아·민간 학원 자료·블로그·요약 노트는 1차 출처로 사용하지 않는다.
2. **추론 금지**: "아마 / 보통은 / 것으로 보인다 / 것으로 추정 / 자주 인용 / 자주 출제" 등 단정·추론 표현 금지. 표준 본문·법령 조문에 명시되지 않은 영역은 "표준 정의" 가 아닌 "용어 사용 사례·운용 사례" 로 약화 표현한다.
3. **충돌 시 우선순위**: 출제기준 → 1차 법령 (국가법령정보센터 현행본) → 1차 표준 (ISO/IEC·NIST SP·FIPS) → 운영 가이드 (KISA·개인정보보호위원회). 동일 사안에 대해 한국 (정통망법·개인정보보호법) 과 국제 (ISO 27001·NIST CSF) 가 다를 경우 양쪽을 모두 명시하고 차이를 표기한다.
4. **한국 법령 시점 정확성 — 5과목 핵심 의무**:
   - 모든 법령 인용은 [법령명 제N조 제M항] 형식 + 시행일자 + 국가법령정보센터 출처 각주로 표기한다.
   - **개정일 vs 시행일을 구분**한다 (예: 전자서명법 2020.6.9 전부개정 / 2020.12.10 시행).
   - 작성 시점 (2026.04.30) 의 현행본을 사용하며, 폐지·제정·전부개정·일부개정 이력을 정확히 표기한다.
   - 조문 번호 (§N), 호 번호 (제M항·제M호), 단서 등은 원문 그대로 인용하고 임의 가공 금지.
5. **표준 식별 정확성**: ISO/IEC 27001:2022 vs 27001:2013 차이, NIST CSF 1.1 vs 2.0 (2024.2 발행) 차이, NIST SP 800-30 Rev. 1 (2012) / SP 800-37 Rev. 2 (2018) / SP 800-53 Rev. 5 (2020) / SP 800-61 Rev. 2 (2012) 발행 시점, ISMS-P 인증기준 통제항목 수 (3영역 21분야 102개 — 작성 시점에 KISA 출처에서 직접 확인) 는 추측하지 않는다.
6. **표기 일관성**: ISMS-P (대시·하이픈 표기 통일), CC (Common Criteria, ISO/IEC 15408), KCMVP (Korea Cryptographic Module Validation Program), CISO (Chief Information Security Officer), CPO (Chief Privacy Officer) 등 약어는 5과목 전체에서 통일.
7. **수치 정확성**: BCP·DRP 사이트 4종 RTO 범위 (미러 즉시 / 핫 수 시간 이내 / 웜 수일~수주 / 콜드 수주~수개월), 포렌식 5대 원칙 5종, 침해사고 대응 절차 7단계, ISMS-P 인증기준 영역·분야·통제 수 등은 1차 출처 표기 그대로 보존.
8. **서술 원칙 (1·2·3·4 과목 정착)**: 각 H2/H3 단위로 "**정의 (인용 박스) → 도입 단락 (서술형) → 본문 → 표/예시 → 시험 포인트**" 흐름을 따른다. 표만으로 구성된 영역 금지 — 분류·체계·법령 조문의 의미를 서술적으로 풀이하는 도입 단락 필수.
9. **cross-ref 정확성**: 1·2·3·4 과목 단원 및 5과목 단원 간 cross-ref 는 참조 대상 파일의 자체 §번호와 정확히 일치시킨다 (H2 = `§N`, H3 = `§N-M`). 동일 파일 내부에서는 anchor 링크를 사용한다.
10. **국제 표준 vs 국내 법령 분리**: ISO/IEC 27001 (국제 ISMS 표준) 과 ISMS-P (한국 인증제도, 정통망법·개인정보보호법 근거) 의 표준화·법적 근거 주체를 혼동하지 않는다. NIST CSF 2.0 (미국 권고) 와 한국 ISMS-P 인증기준은 별개 체계임을 명시한다.
