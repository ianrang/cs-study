# 5과목 02 — 정보보호 위험평가

> 출제기준 매핑: 1.2 정보보호 위험평가 — 위험분석 및 계획수립 / 정보자산 식별 및 분석 / 위험분석 및 평가

## §0. 들어가며

위험평가는 정보보호의 **출발점이자 핵심 의사결정 도구** 이다. 위험을 식별하지 않으면 어디에 어떤 통제를 두어야 할지 결정할 수 없으며, 결국 비용을 낭비하거나 핵심 위험을 방치하게 된다. 본 단원은 위험관리의 5단계 절차, 자산·위협·취약성·보호대책의 정의, 위험분석 접근방법 4종 (기준선·비정형·상세·복합), 정량적·정성적 분석 방식, 위험처리 전략 4종 (수용·감소·전가·회피) 을 다룬다.

본 단원은 ISO/IEC 27005:2022 (Information security, cybersecurity and privacy protection — Guidance on managing information security risks) [^iso27005] 와 NIST SP 800-30 Rev. 1 (Guide for Conducting Risk Assessments) [^nist800-30], NIST SP 800-39 (Managing Information Security Risk) [^nist800-39], NIST SP 800-37 Rev. 2 (Risk Management Framework, RMF) [^nist800-37] 를 1차 출처로 한다. 한국 표준은 KISA 「정보보호 관리체계 인증기준 안내서」 [^isms-p-cert] 와 KISA 위험관리 가이드 [^kisa-rm] 를 1차로 사용한다.

학습 순서:
1. 위험관리의 정의와 5단계 (§1)
2. 위험의 구성요소 — 자산·위협·취약성·보호대책·DoA (§2)
3. 위험분석 접근방법 4종 (§3)
4. 자산 식별·평가·그룹핑 (§4)
5. 위협·취약성 평가 (§5)
6. 위험분석 방식 — 정량 vs 정성 (§6)
7. 위험처리 전략 4종 (§7)
8. 정보보호 대책명세서 (§8)

## §1. 위험관리의 정의와 5단계

> **위험관리 (Risk Management)** : 조직의 자산에 대한 위험을 **수용 가능한 목표 위험수준 (DoA)** 으로 유지하기 위해 자산에 대한 위험을 분석하고, 비용 대비 효율적인 보호대책을 마련하는 일련의 과정.[^isms-def]

ISO/IEC 27005:2022 [^iso27005] 는 위험관리를 "조직의 위험에 대한 의사소통, 의사결정, 분석, 평가, 처리, 모니터링, 검토 활동" 으로 정의한다. 위험관리는 1회성 평가가 아니라 **지속적인 사이클** 이다.

### §1-1. 위험관리 5단계

| 단계 | 활동 | 산출물 |
|---|---|---|
| 1. 위험관리 전략 및 계획 수립 | 조직에 적합한 위험관리 방법 선택 / 비용·시간·인력 등 가용 자원 고려한 계획 수립 | 위험관리 계획서 |
| 2. 위험 분석 | 자산·위협·취약성·기존 보호대책 등을 분석하여 **위험의 종류와 규모** 결정 | 위험 분석 보고서 |
| 3. 위험 평가 | 위험 분석 결과로 위험도를 평가하고 **DoA (수용 가능한 목표 위험수준)** 을 정하여 위험 대응 여부와 우선순위 결정 | 위험 평가 보고서 |
| 4. 정보보호 대책 선정 | DoA 초과 위험을 수용 가능한 수준으로 감소시키기 위해 **위험처리 전략** 설정 + 적절한 통제 선택 | 정보보호 대책명세서 |
| 5. 정보보호 계획 수립 | 선정한 보호대책을 구현하기 위한 **이행 계획** 수립 | 정보보호 이행계획서 |

### §1-2. ISO/IEC 27005 vs NIST SP 800-39 비교

ISO 와 NIST 는 위험관리 절차의 단계 명명이 다르지만 본질은 동일하다.

| ISO/IEC 27005:2022 | NIST SP 800-39 | 핵심 활동 |
|---|---|---|
| Context establishment | Frame | 조직 환경·범위 확립 |
| Risk identification | Identify (Assess 단계 내) | 자산·위협·취약성 식별 |
| Risk analysis | Analyze (Assess 단계 내) | 위험의 가능성·영향 분석 |
| Risk evaluation | Evaluate (Assess 단계 내) | 위험을 기준과 비교하여 우선순위화 |
| Risk treatment | Respond | 위험 대응 (수용·감소·전가·회피) |
| Risk monitoring and review | Monitor | 지속적 모니터링 |

### §1-3. 시험 포인트

- 위험관리의 출발점 = **계획 수립** (분석이 아님 — 분석은 2단계).
- 위험 평가 ≠ 위험 분석. 분석은 **종류와 규모** 결정, 평가는 **DoA 와 비교하여 대응 우선순위** 결정.
- 위험관리의 본질 = **DoA 까지 위험을 낮추는 것** (위험 0 으로 만드는 것이 아님).
- 위험관리는 **지속적·순환적** 활동.

## §2. 위험의 구성요소

> **위험 (Risk)** : 자산의 취약성을 이용한 위협에 의해 원하지 않는 사건이 발생하여 자산에 손실을 미칠 가능성.[^isms-def]

ISO/IEC 27005:2022 [^iso27005] 와 NIST SP 800-30 Rev. 1 [^nist800-30] 은 위험을 "사건의 가능성 (likelihood) × 영향 (impact)" 으로 정의한다.

### §2-1. 위험의 함수적 관계

위험은 자산·위협·취약성을 인자로 한 함수로 표현된다.

```
위험 (Risk) = f (자산, 위협, 취약성)
```

여기에 보호대책 (Safeguard) 을 추가하여 4대 구성요소로 확장한다.

```
잔여 위험 (Residual Risk) = f (자산, 위협, 취약성, 보호대책)
```

### §2-2. 위험의 3대·4대 구성요소

| 구분 | 구성요소 |
|---|---|
| 3대 구성요소 | 자산 / 위협 / 취약성 |
| 4대 구성요소 | 자산 / 위협 / 취약성 / 보호대책 |

### §2-3. 각 구성요소의 정의

| 요소 | 정의 |
|---|---|
| 자산 (Asset) | 조직이 보호해야 할 **유·무형의 대상**. 하드웨어·소프트웨어·데이터·인력·시설·평판 등 |
| 위협 (Threat) | 자산에 손실을 초래할 수 있는 **원하지 않는 사건의 잠재적 원인**. 예: 해커 공격·내부자·천재지변·시스템 오류 |
| 취약성 (Vulnerability) | 위협의 **이용 대상이 되는 자산의 잠재적 속성**. 예: 미패치 시스템·약한 패스워드·미흡한 출입통제 |
| 보호대책 (Safeguard / Control) | 조직의 자산을 위험으로부터 보호하기 위한 **안전 대책 또는 통제** |

> **위협이 발생한다고 반드시 자산에 손실이 발생하는 것은 아니다.** 위협이 자산의 취약성을 이용해야 비로소 위험이 실현된다. 즉 취약성이 없는 자산에 대한 위협은 위험이 되지 않는다.[^isms-def]

### §2-4. 잔여 위험과 DoA

> **DoA (Degree of Assurance, 수용 가능한 목표 위험수준)** : 조직이 **감수하기로 결정한** 위험의 최대 허용 수준.

DoA 는 비즈니스 환경·법규·자원·이해관계자 요구를 종합하여 **경영진이 결정** 한다. ISO/IEC 27005:2022 §7 [^iso27005] 는 이를 "Risk Acceptance Criteria" 로 명명한다.

| 위험 상태 | 의미 |
|---|---|
| 식별된 위험 (Identified Risk) | 분석을 통해 도출된 모든 위험 |
| 처리 대상 위험 (Treated Risk) | DoA 초과로 처리가 결정된 위험 |
| 잔여 위험 (Residual Risk) | 보호대책 구현 후에도 남아 있는 위험 |
| 수용 위험 (Accepted Risk) | DoA 이내로 평가되어 수용 결정된 위험 |

### §2-5. 보호대책 유형 분류

#### 유형에 따른 분류 (3종)

| 유형 | 정의 | 사례 |
|---|---|---|
| 기술적 보호대책 | 보안 솔루션 도입 및 운영 | 방화벽, IPS, 암호화, DLP, EDR |
| 관리적 보호대책 | 보안 정책의 수립 및 시행 관점 | 정책·표준·지침·절차, 교육, 감사 |
| 물리적 보호대책 | 시설물 및 장비 보호 관점 | 출입통제, CCTV, 항온항습, UPS |

> *원본 표기 보정*: 5과목.txt line 113 의 "고나점" 은 "관점 (觀點)" 의 OCR 오류이다.

#### 시점에 따른 분류 (3종)

| 유형 | 정의 | 사례 |
|---|---|---|
| 예방 통제 (Preventive) | 발생할 수 있는 위험을 식별하여 **사전에 대처** 하는 능동적 통제 | 출입통제, 방화벽, 패치 관리, 교육 |
| 탐지 통제 (Detective) | 예방 통제를 우회하여 발생하는 각종 위험을 **탐지** 하는 통제 | IDS, 로그 모니터링, CCTV, 침입 탐지 |
| 교정 통제 (Corrective) | 탐지된 위험에 **대처하거나 감소** 시키는 통제 | 백업 복구, 사고 대응, 패치 적용, 시정조치 |

NIST SP 800-53 Rev. 5 [^nist800-53] 는 통제를 더 세분화하여 다음을 추가한다.

| 추가 유형 | 정의 | 사례 |
|---|---|---|
| 억제 통제 (Deterrent) | 위협 행위자가 공격을 시도하지 못하도록 억제 | 경고 표지, 처벌 규정 |
| 보상 통제 (Compensating) | 표준 통제 적용이 불가할 때 동등한 효과를 제공 | 강력한 모니터링으로 약한 인증 보완 |
| 복구 통제 (Recovery) | 사고 후 시스템·서비스 복원 | 재해복구 계획, 백업 |

### §2-6. 시험 포인트

- 위험의 3대 구성요소 = **자산·위협·취약성** (보호대책 포함하면 4대).
- "위협이 있다 = 위험이 있다" 는 **틀림**. 취약성이 있어야 위험이 됨.
- DoA = 수용 가능한 **목표** 위험수준 (Degree of Assurance).
- 잔여 위험은 보호대책 구현 후에도 남는다 — **0 으로 만들 수 없음**.
- IDS·CCTV = **탐지 통제**, 백업 복구·시정조치 = **교정 통제**, 출입통제·방화벽 = **예방 통제**.

## §3. 위험분석 접근방법 4종

ISO/IEC 27005:2022 [^iso27005] 와 KISA 위험관리 가이드 [^kisa-rm] 는 위험분석 접근방법을 다음 4종으로 분류한다.

### §3-1. 기준선 접근법 (Baseline Approach)

> **기준선 접근법** : 체크리스트 형식의 표준화된 보호대책을 이용하여 보호의 기본 수준을 정하고 위험분석을 수행하는 방법.[^isms-def]

| 장점 | 단점 |
|---|---|
| 시간과 비용을 절약 | 조직의 특성이 고려되지 않음 |
| 기본적인 보호대책 선정 가능 | 과대·과소 보호 가능성 |
| 표준화로 일관성 확보 | 새로운 위험에 대응 부족 |

대표 사례: ISO/IEC 27002:2022 의 93개 통제 [^iso27002], NIST SP 800-53 Rev. 5 의 통제 카탈로그 [^nist800-53] 를 체크리스트로 활용.

### §3-2. 비정형 접근법 (Informal Approach)

> **비정형 접근법** : 경험자의 지식을 기반으로 위험분석을 수행하는 방법.[^isms-def]

| 장점 | 단점 |
|---|---|
| 작은 규모의 조직에 비용 대비 효과적 | 수행자 경험 의존도가 높음 |
| 빠른 의사결정 가능 | 경험이 부족한 위험 영역을 놓칠 가능성 |
| 별도 도구·방법론 불필요 | 일관성·재현성 부족 |

### §3-3. 상세 위험분석 (Detailed Risk Analysis)

> **상세 위험분석** : 정형화되고 구조화된 위험분석 방법론에 따라 자산·위협·취약성 분석 등의 각 단계를 통해 조직의 위험을 모두 상세하게 분석하는 접근 방법.[^isms-def]

| 장점 | 단점 |
|---|---|
| 조직에 가장 적절한 보호대책 수립 | 전문적 지식 필요 |
| 보안환경의 변화에 유연하게 대처 | 많은 시간·비용·노력 필요 |
| 의사결정에 강력한 근거 제공 | 분석 자체가 부담이 될 수 있음 |

### §3-4. 복합 접근법 (Combined Approach)

> **복합 접근법** : 고위험 영역을 식별하여 상세 위험분석을 수행하고, 그 외 영역은 기준선 접근법을 사용하는 방법.[^isms-def]

| 장점 | 단점 |
|---|---|
| 위험분석 비용·자원을 효과적 사용 | 고위험 영역을 잘못 식별하면 문제 발생 |
| 핵심 자산에 집중적 분석 | 초기 식별 단계의 정확성에 의존 |
| 대부분의 조직에 권장됨 | 두 가지 방법 모두 숙지 필요 |

ISO/IEC 27005:2022 [^iso27005] 는 대부분의 조직에 **복합 접근법** 을 권장한다.

### §3-5. 4종 비교표

| 접근법 | 정형성 | 시간·비용 | 정확성 | 권장 대상 |
|---|---|---|---|---|
| 기준선 | 표준화 | 낮음 | 보통 | 소규모·일관성 중시 |
| 비정형 | 비공식 | 매우 낮음 | 낮음 (경험 의존) | 매우 작은 조직 |
| 상세 | 정형 | 매우 높음 | 매우 높음 | 고위험·대규모 조직 |
| 복합 | 혼합 | 중간 | 높음 (선택적) | 대부분의 조직 |

### §3-6. 시험 포인트

- 4종 중 **상세 위험분석** 만이 정형화된 방법론을 사용.
- **복합 접근법** = 고위험 영역은 상세 + 그 외는 기준선.
- 비정형 접근법의 가장 큰 단점 = **수행자 경험 부족 영역의 위험 누락**.
- ISO/IEC 27005 권장 방법 = 복합 접근법.

## §4. 자산 식별·평가·그룹핑

위험분석의 첫 단계는 자산 식별이다. 자산의 가치를 알지 못하면 위험의 영향을 산정할 수 없다.

### §4-1. 자산 식별·평가·그룹핑 절차

| 단계 | 활동 |
|---|---|
| 자산 식별 | 조직 내 모든 정보 자산 목록화 (HW, SW, 데이터, 인력, 시설, 서비스) |
| 자산 가치 평가 | 각 자산의 기밀성·무결성·가용성 측면의 가치 산정 |
| 자산 그룹핑 | 유사한 가치·위협·취약성을 가진 자산을 그룹으로 묶어 분석 효율화 |

### §4-2. 자산 분류 — ISO/IEC 27002:2022 §5.9

ISO/IEC 27002:2022 §5.9 (Inventory of information and other associated assets) [^iso27002] 는 자산을 다음 카테고리로 분류한다.

| 자산 유형 | 사례 |
|---|---|
| 정보 (Information) | 데이터베이스, 파일, 문서, 매뉴얼, 계약서 |
| 소프트웨어 (Software) | 응용 프로그램, 시스템 SW, 개발 도구 |
| 물리적 자산 (Physical) | 서버, 통신장비, 매체, 시설 |
| 서비스 (Service) | 컴퓨팅 서비스, 통신 서비스, 일반 서비스 (전기·공조) |
| 인력 (People) | 직원, 전문가, 외부 위탁 인력 |
| 무형 자산 (Intangible) | 평판, 브랜드, 노하우 |

### §4-3. 자산 가치 평가 기준

자산 가치는 CIA 침해 시 발생할 영향을 기준으로 평가한다.

| 평가 기준 | 측정 항목 |
|---|---|
| 기밀성 영향 | 정보 노출 시 비즈니스·법적·평판 손실 |
| 무결성 영향 | 정보 변조 시 의사결정·업무·법적 손실 |
| 가용성 영향 | 정보 사용 불가 시 업무 중단·매출·법적 손실 |

평가 결과는 통상 3~5단계 척도 (예: 매우 높음·높음·보통·낮음·매우 낮음) 로 표현한다.

### §4-4. 시험 포인트

- 자산 가치는 **CIA 측면** 으로 평가.
- 자산 그룹핑의 목적 = **분석 효율화** (개별 분석 부담 감소).
- 인력·평판도 자산 — HW/SW 만이 자산이 아님.

## §5. 위협·취약성 평가

### §5-1. 위협 평가

위협은 다음과 같이 분류된다.

| 분류 기준 | 종류 |
|---|---|
| 출처 | 인적 (의도적·비의도적), 자연재해, 환경적 (정전·화재·누수) |
| 행위 의도 | 의도적 (해킹·내부자 악의), 비의도적 (실수·부주의) |
| 영향 대상 | 기밀성 위협, 무결성 위협, 가용성 위협 |

위협 평가는 **위협의 발생 가능성 (Likelihood)** 을 산정하는 활동이다.

### §5-2. 취약성 평가

취약성은 자산이 위협에 노출되는 통로이다.

| 취약성 유형 | 사례 |
|---|---|
| 기술적 취약성 | 미패치 OS, 약한 암호 알고리즘, 미암호화 통신 |
| 관리적 취약성 | 미흡한 정책, 교육 부재, 직무 미분리 |
| 물리적 취약성 | 출입통제 미흡, 노후 시설, 환경 통제 부재 |

취약성 평가는 자동화 도구 (취약점 스캐너 — 1과목 07 §3 참조) 와 수동 점검 (모의해킹·관리적 점검) 을 병행한다.

### §5-3. 우려사항 평가 — 위협-취약성 결합

> **우려사항 (Concern)** : 특정 위협이 특정 취약성을 이용했을 때 발생할 수 있는 사건의 시나리오.[^isms-def]

위협만으로 위험이 되지 않으며 취약성만으로도 위험이 되지 않는다. **위협이 취약성을 이용** 할 때 비로소 우려사항이 되고, 자산 가치와 결합하여 위험이 산정된다.

```
우려사항 = 위협 × 취약성
위험 = 우려사항 × 자산 가치 = 자산 × 위협 × 취약성
```

### §5-4. 시험 포인트

- "위협이 있다 = 취약성이 있다" 는 **틀림**. 별개 개념.
- 취약점 스캐너로 식별 가능한 것 = **기술적 취약성** (관리적·물리적 취약성은 별도 점검).
- 우려사항은 **위협 + 취약성** 의 결합.

## §6. 위험분석 방식 — 정량 vs 정성

### §6-1. 정량적 위험분석

> **정량적 위험분석 (Quantitative Risk Analysis)** : 손실이나 위험의 크기를 **금전적 가치** 로 표현하는 방식.[^isms-def]

정량적 위험분석은 객관적·수학적 결과를 제공한다. 핵심 산출 공식은 다음과 같다.[^nist800-30]

```
SLE (Single Loss Expectancy, 단일 손실 예상액) = 자산 가치 × EF (Exposure Factor, 노출계수)
ARO (Annual Rate of Occurrence, 연간 발생 빈도) = 1년에 사건이 발생할 횟수
ALE (Annual Loss Expectancy, 연간 손실 예상액) = SLE × ARO
```

| 변수 | 의미 |
|---|---|
| 자산 가치 (Asset Value) | 자산의 금전적 가치 |
| EF | 사건 발생 시 자산이 입을 손실 비율 (0~1 또는 0~100%) |
| SLE | 1회 사건당 예상 손실액 |
| ARO | 1년에 사건이 발생하는 횟수 (예: 0.5 = 2년에 1번) |
| ALE | 연간 평균 예상 손실액 — 보호대책 투자 결정의 기준 |

| 장점 | 단점 |
|---|---|
| 객관적 비용 산정 가능 | 모든 손실을 금전적으로 환산하기 어려움 |
| 비용 대비 효과 (ROSI) 분석 가능 | 데이터 수집 비용·시간 큼 |
| 경영진 의사결정에 강한 근거 | 무형 자산 (평판·고객 신뢰) 정량화 어려움 |

### §6-2. ROSI — 보호대책 투자수익률

> **ROSI (Return on Security Investment)** : 보호대책 투자의 효율을 측정하는 지표.

```
ROSI = (위험 감소량 - 보호대책 비용) / 보호대책 비용
     = (ALE_before - ALE_after - 보호대책 비용) / 보호대책 비용
```

ROSI > 0 이면 보호대책 투자가 비용을 정당화한다.

### §6-3. 정성적 위험분석

> **정성적 위험분석 (Qualitative Risk Analysis)** : 손실이나 위험 크기를 **개략적인 크기** 로 비교하는 방식.[^isms-def]

정성적 분석은 비교적 간단하고 시간·노력이 적게 들지만 객관적 검증이 어렵고 평가자 개인에 따라 결과가 달라질 수 있다.

| 기법 | 설명 |
|---|---|
| 델파이 (Delphi) | 전문가 집단이 익명 토론을 통해 합의에 도달 |
| 시나리오 (Scenario) | 어떠한 사건도 기대대로 발생하지 않는다는 사실에 근거하여 일정 조건에서 위협의 발생 가능 결과를 추정 |
| 순위 결정법 (Ranking) | 비교 우위 순위 결정표에 의해 위험 항목들의 서술적 순위 결정 |
| 퍼지행렬법 (Fuzzy Matrix) | 위험분석 요소들을 정성적인 언어 (높음·보통·낮음) 로 표현된 값으로 기대 손실 평가 |

### §6-4. 정량 vs 정성 비교

| 비교 항목 | 정량적 | 정성적 |
|---|---|---|
| 표현 방식 | 금전적 가치 (₩, $) | 개략적 크기 (높음·보통·낮음) |
| 객관성 | 높음 | 낮음 |
| 시간·비용 | 큼 | 작음 |
| 데이터 요구 | 많은 양의 정량 데이터 | 전문가 의견 |
| 무형 자산 처리 | 어려움 | 용이 |
| 경영진 의사결정 근거 | 강함 (ROSI 등) | 보통 |
| 대표 산출 | ALE = SLE × ARO | 위험 수준 (Risk Level) |

### §6-5. 시험 포인트

- 정량적 분석의 핵심 공식 = **ALE = SLE × ARO**.
- SLE = **자산 가치 × EF**.
- 정성적 분석 4종 = **델파이·시나리오·순위결정·퍼지행렬**.
- 델파이 = **전문가 집단의 익명 토론** 으로 합의 (개인 의견이 아님).
- ROSI > 0 = 투자 정당화.

## §7. 위험처리 전략 4종

> **위험처리 (Risk Treatment)** : 위험 평가 결과 DoA 를 초과하는 위험에 대해 대응 방법을 결정하는 활동.[^iso27005]

ISO/IEC 27005:2022 [^iso27005] 와 NIST SP 800-39 [^nist800-39] 는 위험처리 전략을 4종으로 정의한다.

### §7-1. 4가지 위험처리 전략

| 전략 | 정의 | 사례 |
|---|---|---|
| 위험 수용 (Risk Acceptance) | 현재의 위험을 받아들이고 잠재적 손실 비용을 감수 | 발생 빈도·영향이 낮은 위험 / DoA 이내 위험 |
| 위험 감소 (Risk Mitigation / Reduction) | 위험을 감소시킬 수 있는 보호대책 채택 | 방화벽·암호화·교육·정책 수립 |
| 위험 전가 (Risk Transfer) | 보험이나 외주 등 **제3자에게 이전** | 사이버 보험 가입, 보안 운영 외주 |
| 위험 회피 (Risk Avoidance) | 프로세스나 사업을 **수행하지 않고 포기** | 위험 높은 신규 서비스 출시 중단 |

> *주의*: ISO/IEC 27005:2022 [^iso27005] 와 NIST SP 800-39 [^nist800-39] 의 영문 용어 표기는 "Risk Avoidance / Risk Modification / Risk Sharing / Risk Retention" 으로 되어 있으나, 한국 정보보안기사 시험 및 KISA 가이드는 "감소·전가·수용·회피" 의 4분류를 표준 용어로 사용한다. 시험 답안은 한국 표준 용어를 따른다.

### §7-2. 전략 선택 기준

| 위험 발생 가능성 | 위험 영향 | 권장 전략 |
|---|---|---|
| 높음 | 높음 | **회피** 또는 **감소** + 전가 |
| 높음 | 낮음 | **수용** 또는 **감소** |
| 낮음 | 높음 | **전가** (보험) |
| 낮음 | 낮음 | **수용** |

### §7-3. 시험 포인트

- 4가지 전략 = **수용·감소·전가·회피** (4가지 모두 암기 필수).
- 보험 가입 = **위험 전가** (감소 아님).
- 사업 자체를 안 함 = **위험 회피** (감소 아님).
- 방화벽·암호화 = **위험 감소**.
- DoA 이내 위험 = **수용**.

## §8. 정보보호 대책명세서

> **정보보호 대책명세서 (Statement of Applicability, SoA)** : 위험처리 결정의 결과로 선정된 보호대책 (통제) 의 목록과 적용 여부, 그 사유를 문서화한 산출물.[^iso27001]

ISO/IEC 27001:2022 §6.1.3 (d) [^iso27001] 는 SoA 작성을 ISMS 의 **필수 요구사항** 으로 명시한다. SoA 는 ISO/IEC 27001 인증 심사의 핵심 산출물이며, 다음 정보를 포함해야 한다.

| 항목 | 내용 |
|---|---|
| 통제 식별 | ISO/IEC 27001 Annex A 의 93개 통제 [^iso27002] 각각에 대한 식별 |
| 적용 여부 | 적용 / 미적용 결정 |
| 적용 사유 | 적용한 통제가 어떤 위험에 대응하는지 |
| 미적용 사유 | 미적용한 통제의 미적용 사유 (정당화) |
| 구현 상태 | 현재 구현 여부 및 진행 상황 |

### §8-1. ISO/IEC 27001:2022 Annex A 통제 카테고리

ISO/IEC 27002:2022 [^iso27002] 는 통제를 4개 카테고리·총 93개 통제로 재편성하였다 (이전 27002:2013 의 14개 카테고리 114개 통제 대비 변경).

| 카테고리 | 통제 수 | 사례 |
|---|---|---|
| 조직적 통제 (Organizational) | 37개 | 정책·역할·자산관리·접근통제·공급자 관계 |
| 인적 통제 (People) | 8개 | 인력 심사·기밀 유지·교육·징계 |
| 물리적 통제 (Physical) | 14개 | 출입통제·시설·매체 |
| 기술적 통제 (Technological) | 34개 | 인증·암호화·로깅·취약점 관리·안전한 개발 |

### §8-2. 시험 포인트

- SoA 는 ISO/IEC 27001:2022 의 **필수 산출물**.
- ISO/IEC 27002:2022 통제 수 = **93개** (4개 카테고리).
- 통제 미적용 시에도 **사유 문서화** 가 요구됨.

## §9. 단원 요약

| 영역 | 핵심 |
|---|---|
| 위험관리 5단계 | 전략·계획 → 분석 → 평가 → 대책 선정 → 계획 수립 |
| 위험 구성요소 | 3대 = 자산·위협·취약성 / 4대 = + 보호대책 |
| 위험 함수 | 위험 = f (자산, 위협, 취약성) / 잔여 위험 = f (..., 보호대책) |
| DoA | 수용 가능한 목표 위험수준 — 경영진이 결정 |
| 접근방법 4종 | 기준선·비정형·상세·복합 (ISO 27005 권장 = 복합) |
| 분석 방식 | 정량 (ALE = SLE × ARO) / 정성 (델파이·시나리오·순위·퍼지) |
| 처리 전략 4종 | 수용·감소·전가·회피 |
| SoA | ISO/IEC 27001:2022 필수 산출물 — Annex A 93개 통제 |

## §10. 시험 함정 정리

| 함정 | 정답 |
|---|---|
| 위험관리 1단계는 위험 분석이다 | **틀림**. 1단계 = 위험관리 전략 및 계획 수립 |
| 위협만 있으면 위험이 발생한다 | **틀림**. 취약성이 함께 있어야 위험 |
| 잔여 위험은 0 으로 만들 수 있다 | **틀림**. 보호대책 구현 후에도 남음 |
| ALE = SLE + ARO | **틀림**. ALE = SLE **×** ARO (곱) |
| SLE = 자산 가치 × ARO | **틀림**. SLE = 자산 가치 × EF (노출계수) |
| 정성적 분석은 객관적이다 | **틀림**. 평가자 개인에 따라 결과 변동 |
| 보험 가입은 위험 감소이다 | **틀림**. 위험 전가 |
| 방화벽 도입은 위험 회피이다 | **틀림**. 위험 감소 |
| 사업 중단은 위험 수용이다 | **틀림**. 위험 회피 |
| ISO/IEC 27001 통제 수는 114개이다 | **틀림**. 27002:2022 (27001:2022 Annex A) = 93개. 27002:2013 = 114개 |
| ISO/IEC 27005 권장 방법은 상세 위험분석이다 | **틀림**. 복합 접근법 권장 |
| 델파이 기법은 1인 전문가가 수행한다 | **틀림**. **전문가 집단** 의 익명 토론 |
| SoA 는 선택 산출물이다 | **틀림**. ISO/IEC 27001:2022 §6.1.3 (d) 필수 요구사항 |
| IDS 는 예방 통제이다 | **틀림**. **탐지** 통제 |
