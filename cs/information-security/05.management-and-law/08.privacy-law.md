# 5과목 08 — 개인정보보호 관련 법제

> 출제기준 매핑: 2.3 개인정보보호 관련 법제 — 개인정보보호 관련 용어의 정의 / 개인정보보호법의 이해

## §0. 들어가며

본 단원은 5과목에서 출제 비중이 가장 높은 영역인 **[개인정보보호법]** 을 상세히 다룬다. 한국의 개인정보보호 법제는 2020년 데이터 3법 개정 (2020.2.4 공포 / 2020.8.5 시행) 으로 [정보통신망법] 의 개인정보 조항이 [개인정보보호법] 으로 이관되어 통합되었으며, 2023년 3월 14일 일부개정 (2023.9.15 시행, 부분 시행 2024.3.15) 으로 정보주체의 권리 강화·자동화된 결정에 대한 권리 신설 등 추가 변경이 이루어졌다.

본 단원의 모든 1차 출처는 **국가법령정보센터 현행본 (2026.04.30 기준)** 의 [개인정보 보호법] [^pipa], [개인정보 보호법 시행령] [^pipa-enf], [개인정보 보호법 시행규칙] [^pipa-reg], 「개인정보의 안전성 확보조치 기준」 (개인정보보호위원회 고시) [^safety-notice], 「표준 개인정보 처리방침」 (개인정보보호위원회 고시) [^std-policy] 이다. 시험 응시 시점에 정기 개정 여부를 직접 확인해야 한다.

## §2. 핵심 용어의 정의 — §2

[개인정보 보호법] §2 [^pipa] 의 정의는 5과목 시험의 빈출 영역이다. 각 용어의 정확한 정의·구분이 필수이다.

### §2-1. 개인정보 (§2 1호)

> **개인정보** : 살아 있는 개인에 관한 정보로서 다음 각 목의 어느 하나에 해당하는 정보를 말한다. [^pipa]
> 가. 성명, 주민등록번호 및 영상 등을 통하여 개인을 알아볼 수 있는 정보
> 나. 해당 정보만으로는 특정 개인을 알아볼 수 없더라도 다른 정보와 쉽게 결합하여 알아볼 수 있는 정보. 이 경우 쉽게 결합할 수 있는지 여부는 다른 정보의 입수 가능성 등 개인을 알아보는 데 소요되는 시간·비용·기술 등을 합리적으로 고려하여야 한다
> 다. 가목 또는 나목을 §2 1의2호에 따라 가명처리함으로써 원래의 상태로 복원하기 위한 추가 정보의 사용·결합 없이는 특정 개인을 알아볼 수 없는 정보 (이하 **가명정보**)

> *핵심 포인트*: "**살아 있는 개인**" — 사망자는 개인정보 보호법의 대상이 아니다.

### §2-2. 가명정보·익명정보 (§2 1의2호 + §58-2)

| 구분 | 정의 |
|---|---|
| **가명정보** (§2 1의2호 + §2 1호 다목) | 추가 정보의 사용·결합 없이는 특정 개인을 알아볼 수 없는 정보. **가명처리** 의 결과물. 여전히 개인정보보호법의 적용 대상 |
| **익명정보** (§58-2) | 시간·비용·기술 등을 합리적으로 고려할 때 다른 정보를 사용하여도 더 이상 개인을 알아볼 수 없는 정보. **개인정보보호법 적용 제외** |

가명정보는 통계작성·과학적 연구·공익적 기록보존 등의 목적에 한해 정보주체 동의 없이 처리 가능 (§28-2). 익명정보는 개인정보 자체가 아니므로 자유롭게 처리 가능.

### §2-3. 민감정보 (§23)

> **민감정보** : 사상·신념, 노동조합·정당의 가입·탈퇴, 정치적 견해, 건강, 성생활 등에 관한 정보, 그 밖에 정보주체의 사생활을 현저히 침해할 우려가 있는 개인정보로서 대통령령 (시행령 §18) 으로 정하는 정보. [^pipa]

[개인정보 보호법 시행령] §18 [^pipa-enf] 의 추가 민감정보:

| 추가 민감정보 |
|---|
| 유전정보 |
| 범죄경력자료 ([형의 실효 등에 관한 법률] §2 5호) |
| **개인의 신체적·생리적·행동적 특징에 관한 정보로서 특정 개인을 알아볼 목적으로 일정한 기술적 수단을 통해 생성된 정보** (생체인식정보) |
| 인종·민족 정보 |

처리 요건: §23 — **별도 동의** 또는 **법령에 명시 또는 법령에서 허용** 한 경우에만 처리 가능.

### §2-4. 고유식별정보 (§24)

> **고유식별정보** : 법령에 따라 개인을 고유하게 식별하기 위하여 부여된 식별정보 중 대통령령 (시행령 §19) 으로 정하는 정보. [^pipa]

[개인정보 보호법 시행령] §19 [^pipa-enf] 의 고유식별정보 4종:

| 고유식별정보 | 법적 근거 |
|---|---|
| 주민등록번호 | [주민등록법] |
| 여권번호 | [여권법] |
| 운전면허번호 | [도로교통법] |
| 외국인등록번호 | [출입국관리법] |

처리 요건: §24 — **별도 동의** 또는 **법령에 명시** 가 있는 경우에만 처리 가능. 특히 **주민등록번호** 는 §24-2 에 의해 더욱 엄격한 요건이 적용된다 (법률·시행령에 명시된 경우 또는 정보주체의 보호를 위해 명백히 필요한 경우만).

### §2-5. 정보주체·개인정보처리자·개인정보취급자

| 용어 | 정의 (§2) | 사례 |
|---|---|---|
| **정보주체** (§2 3호) | 처리되는 정보에 의하여 알아볼 수 있는 사람으로서 그 정보의 **주체가 되는 사람** | 회원가입한 이용자, 환자, 직원 |
| **개인정보처리자** (§2 5호) | 업무를 목적으로 개인정보파일을 운용하기 위하여 스스로 또는 다른 사람을 통하여 개인정보를 처리하는 **공공기관·법인·단체 및 개인 등** | 회사·병원·학교·정부기관 |
| 개인정보취급자 (§28) | 개인정보처리자의 **지휘·감독을 받아** 개인정보를 처리하는 임직원·파견근로자·시간제근로자 등 | 개인정보 시스템 운영자, 콜센터 직원 |
| 처리 (§2 2호) | 개인정보의 수집·생성·연계·연동·기록·저장·보유·가공·편집·검색·출력·정정·복구·이용·제공·공개·파기·그 밖에 이와 유사한 행위 | – |

> 시험에서 가장 자주 혼동되는 부분: **개인정보처리자 ≠ 개인정보취급자**. 처리자 = 사용자 (조직), 취급자 = 처리자의 지휘 하의 직원.

### §2-6. 시험 포인트

- 개인정보 = **살아 있는 개인** 의 정보 (사망자 미포함).
- 가명정보 ≠ 익명정보 — 가명정보는 개인정보, 익명정보는 적용 제외.
- 민감정보 = §23 + 시행령 §18 (생체인식·유전·범죄경력·인종 추가).
- 고유식별정보 4종 = **주민등록번호·여권번호·운전면허번호·외국인등록번호**.
- **주민등록번호**는 §24-2 의 가장 엄격한 요건.
- 개인정보처리자 ≠ 개인정보취급자.

## §3. 개인정보 처리 원칙 — §3

[개인정보 보호법] §3 [^pipa] 은 8개 처리 원칙을 명시한다.

### §3-1. 8대 원칙

| 호 | 원칙 |
|---|---|
| 1호 | 처리 목적을 명확하게 하고 그 목적에 필요한 범위에서 **최소한의 개인정보** 만을 적법하고 정당하게 수집 |
| 2호 | 개인정보의 처리 목적에 필요한 범위에서 **적합** 하게 처리하고, 그 목적 외의 용도로 활용하지 아니할 것 |
| 3호 | 개인정보의 처리 목적에 필요한 범위에서 **정확성·완전성·최신성** 보장 |
| 4호 | 처리 방법 및 종류 등에 따라 정보주체의 권리가 침해받을 가능성과 그 위험 정도를 고려하여 **안전하게 관리** |
| 5호 | **처리방침 등 공개** 하고 열람청구권 등 정보주체의 권리 보장 |
| 6호 | 정보주체의 **사생활 침해를 최소화** 하는 방법으로 처리 |
| 7호 | 가능한 경우 **익명·가명 처리 우선** (목적 달성 가능 범위에서) |
| 8호 | 이 법 및 관련 법령에서 규정하고 있는 책임과 의무를 준수하고 **신뢰** 받기 위하여 노력 |

### §3-2. OECD 개인정보 8원칙과의 관계

[개인정보 보호법] §3 의 8원칙은 OECD Privacy Guidelines (1980 제정, 2013 개정) [^oecd-priv] 의 8원칙을 한국법으로 수용한 것이다.

| OECD 8원칙 | 개보법 §3 매핑 |
|---|---|
| 수집제한 (Collection Limitation) | 1호 (최소 수집·적법·정당) |
| 정보 정확성 (Data Quality) | 3호 (정확성·완전성·최신성) |
| 목적 명확화 (Purpose Specification) | 1호·2호 (목적 명확화·목적 외 사용 금지) |
| 이용 제한 (Use Limitation) | 2호 (목적 외 활용 금지) |
| 안전성 확보 (Security Safeguards) | 4호 (안전한 관리) |
| 공개 (Openness) | 5호 (처리방침 등 공개) |
| 개인 참여 (Individual Participation) | 5호 (정보주체 권리 보장) |
| 책임 (Accountability) | 8호 (책임·의무 준수) |

### §3-3. 시험 포인트

- 개보법 §3 = **8개 원칙**.
- OECD 8원칙 = 1980 제정 / 2013 개정.
- 가능한 경우 **익명·가명 처리 우선** (§3 7호).
- 최소 수집의 원칙 (§3 1호) — 목적에 필요한 범위.

## §4. 개인정보의 수집·이용·제공

### §4-1. §15 — 수집·이용

[개인정보 보호법] §15 [^pipa] 는 개인정보 수집·이용의 6 가지 적법 사유를 정의한다.

| 호 | 적법 사유 |
|---|---|
| **1호** | **정보주체의 동의** |
| 2호 | 법률에 특별한 규정이 있거나 법령상 의무 이행을 위해 불가피한 경우 |
| 3호 | 공공기관이 법령 등에서 정하는 소관 업무의 수행을 위하여 불가피한 경우 |
| 4호 | 정보주체와 체결한 **계약의 이행 또는 체결을 위한 절차** 에 필요한 경우 |
| 5호 | 정보주체 또는 그 법정대리인이 의사표시를 할 수 없는 상태이거나 주소불명 등으로 사전 동의를 받을 수 없는 경우로서 명백히 정보주체 또는 제3자의 **급박한 생명·신체·재산의 이익** 을 위하여 필요한 경우 |
| 6호 | 개인정보처리자의 **정당한 이익** 을 달성하기 위하여 필요한 경우로서 명백하게 정보주체의 권리보다 우선하는 경우 (이 경우 합리적 범위 초과 금지) |
| 7호 | 공중위생 등 공공의 안전과 안녕을 위하여 긴급히 필요한 경우 |

§15 ②항: 동의를 받을 때는 다음 사항을 알리고 변경 시에도 알린다.

| 통지 항목 |
|---|
| 1. 개인정보의 수집·이용 목적 |
| 2. 수집하려는 개인정보의 항목 |
| 3. 개인정보의 보유·이용 기간 |
| 4. 동의를 거부할 권리가 있다는 사실 및 동의 거부에 따른 불이익이 있는 경우 그 불이익의 내용 |

### §4-2. §16 — 수집 제한

[개인정보 보호법] §16 [^pipa]:

| 항 | 내용 |
|---|---|
| §16 ①항 | 개인정보처리자는 §15 ①항 각 호의 어느 하나에 해당하여 개인정보를 수집하는 경우에는 그 **목적에 필요한 최소한의 개인정보** 를 수집 |
| §16 ②항 | 개인정보처리자는 정보주체의 **동의를 받아 개인정보를 수집하는 경우** 필요한 최소한의 정보 외의 개인정보 수집에는 동의하지 아니할 수 있다는 사실을 구체적으로 알리고 동의를 받음 |
| §16 ③항 | 개인정보처리자는 정보주체가 필요한 최소한의 정보 외의 개인정보 수집에 동의하지 아니한다는 이유로 정보주체에게 **재화 또는 서비스의 제공을 거부하여서는 아니 된다** |

### §4-3. §17 — 제3자 제공

[개인정보 보호법] §17 [^pipa]:

| 항 | 내용 |
|---|---|
| §17 ①항 | 개인정보처리자는 다음 각 호의 어느 하나에 해당되는 경우에는 정보주체의 개인정보를 제3자에게 제공할 수 있음. 1호 정보주체의 동의를 받은 경우 / 2호 §15 ①항 2·3·5·6·7호에 따라 개인정보를 수집한 목적 범위에서 개인정보를 제공하는 경우 |
| §17 ②항 | 동의 시 다음 알림: 1. 받는 자 / 2. 받는 자의 이용목적 / 3. 제공 항목 / 4. 받는 자의 보유·이용 기간 / 5. 동의 거부권·불이익 |

### §4-4. §18 — 목적 외 이용·제공의 제한

[개인정보 보호법] §18 [^pipa] 는 수집 목적 외의 이용·제공을 원칙적으로 금지한다.

| 항 | 내용 |
|---|---|
| §18 ①항 | 개인정보처리자는 개인정보를 §15 ①항 및 §39-3 ①항·②항에 따른 범위를 초과하여 이용하거나 §17 ①항·③항·④항에 따른 범위를 초과하여 제3자에게 제공하여서는 아니 됨 |
| §18 ②항 | 다만 다음 사유 (§18 ②항 1~9호) 의 경우 목적 외 이용·제공 가능: 1. 정보주체로부터 별도의 동의를 받은 경우 / 2. 다른 법률에 특별한 규정이 있는 경우 / 3. 정보주체 또는 그 법정대리인이 의사표시를 할 수 없는 상태이거나 주소불명 등으로 사전 동의를 받을 수 없는 경우로서 명백히 정보주체 또는 제3자의 급박한 생명·신체·재산의 이익을 위하여 필요한 경우 / 4~9호 (공공기관·통계·연구·범죄수사·법원 재판·형 집행 등) |

### §4-5. §15 vs §17 vs §18 비교

| 비교 항목 | §15 (수집·이용) | §17 (제공) | §18 (목적 외) |
|---|---|---|---|
| 적용 단계 | 처리자가 직접 수집·이용 | 제3자에게 전달 | 수집 목적과 다른 용도 |
| 동의 항목 | 4 가지 (목적·항목·기간·거부권) | 5 가지 (받는 자·이용목적·항목·기간·거부권) | 별도 동의 + 추가 제한 |
| 적법 사유 수 | 7 가지 | 2 가지 (동의 + §15 일부) | 9 가지 |

### §4-6. §22-2 — 14세 미만 아동의 개인정보 처리에 관한 특례

[개인정보 보호법] §22-2 [^pipa]:

| 항 | 내용 |
|---|---|
| §22-2 ①항 | 개인정보처리자는 만 **14세 미만 아동** 의 개인정보를 처리하기 위하여 동의를 받아야 할 때에는 그 **법정대리인의 동의** 를 받아야 하며, 법정대리인이 동의하였는지를 확인하여야 함 |
| §22-2 ②항 | 법정대리인의 동의를 받기 위하여 필요한 최소한의 정보는 법정대리인의 동의 없이 해당 아동으로부터 직접 수집할 수 있음 (예: 법정대리인의 성명·연락처) |

### §4-7. 시험 포인트

- §15 = 수집·이용 / §17 = 제3자 제공 / §18 = 목적 외 이용·제공.
- §16 ③항 = **최소한 외 정보 동의 거부 시 서비스 거부 금지**.
- §17 ②항 동의 항목 = **5개** (§15 보다 1개 많음 — "받는 자" 추가).
- 14세 미만 아동 = **법정대리인 동의** (§22-2).
- 주민등록번호 처리 = §24-2 (가장 엄격).

## §5. 가명정보 특례 — §28-2~§28-7 (2020 데이터 3법 신설)

2020년 데이터 3법 개정으로 신설된 가명정보 처리 특례는 한국 개인정보보호법의 가장 큰 변화이다.

### §5-1. §28-2 — 가명정보의 처리

> "개인정보처리자는 통계작성·과학적 연구·공익적 기록보존 등을 위하여 **정보주체의 동의 없이** 가명정보를 처리할 수 있다." [^pipa]

가능 목적 (§28-2 ①항):

| 목적 |
|---|
| 통계작성 (상업적 목적 포함) |
| 과학적 연구 (산업적 연구 포함) |
| 공익적 기록보존 |

### §5-2. §28-3 — 가명정보의 결합 제한

[개인정보 보호법] §28-3 [^pipa]:

| 항 | 내용 |
|---|---|
| §28-3 ①항 | 통계작성·과학적 연구·공익적 기록보존 등을 위한 서로 다른 개인정보처리자 간의 가명정보의 결합은 **개인정보보호위원회 또는 관계 중앙행정기관의 장이 지정하는 전문기관** 이 수행 |
| §28-3 ②항 | 결합을 수행한 기관 외부로 결합된 정보를 반출하려는 개인정보처리자는 가명정보 또는 익명정보로 처리한 후 **전문기관의 장의 승인** 을 받아야 함 |

### §5-3. §28-4 — 안전조치의무

가명정보 처리 시 다음 조치가 의무화된다.

| 의무 |
|---|
| 가명정보의 안전성 확보에 필요한 기술적·관리적·물리적 조치 |
| 가명정보 처리 목적·기간·조치 등 처리 내용 기록 및 보관 |
| 추가 정보의 분리 보관·관리 |

### §5-4. §28-5 — 가명정보 처리 시 금지의무

[개인정보 보호법] §28-5 [^pipa]:

| 항 | 내용 |
|---|---|
| §28-5 ①항 | **누구든지 특정 개인을 알아보기 위한 목적으로 가명정보를 처리하여서는 아니 됨** |
| §28-5 ②항 | 개인정보처리자는 가명정보를 처리하는 과정에서 특정 개인을 알아볼 수 있는 정보가 생성된 경우에는 즉시 해당 정보의 처리를 중지하고, 지체 없이 회수·파기 |

### §5-5. §28-6 — 과징금

가명정보 특례 위반 시 매출액의 3% 이하 과징금 부과 가능 (§28-6).

### §5-6. §28-7 — 적용제외

가명정보에 대해서는 다음 조항의 적용이 제외된다 (§28-7).

| 적용제외 조항 |
|---|
| §20 (정보주체 이외로부터 수집한 개인정보의 수집 출처 등 통지) |
| §21 (개인정보의 파기) |
| §27 (영업양도 등에 따른 개인정보의 이전 제한) |
| §34 ①항 (유출 등의 통지) — 단 정보주체 통지 의무는 일부 제외 |
| §35 (개인정보의 열람) |
| §36 (개인정보의 정정·삭제) |
| §37 (개인정보의 처리정지 등) |

> 즉 가명정보는 통상의 개인정보 권리 (열람·정정·삭제·처리정지) 가 적용되지 않는다.

### §5-7. 시험 포인트

- 가명정보 특례 = **2020 데이터 3법** 신설 (§28-2~§28-7).
- §28-2 = 통계작성·과학적 연구·공익적 기록보존 — **동의 없이** 가능.
- §28-3 = 서로 다른 처리자 간 결합 = **지정 전문기관** 수행.
- §28-5 = **특정 개인 식별 시도 금지**.
- §28-7 = 가명정보는 정보주체 권리 (열람·정정·삭제 등) 적용 제외.

## §6. 개인정보 보호책임자 (CPO) — §31

[개인정보 보호법] §31 [^pipa] 은 CPO 지정 의무를 부과한다 (5과목 01 §7-2 cross-ref).

### §6-1. §31 ①항 — CPO 지정 의무

> "개인정보처리자는 개인정보의 처리에 관한 업무를 총괄해서 책임질 **개인정보 보호책임자** 를 지정하여야 한다." [^pipa]

### §6-2. §31 ③항 — CPO 의 업무

| 호 | 업무 |
|---|---|
| 1호 | 개인정보 보호 계획의 수립 및 시행 |
| 2호 | 개인정보 처리 실태 및 관행의 정기적인 조사 및 개선 |
| 3호 | 개인정보 처리와 관련한 불만의 처리 및 피해 구제 |
| 4호 | 개인정보 유출 및 오·남용 방지를 위한 내부통제시스템의 구축 |
| 5호 | 개인정보 보호 교육 계획의 수립 및 시행 |
| 6호 | 개인정보파일의 보호 및 관리·감독 |
| 7호 | 그 밖에 개인정보의 적절한 처리를 위하여 대통령령으로 정한 업무 |

### §6-3. CISO vs CPO 비교

(5과목 01 §7-2 의 표 재인용 + 보강)

| 비교 항목 | CISO | CPO |
|---|---|---|
| 법적 근거 | [정통망법] §45-3 | [개인정보보호법] §31 |
| 보호 대상 | 정보보호 일반 (CIA) | 개인정보 |
| 임명 의무자 | 일정 규모 이상 정보통신서비스 제공자 | **모든 개인정보처리자** |
| 신고 의무 | 과기정통부 신고 | (지정만, 신고 일반 의무 아님) |
| 자격 요건 | 정보보호 또는 정보기술 분야 일정 경력 (시행령) | 사업주 또는 대표자, 임원 등 (시행령 §32) |

### §6-4. 시험 포인트

- CPO 법적 근거 = [개인정보보호법] **§31**.
- CPO 지정 의무 = **모든 개인정보처리자** (CISO 와 달리 규모 제한 없음).
- CPO 업무 = 7개 (보호계획·실태조사·민원처리·내부통제·교육·파일관리·기타).

## §7. 개인정보 처리방침 — §30

### §7-1. §30 ①항 — 처리방침 수립·공개 의무

[개인정보 보호법] §30 [^pipa]:

> "개인정보처리자는 다음 각 호의 사항이 포함된 개인정보의 처리 방침 (이하 '개인정보 처리방침') 을 정하여야 한다. 이 경우 공공기관은 §32 에 따라 등록 대상이 되는 개인정보파일에 대하여 개인정보 처리방침을 정한다." [^pipa]

### §7-2. 처리방침의 필수 포함 항목

[개인정보 보호법] §30 ①항 [^pipa] 의 필수 포함 항목:

| 호 | 항목 |
|---|---|
| 1호 | 개인정보의 처리 목적 |
| 2호 | 개인정보의 처리 및 보유 기간 |
| 3호 | 개인정보의 제3자 제공에 관한 사항 (해당하는 경우만) |
| 3의2호 | 개인정보의 파기절차 및 파기방법 |
| 4호 | 개인정보처리의 위탁에 관한 사항 (해당하는 경우만) |
| 4의2호 | 정보주체와 법정대리인의 권리·의무 및 그 행사방법 |
| 5호 | 개인정보 보호책임자의 성명 또는 개인정보 보호 업무 및 관련 고충사항을 처리하는 부서의 명칭과 전화번호 등 연락처 |
| 6호 | 인터넷 접속정보파일 등 개인정보를 자동으로 수집하는 장치의 설치·운영 및 그 거부에 관한 사항 (해당하는 경우만) |
| 7호 | 그 밖에 개인정보의 처리에 관하여 대통령령으로 정한 사항 |

### §7-3. §30 ②항 — 공개 방법

처리방침은 다음 방법으로 정보주체가 쉽게 확인할 수 있도록 공개해야 한다.

- 개인정보처리자의 인터넷 홈페이지에 게재
- 인터넷 홈페이지가 없는 경우, 사업장 등 정보주체에게 보일 수 있는 곳에 게시 또는 비치 등 (시행령 §31)

### §7-4. 표준 개인정보 처리방침

「표준 개인정보 처리방침」 (개인정보보호위원회 고시) [^std-policy] 은 처리방침 작성을 위한 표준 양식을 제공한다. 시험에서는 표준 양식의 항목과 §30 ①항의 필수 항목 매핑이 출제된다.

### §7-5. 시험 포인트

- 처리방침 법적 근거 = **[개인정보보호법] §30**.
- 필수 항목 = §30 ①항의 1호~7호.
- 공개 방법 = 홈페이지 게재 또는 사업장 게시.
- 표준 처리방침 = 개인정보보호위원회 고시.

## §8. 안전성 확보조치 — §29

### §8-1. §29 — 안전조치의무

[개인정보 보호법] §29 [^pipa]:

> "개인정보처리자는 개인정보가 분실·도난·유출·위조·변조 또는 훼손되지 아니하도록 **내부 관리계획 수립·접속기록의 보관 등 대통령령으로 정하는 바에 따라 안전성 확보에 필요한 기술적·관리적 및 물리적 조치** 를 하여야 한다." [^pipa]

### §8-2. 「개인정보의 안전성 확보조치 기준」 (개인정보보호위원회 고시)

「개인정보의 안전성 확보조치 기준」 [^safety-notice] 은 §29 의 구체적 시행 기준을 명시한다. 핵심 항목 (작성 시점 현행본 기준):

| 영역 | 핵심 통제 |
|---|---|
| 내부 관리계획 | 개인정보 보호조직, 보호책임자 지정, 처리방침, 위험분석, 점검·훈련 |
| 접근 권한의 관리 | 권한 부여·변경·말소 기록, 5년 보관 (또는 시행령 기준), 비밀번호 작성 규칙 |
| 접근 통제 | 안전한 인증, 외부 접근 통제, 무선망 보호, 망분리 (일정 규모 이상) |
| 개인정보의 암호화 | 정보통신망 전송·고유식별정보 저장 시 암호화, 비밀번호는 일방향 (해시) 저장 |
| 접속기록의 보관·점검 | 1년 이상 (시행령 기준), 위·변조 방지, 정기 점검 |
| 악성프로그램 방지 | 백신·보안 업데이트 |
| 물리적 안전조치 | 보호구역 지정, 매체 통제, 출력·복사 통제 |

> *주의*: 위 기준의 구체적 수치 (보관 기간·암호화 적용 범위·망분리 의무 규모 등) 는 「개인정보의 안전성 확보조치 기준」 고시의 정기 개정으로 변경된다. 시험 응시 시점에 개인정보보호위원회 (https://www.pipc.go.kr/) 에서 직접 확인 필요.

### §8-3. 시험 포인트

- 안전성 확보조치 법적 근거 = **[개인정보보호법] §29 + 시행령 §30 + 「안전성 확보조치 기준」 고시**.
- 비밀번호 저장 = **일방향 암호화 (해시)** 의무.
- 정보통신망 전송 = **암호화** 의무.
- 접속기록 보관 = **1년 이상** (시행령 기준 — 정기 개정 가능).

## §9. 정보주체의 권리

### §9-1. §4 — 정보주체의 권리 (총칙)

[개인정보 보호법] §4 [^pipa]:

| 호 | 권리 |
|---|---|
| 1호 | 개인정보의 처리에 관한 정보를 제공받을 권리 |
| 2호 | 개인정보의 처리에 관한 동의 여부, 동의 범위 등을 선택하고 결정할 권리 |
| 3호 | 개인정보의 처리 여부를 확인하고 개인정보에 대하여 열람 (사본의 발급 포함) 및 전송을 요구할 권리 |
| 4호 | 개인정보의 처리 정지·정정·삭제·파기 요구 권리 |
| 5호 | 개인정보의 처리로 인하여 발생한 피해를 신속하고 공정한 절차에 따라 구제받을 권리 |
| 6호 | 완전히 자동화된 개인정보 처리에 따른 결정을 거부하거나 그에 대한 설명 등을 요구할 권리 |

### §9-2. §35 — 열람권

[개인정보 보호법] §35 [^pipa]:

| 항 | 내용 |
|---|---|
| §35 ①항 | 정보주체는 개인정보처리자가 처리하는 자신의 개인정보에 대한 열람을 해당 개인정보처리자에게 요구할 수 있음 |
| §35 ②항 | 공공기관에 대해서는 개인정보보호위원회에 대해서도 열람 요구 가능 |
| §35 ③항 | 개인정보처리자는 요구를 받은 날부터 **10일 이내** 에 정보주체가 해당 개인정보를 열람할 수 있도록 함 |
| §35 ④항 | 일정 사유 시 열람 제한·거절 가능 (법률 규정·국가안보·공공이익 등) |

### §9-3. §35-2 — 개인정보 전송요구권 (마이데이터, 2023 신설)

[개인정보 보호법] §35-2 [^pipa] (2023.3.14 일부개정으로 신설):

> 정보주체는 개인정보처리자에 대하여 **본인에 관한 개인정보를 본인 또는 본인이 지정한 다른 개인정보처리자나 개인정보관리 전문기관에게 전송할 것을 요구** 할 수 있다.

이는 "마이데이터 (MyData)" 의 법적 근거로, 정보주체가 자신의 개인정보를 다른 사업자로 이동시킬 수 있는 권리이다.

### §9-4. §36 — 정정·삭제

[개인정보 보호법] §36 [^pipa]:

| 항 | 내용 |
|---|---|
| §36 ①항 | 정보주체는 개인정보처리자에게 자신의 개인정보의 **정정·삭제** 를 요구할 수 있음 (다만 다른 법령에서 그 개인정보가 수집 대상으로 명시되어 있는 경우는 그 삭제를 요구할 수 없음) |
| §36 ②항 | 처리자는 요구를 받은 경우 **10일 이내** 에 조사하여 결과를 통지 |

### §9-5. §37 — 처리정지

[개인정보 보호법] §37 [^pipa]:

| 항 | 내용 |
|---|---|
| §37 ①항 | 정보주체는 개인정보처리자에 대하여 자신의 개인정보 처리의 정지를 요구할 수 있음 |
| §37 ②항 | 일정 사유 시 처리정지 거절 가능 (법률 특별 규정·생명·신체·재산·다른 사람 권리·공공기관 소관업무 등) |
| §37 ③항 | 거절 시 사유 통지 |

### §9-6. §37-2 — 자동화된 결정에 대한 권리 (2023 신설)

[개인정보 보호법] §37-2 [^pipa] (2023.3.14 일부개정으로 신설, 2024.3.15 시행):

> 정보주체는 **완전히 자동화된 시스템 (인공지능 기술 적용 시스템 포함) 으로 개인정보를 처리하여 이루어지는 결정** 으로서 자신의 권리 또는 의무에 중대한 영향을 미치는 경우에는 해당 결정을 **거부** 하거나 그에 대한 **설명 등** 을 개인정보처리자에게 요구할 수 있다.

| 권리 | 내용 |
|---|---|
| 거부권 | 자동화 결정 자체의 거부 |
| 설명요구권 | 결정의 기준·근거 등에 대한 설명 요구 |
| 인적 개입 요구 | 자동화 결정에 사람의 개입 요구 |

이는 EU GDPR §22 (자동화된 의사결정의 권리) [^gdpr-22] 와 유사한 권리이다.

### §9-7. §39 — 손해배상책임

[개인정보 보호법] §39 [^pipa]:

| 항 | 내용 |
|---|---|
| §39 ①항 | 정보주체는 개인정보처리자가 이 법을 위반한 행위로 손해를 입은 경우 손해배상을 청구할 수 있음. 이 경우 처리자는 **고의 또는 과실이 없음을 입증** 하지 않으면 책임 면제 불가 (입증책임 전환) |
| §39 ②항 | 손해액 산정이 곤란한 경우 변론 전체 취지와 증거조사 결과에 의해 인정되는 모든 사정을 종합하여 상당하다고 인정되는 금액을 손해액으로 산정 가능 |
| §39 ③항 | **법정손해배상** — 정보주체는 1인당 **300만원 이하** 의 범위에서 법정손해배상 청구 가능 (실제 손해액 입증 불요) |
| §39 ④항 | **징벌적 손해배상** — 고의 또는 중대한 과실로 손해 발생 시 손해액의 **5배** 를 넘지 않는 범위에서 배상 (2023 개정으로 강화) |

### §9-8. 시험 포인트

- §35 (열람) 처리 기한 = **10일 이내**.
- §36 (정정·삭제) 처리 기한 = **10일 이내**.
- §35-2 (전송요구권) = 마이데이터 — **2023 신설**.
- §37-2 (자동화 결정) = **2023 신설** / **2024.3.15 시행**.
- §39 ③항 법정손해배상 = **1인당 300만원 이하**.
- §39 ④항 징벌적 손해배상 = **손해액의 5배 이하** (2023 강화).

## §10. 개인정보 영향평가 — §33

[개인정보 보호법] §33 [^pipa] 는 PIA 의무를 부과한다 (5과목 05 §9 cross-ref).

### §10-1. §33 ①항 — PIA 의무

> "공공기관의 장은 대통령령으로 정하는 기준에 해당하는 개인정보파일의 운용으로 인하여 정보주체의 개인정보 침해가 우려되는 경우에는 그 위험 요인의 분석과 개선사항 도출을 위한 평가 (이하 '영향평가') 를 하고 그 결과를 개인정보보호위원회에 제출하여야 한다." [^pipa]

### §10-2. PIA 의무 대상 (시행령 §35)

[개인정보 보호법 시행령] §35 [^pipa-enf] 의 의무 대상 (공공기관 중심):

| 분류 | 기준 |
|---|---|
| 1호 | 5만 명 이상의 정보주체에 관한 민감정보 또는 고유식별정보의 처리가 수반되는 개인정보파일 |
| 2호 | 해당 공공기관의 내부 또는 외부의 다른 개인정보파일과 연계하려는 경우로서 연계 결과 50만 명 이상의 정보주체에 관한 개인정보가 포함되는 개인정보파일 |
| 3호 | 100만 명 이상의 정보주체에 관한 개인정보파일 |

> *주의*: 위 수치는 시행령 §35 의 작성 시점 현행본 기준이며 정기 개정으로 변경 가능.

### §10-3. 시험 포인트

- PIA 법적 근거 = [개인정보보호법] **§33**.
- 의무 대상 = **공공기관** + 시행령 §35 의 일정 규모 이상. **민간은 자율**.
- PIA 결과 제출 = **개인정보보호위원회**.

## §11. 개인정보 유출 통지·신고 — §34

### §11-1. §34 ①항 — 정보주체 통지

[개인정보 보호법] §34 [^pipa]:

> "개인정보처리자는 개인정보가 분실·도난·유출 (이하 '유출 등') 되었음을 알게 되었을 때에는 **지체 없이** 해당 정보주체에게 다음 각 호의 사항을 알려야 한다." [^pipa]

| 호 | 통지 항목 |
|---|---|
| 1호 | 유출 등이 된 개인정보의 항목 |
| 2호 | 유출 등이 된 시점과 그 경위 |
| 3호 | 유출 등으로 인하여 발생할 수 있는 피해를 최소화하기 위하여 정보주체가 할 수 있는 방법 등에 관한 정보 |
| 4호 | 개인정보처리자의 대응조치 및 피해 구제절차 |
| 5호 | 정보주체에게 피해가 발생한 경우 신고 등을 접수할 수 있는 담당부서 및 연락처 |

### §11-2. §34 ③항 — 위원회 신고

[개인정보 보호법] §34 ③항 [^pipa]:

> "개인정보처리자는 대통령령으로 정한 규모 이상의 개인정보가 유출 등이 된 경우에는 §34 ①항·②항에 따른 통지 및 조치 결과를 **지체 없이 개인정보보호위원회 또는 한국인터넷진흥원에 신고** 하여야 한다."

[개인정보 보호법 시행령] §39 [^pipa-enf] 의 신고 대상 규모 (작성 시점 현행본 기준):

| 분류 | 기준 |
|---|---|
| 1호 | **1천 명 이상** 의 정보주체에 관한 개인정보가 유출 등이 된 경우 |
| 2호 | 민감정보 또는 고유식별정보가 유출 등이 된 경우 (규모 무관) |
| 3호 | 개인정보처리자의 처리시스템 또는 개인정보 취급자가 개인정보 처리에 사용하는 정보기기 등에 대한 외부로부터의 불법적인 접근에 의해 개인정보가 유출 등이 된 경우 (규모 무관) |

### §11-3. 시험 포인트

- 정보주체 통지 의무 = **§34 ①항** (모든 유출 시 — 규모 무관).
- 위원회·KISA 신고 의무 = **§34 ③항** + 시행령 §39 (1천 명 이상 / 민감·고유 / 외부 침해).
- 통지 항목 = **5개** (항목·시점·피해 최소화·대응조치·연락처).
- 통지·신고 시기 = **지체 없이**.

## §12. 개인정보보호위원회 — §7~§14

### §12-1. 개인정보보호위원회의 지위

[개인정보 보호법] §7 [^pipa]:

> "개인정보 보호에 관한 사무를 독립적으로 수행하기 위하여 국무총리 소속으로 개인정보보호위원회 (이하 '보호위원회') 를 둔다. 보호위원회는 「정부조직법」 §2 에 따른 **중앙행정기관** 으로 본다." [^pipa]

2020년 데이터 3법 개정으로 보호위원회는 **중앙행정기관** 으로 격상되었다 (이전: 대통령 소속 합의제 행정기관).

### §12-2. 위원회의 구성

[개인정보 보호법] §7-2 [^pipa]:

| 구성 | 인원 |
|---|---|
| 위원장 | 1명 (장관급) |
| 부위원장 | 1명 (차관급) |
| 위원 | 7명 (총 9명) |

### §12-3. 위원회의 소관 사무 — §7-8

[개인정보 보호법] §7-8 [^pipa]:

| 호 | 사무 |
|---|---|
| 1호 | 개인정보의 보호와 관련된 법령의 개선에 관한 사항 |
| 2호 | 개인정보 보호와 관련된 정책·제도·계획의 수립·집행에 관한 사항 |
| 3호 | 정보주체의 권리침해에 대한 조사 및 그에 따른 처분에 관한 사항 |
| 4호 | 개인정보의 처리와 관련한 고충처리·권리구제 및 개인정보에 관한 분쟁의 조정 |
| 5호 | 개인정보 보호를 위한 국제기구 및 외국 개인정보 보호기구와의 교류·협력 |
| 6호 | 개인정보 보호에 관한 법령·정책·제도·실태 등의 조사·연구·교육 및 홍보에 관한 사항 |
| 7호 | 개인정보 보호에 관한 기술 개발의 지원·보급 및 전문인력의 양성에 관한 사항 |
| 8호 | 그 밖에 이 법 또는 다른 법령에 따라 보호위원회의 사무로 규정된 사항 |

### §12-4. 시험 포인트

- 보호위원회 = **국무총리 소속 중앙행정기관** (2020 데이터 3법 격상).
- 위원회 구성 = **위원장 1 + 부위원장 1 + 위원 7 = 총 9명**.
- 소관 사무 = 8가지 (법령·정책·조사·분쟁조정·국제협력·홍보·기술 등).

## §13. 처벌 조문

[개인정보 보호법] 제9장 (§70~§76) 의 핵심 처벌 조문:

| 위반 행위 | 조문 | 형량 (작성 시점 현행본 기준) |
|---|---|---|
| 거짓 동의 취득·누설·유출 등 (§59 위반) | §71 | 5년 이하 / 5천만원 이하 |
| 동의 없는 수집·이용·제공 (§15·§17·§18 위반) | §71 | 5년 이하 / 5천만원 이하 |
| 가명정보 특정 개인 식별 시도 (§28-5 ①항 위반) | §71 | 5년 이하 / 5천만원 이하 |
| 안전성 확보조치 미이행 (§29 위반) — 유출 시 | §73 | 2년 이하 / 2천만원 이하 |
| 처리방침 미공개 (§30 위반) | §75 | 과태료 |
| CPO 미지정 (§31 위반) | §75 | 과태료 |
| 개인정보 영향평가 미수행 (§33 위반, 공공기관) | §75 | 과태료 |
| 유출 통지·신고 미이행 (§34 위반) | §75 | 과태료 |
| 양벌규정 (§74) | §74 | 법인·개인·사용자에게 동시 처벌 |

### §13-1. 과징금 — §64-2

[개인정보 보호법] §64-2 [^pipa] (2023 개정 강화):

> "개인정보보호위원회는 개인정보처리자가 다음 각 호의 어느 하나에 해당하는 경우에는 해당 개인정보처리자에 대하여 **전체 매출액의 100분의 3 이하** 에 해당하는 금액을 과징금으로 부과할 수 있다."

> *주의*: 과징금 상한 (전체 매출액의 3%) 은 2023.3.14 개정으로 강화된 수치이며, 시험 응시 시점에 [개인정보보호법] §64-2 현행본 직접 확인 필요.

### §13-2. 시험 포인트

- §59 (금지행위) 위반 = §71 / **5년 이하 / 5천만원 이하**.
- §29 (안전성 확보조치) 위반 = §73 / **2년 이하 / 2천만원 이하**.
- §30·§31·§33·§34 위반 = §75 / **과태료**.
- §64-2 과징금 = **매출액의 3% 이하** (2023 강화).

## §15. 시험 함정 정리

| 함정 | 정답 |
|---|---|
| 개인정보 = 사망자 정보 포함 | **틀림**. **살아 있는 개인** 의 정보 (§2 1호) |
| 가명정보 = 익명정보 | **틀림**. 가명정보 = 개인정보 (적용 대상) / 익명정보 = 적용 제외 (§58-2) |
| 민감정보 = §24 | **틀림**. 민감정보 = **§23** / 고유식별정보 = §24 / 주민등록번호 = §24-2 |
| 고유식별정보 = 4종 (주민등록·여권·운전면허·외국인등록) | **맞음** ([개인정보 보호법 시행령] §19) |
| 14세 미만 아동 동의 = 본인 | **틀림**. **법정대리인** 동의 (§22-2) |
| §17 동의 항목 = 4개 | **틀림**. **5개** (§17 의 경우 "받는 자" 추가) |
| 가명정보는 정보주체 동의 필요 | **틀림**. 통계·과학연구·공익기록보존 목적 시 **동의 없이** 가능 (§28-2) |
| 가명정보 결합은 자유 | **틀림**. **지정 전문기관** 만 수행 (§28-3) |
| 가명정보에 열람·정정·삭제권 적용 | **틀림**. **§28-7 적용 제외** |
| CPO 지정 의무 = 일정 규모 이상만 | **틀림**. **모든 개인정보처리자** (§31). CISO 와 다름 |
| 처리방침 필수 항목 = 5개 | **틀림**. **7개** (§30 ①항 1호~7호) |
| 안전성 확보조치 = 정통망법 | **틀림**. **[개인정보보호법] §29 + 안전성 확보조치 기준 고시** |
| 비밀번호 양방향 암호화 의무 | **틀림**. **일방향 (해시)** 의무 |
| 열람 처리 기한 = 30일 | **틀림**. **10일 이내** (§35 ③항) |
| 전송요구권 = 2020 신설 | **틀림**. **2023.3.14 개정 신설** (§35-2) |
| 자동화 결정 권리 = 2023 시행 | **틀림**. 2023.3.14 개정 / **2024.3.15 시행** (§37-2) |
| 법정손해배상 = 1천만원 이하 | **틀림**. **300만원 이하** (§39 ③항) |
| 징벌적 손해배상 = 3배 이하 | **틀림**. **5배 이하** (2023 강화, §39 ④항) |
| PIA 민간 의무 | **틀림**. **공공기관** 의무 (§33). 민간은 자율 |
| 유출 시 정보주체 통지 = 1천명 이상 | **틀림**. 통지 = 모든 유출 (§34 ①항). **신고**가 1천명 이상 (§34 ③항 + 시행령 §39) |
| 보호위원회 = 대통령 소속 | **틀림**. **국무총리 소속 중앙행정기관** (2020 데이터 3법 격상) |
| 보호위원회 위원 수 = 7명 | **틀림**. **9명** (위원장 1 + 부위원장 1 + 위원 7) |
| 과징금 상한 = 매출 1% | **틀림**. **3% 이하** (§64-2, 2023 강화) |
| §59 위반 처벌 = §72 | **틀림**. **§71** (5년 이하 / 5천만원 이하) |
